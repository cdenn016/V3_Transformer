# Experiment Criteria

Score each applicable criterion from 0 to 20: hypothesis and endpoint definition, protocol fidelity, data provenance, configuration identity, seed and split control, statistical treatment, reproduced-output agreement, robustness, and alternative explanations.

The experiment verifier reproduces the claimed output or runs a current mechanical check under the recorded configuration. The skeptic checks configuration drift, data leakage, insufficient repetitions, selective reporting, broken baselines, and mismatch between the observed endpoint and the stated conclusion.

Code and experiment closure requires current `mechanical` or `reproduced_output` evidence. Do not claim benchmark improvement unless repeated with-skill and without-skill runs have actually been executed and preserved. A single run, a recalled metric, or a stale artifact cannot close the claim.
