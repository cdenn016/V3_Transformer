# 2026-07-18 edits

## Outer-objective phi pullback-group descent completion

The opt-in `m_phi_update_mode="pullback_group"` path now consumes the phi
covector produced by the existing completed outer supervised scalar. That
scalar is cross-entropy plus the regularizers already included by the training
loop; its phi gradient reaches the geometry seam only after accumulation,
GradScaler unscaling, clipping, and the established nonfinite guards. The
default `m_phi_update_mode="adamw"` route and the separate `omega_direct`
route retain their established optimizer topology and state.

This is not an implementation of the canonical fixed-returned-state VFE phi
M-step. The single `TODO(canonical-vfe-phi-mstep)` at the completed-loss seam
marks that later objective. The implemented geometry converts the current
outer-loss coordinate covector through the strict exponential-map pullback
system, forms the left-trivialized velocity, applies the learning-rate-weighted
right-group factor, and accepts a BCH4 chart candidate only after trust,
factor-radius, and exact right-product residual checks. The numerical work is
performed in an autocast-disabled float64 island and committed through one
float32 cast.

The previous stateful heavy-ball and coordinatewise Adam phi modes, their
public controls, and their optimizer moment slots are removed. Serialized
compatibility recognizes only the three retired names at the typed migration
boundary. A legacy false natural-gradient flag migrates to AdamW; a legacy true
stateful optimizer is rejected before mutable resume state is changed, while
weight-only loading remains available. Raw historical fingerprints are
verified before effective-config migration. Two older generation tests were
updated to assert the intended effective migrated configuration rather than a
stale two-key raw mapping.

## Exact hot-path remediation

The strict pullback kernel now reuses the certified largest damped eigenvalue
for the operator norm when the prepared Gram is orthonormal and retains the
literal spectral-norm fallback for nonorthonormal bases. The left and right
exponential differentials share one matrix-power recurrence without changing
the adaptive order or tail certificate. A bounded 32-entry weak preparation
cache keys identity, tensor version, device, dtype, irrep partition, and
coordinate layout; generator tensors requiring gradients bypass it.

Registered row-major equal direct-sum blocks are evaluated as one exact batch
only when their bases, sizes, row shapes, and first certified series orders
match. Mixed orders, validation exceptions, broadcasting, unequal blocks,
nonregistered layouts, nonidentical bases, scalar inputs, and generator
autograd retain the sequential route and its exception precedence. Custom
registered directions receive the established float64 device-aligned basis;
only the two cache-aware built-ins explicitly preserve source-basis identity.
Positional-only `coordinate_layout` parameters are no longer misclassified as
keyword-compatible. Independent code and mathematical re-review found no
remaining Critical, Important, or Minor issue in these changes.

## Scaling restart completed-record preservation

The scaling restart gate conflated two different questions: whether a cell is
a complete, internally bound record and whether it was produced by the current
Git checkout. It also compared raw serialized configuration dictionaries. The
active-inference field retirement and phi-optimizer field migration therefore
made an otherwise equivalent saved configuration look different, while the
bf16 diagnostic fix changed the Git identity. A restart classified every such
cell as stale and entered `_reset_stale_scaling_artifacts`, which removes the
runner-owned completed files before starting the replacement run.

Restart handling now applies the strict serialized-config migration before
comparing effective configurations. It still requires a schema-2 cell, a valid
reuse-contract digest, an internally consistent finite metric payload, matching
train/validation/test source identities, and matching persisted provenance.
An exact current-code match remains `CACHED`. A complete matching experiment
from another code identity is `RECORDED`: it is skipped rather than overwritten,
its saved provenance is retained, and `scaling_design.json` records
`cache_status=recorded_prior_code`. The downstream analysis rule that rejects
mixed-code aggregation is unchanged.

Automatic restart no longer deletes or overwrites an existing cell attempt.
The destructive `_reset_stale_scaling_artifacts` path and its `unlink`/`rmtree`
operations were removed. Cached and recorded cells receive no per-cell writes.
When a partial, failed, corrupt, or incompatible owned attempt must be rerun,
the entire directory, including unrecognized user files, is moved intact to
`<route>/<label>/.attempts/s<seed>__attempt_NNNN/`. Training then receives a new
canonical `s<seed>` directory, satisfying `RunArtifacts`' fresh-directory
contract without data loss. Scaling analysis ignores `.attempts` archives so a
preserved historical summary cannot become a duplicate active observation.
Before a new invocation creates `scaling_design.json`, any prior manifest is
moved byte-for-byte into
`.invocations/scaling_design__invocation_NNNN.json`; an unsafe archive path
fails closed instead of permitting replacement.

K60 seed-6 record uses retired serialized fields and a prior code identity, and

## Fail-closed structural self-link wrappers

Structural self-link correction now requires an explicit Boolean certificate
on a RoPE wrapper as well as on its factored base. Arbitrary direct wrappers
default false, oracle conversion preserves an existing certificate, and the
compact full-gauge composition copies only the wrapper certificate. The
same-frame model and diagnostic construction seams certify only an already
certified base; the filtered query/key seam remains false.

Reflection preserves certification only when the key sign table is omitted
and therefore shares the query table. Supplying any explicit key table clears
the certificate, including an equal-valued table. Regression coverage pins
the supported model-RoPE path, nonorthogonal and off-block direct wrappers,
implicit and explicit reflection behavior, and strict Boolean validation for
all three certificate-bearing containers. The focused matrix records 287
tests, zero failures, zero errors, and one skip; `git diff --check` exits zero.
