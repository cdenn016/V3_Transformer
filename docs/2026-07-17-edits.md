## Ten-agent post-remediation deep audit

Exactly ten expert agents reviewed the executable code after the first July 16
audit remediation. Their independent scopes covered runtime reachability,
numerical analysis, differential geometry, information geometry, variational
semantics, transformer and attention behavior, process and filesystem
contracts, data and artifact integrity, performance and concurrency, and
cross-cutting integration. The full finding and disposition record is in
`docs/audits/deep-audit-codebase-2026-07-16.md`.

The audit closed further defects in experiment exit status, analysis-process
isolation, full-SPD certification and distance, represented transport inverses,
head-mixer invertibility, full model-channel covariance, Fisher precision,
inner-energy naming, explicit training versus evaluation semantics, cache
bounds, Windows process cleanup, scaling and ablation ownership, figure
manifest deletion authority, Windows reserved slugs, raw-resume and tokenizer
provenance, atomic metrics publication, and report memory accounting. The
report guard now budgets full pair matrices and twelve float32-equivalent
coordinate worksets, including the reachable near-KL float64 branch.

A final Windows review found that the new atomic durability barrier reopened
temporary files as read-only before `os.fsync`. Since Windows implements that
flush through a write-capable operation, it could have blocked every new run
during initial artifact publication. Atomic replacement now reopens the
temporary as `r+b`, flushes it, and removes it on pre-publication failure.
The non-Windows directory durability flush now runs after and outside the
destination-lock retry loop, so a post-rename flush error cannot trigger a
second rename from an already-consumed temporary.
Ablation cache admission and collection also require the exact ownership
sentinel, so a setup-rejected stale marker cannot make a sweep appear complete.

The EFE implementation remained excluded and unchanged. The approved slow UMAP
test deletions remain accepted. Per the user's instruction, no pytest, Python,
Ruff, compilation, import, GPU, or executable smoke command was run. Static
verification consisted of diff checks, conflict-marker and call-site searches,
and independent source review; no runtime pass count is claimed.

## Ablation `Sequence` import repair

The July 16 ablation-contract remediation added runtime-evaluated
`Sequence[...]` annotations but left `Sequence` out of the module's `typing`
import. Since `ablation.py` does not postpone annotation evaluation, importing
the module failed at the first affected function definition with
`NameError: name 'Sequence' is not defined`. The import now includes
`Sequence`; no configuration dictionary or setting changed.

The focused regression was run red and green against
`tests/test_ablation_sweep_route_compatibility_20260711.py`. Before the fix,
pytest collection stopped at `ablation.py:2465` with the reported `NameError`.
After the fix, all five tests in that file passed. The nine emitted warnings
were existing configuration-validity disclosures from `VFE3Config`, not test
failures.
