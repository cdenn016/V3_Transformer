# 2026-07-17 edits

## VFE 4.0 modular architecture and build plan

A six-lens expert panel converted the exact-ELBO white paper into a greenfield
software architecture for a new sibling repository, `VFE_4.0`. The panel
covered distribution families and information geometry, structured variational
inference, principal and associated bundles, numerical linear algebra, runtime
and configuration architecture, and causal language modeling. The resulting
design and phased implementation plan are recorded in
`docs/superpowers/specs/2026-07-17-vfe4-modular-architecture-design.md` and
`docs/superpowers/plans/2026-07-17-vfe4-modular-build-plan.md`.

The selected architecture is an immutable typed generative and recognition
graph, compiled once against instance-scoped frozen component catalogs and
lowered into resolved inference and numerical backends. It keeps the normalized
model and ordinary ELBO fixed while allowing families, mixture constructions,
gauge groups, representations, irreducible decompositions, fiber maps,
transports, source priors, hierarchy profiles, and numerical methods to be
added without editing runtime call sites. Natural and expectation coordinates
are first-class; the structured Gaussian core uses information form, while
moment coordinates remain a derived view for decoding and diagnostics.

Two independent post-draft audits tightened the contracts before publication.
Recognition factors must now form either one normalized joint law or an acyclic
ordered disintegration whose nonoverlapping target blocks cover the complete
latent event exactly once. Exponential-family membership is separated from a
certified Legendre-dual chart, generalized-EM acceptance must clear declared
evaluation uncertainty, and connection implementations join the required
extension seams. The implementation plan contains 36 test-first tasks.

The build order now tests the language-model premise before optional
infrastructure. A bounded dense-CPU B0-versus-minimal-`pq` pilot follows the
causal ELBO reference and records a go, revise, or stop decision before
structured production, CUDA, sparse, gauge, hierarchy, mixture, or irreducible
representation work. Direct dense and block-banded RTX 5090 execution is a
separate later promotion gate, followed by sparse and matrix-free paths.

Both independent reviewers reran their targeted checks after these revisions
and returned pass results. The plan contains 204 unchecked execution boxes,
balanced code fences, no duplicate create actions, no modify-before-create
actions, and valid local links. Strict UTF-8 and the repository's banned
wording, spelling, markup, placeholder, and extra-pytest-quiet scans also
passed for all three edited documents.

The plan preserves a dense float64 reference path and requires every scalable
or approximate path to state its objective, variational-family, evaluation,
and update guarantees separately. It also fixes the causal language-model
boundary: training recognition may assimilate the observed token, but
prior-predictive scoring and generation cannot access it. Cross-entropy remains
the normalized categorical observation term inside the ELBO. A direct KL is
permitted only between laws on the same declared event space; different
hierarchy or channel dimensions require a normalized conditional or an
explicit typed pushforward.

No VFE 4.0 implementation code, configuration, experiment, or Research-vault
file changed in this planning phase. Before the documentation edits, the clean
`origin/main` baseline suite produced a JUnit record with 3,772 tests, 14
failures, zero errors, and 17 skips; the 14 failures therefore predate this
documentation-only branch. Git publication evidence is recorded at closeout
rather than being inferred from that red baseline.

## Global ELBO and language-model observations investigation

A focused multi-agent theory and executable-path investigation examined whether
the model-channel refinement, belief refinement, and decode update can be
coordinates of one global ELBO, and whether language-model training tokens can
serve as observations in that construction. The resulting report is
`docs/2026-07-17-global-elbo-language-model-observations-investigation.md`.

The report treats the complete VFE rather than reducing the question to the
peer-belief KL sector. It maps the self, hyperprior, belief and model consensus,
attention-entropy, gauge, adaptive-precision, and observation terms to their
possible probabilistic roles. It also accepts and analyzes the objection to a
mean-field default. The existing product-family obstruction is not presented as
a universal no-go; structured sequential, sparse joint-Gaussian, Bethe/Kikuchi,
distributed-consensus, and belief-configuration Gibbs routes are distinguished.

The investigation concludes that tokens can be observations, with categorical
cross-entropy serving as the discrete observation NLL. The current node-marginal
functional is not yet a structured state-level ELBO because it has no joint
entropy, cross-token covariance or edge beliefs, or marginal-consistency
constraints. A structured causal ELBO is recommended for a conventional
state-level claim, while the existing configuration-space Gibbs lift is the
exact route that preserves the non-observation population energy as a
configuration prior and attaches the observation likelihood once. No source,
configuration, test, or training behavior changed. Verification for this
documentation-only addition is limited to link and source-location checks,
`git diff --check`, and independent expert review; no test pass count is
claimed.

## Ten-agent post-remediation deep audit

Exactly ten expert agents reviewed the executable code after the first July 16
audit remediation. Their independent scopes covered runtime reachability,
numerical analysis, differential geometry, information geometry, variational
semantics, transformer and attention behavior, process and filesystem
contracts, data and artifact integrity, performance and concurrency, and
cross-cutting integration. The full finding and disposition record is in
`docs/audits/deep-audit-codebase-2026-07-16.md`.

The audit closed further defects in experiment exit status, analysis-process
isolation, full-SPD certification and distance, represented transport inverses,
head-mixer invertibility, full model-channel covariance, Fisher precision,
inner-energy naming, explicit training versus evaluation semantics, cache
bounds, Windows process cleanup, scaling and ablation ownership, figure
manifest deletion authority, Windows reserved slugs, raw-resume and tokenizer
provenance, atomic metrics publication, and report memory accounting. The
report guard now budgets full pair matrices and twelve float32-equivalent
coordinate worksets, including the reachable near-KL float64 branch.

A final Windows review found that the new atomic durability barrier reopened
temporary files as read-only before `os.fsync`. Since Windows implements that
flush through a write-capable operation, it could have blocked every new run
during initial artifact publication. Atomic replacement now reopens the
temporary as `r+b`, flushes it, and removes it on pre-publication failure.
The non-Windows directory durability flush now runs after and outside the
destination-lock retry loop, so a post-rename flush error cannot trigger a
second rename from an already-consumed temporary.
Ablation cache admission and collection also require the exact ownership
sentinel, so a setup-rejected stale marker cannot make a sweep appear complete.

The EFE implementation remained excluded and unchanged. The approved slow UMAP
test deletions remain accepted. Per the user's instruction, no pytest, Python,
Ruff, compilation, import, GPU, or executable smoke command was run. Static
verification consisted of diff checks, conflict-marker and call-site searches,
and independent source review; no runtime pass count is claimed.
