- **CR-1** (`run_artifacts.py`): `load_checkpoint` now loads with `torch.load(weights_only=True)` by
  default; a bundle that fails the safe load raises unless the new `trust_resume_checkpoint` config
  flag (default `False`) opts into the legacy `weights_only=False` path. Field added in `config.py`
  and excluded from resume-drift warnings.
- **CR-2** (`config.py`): `__post_init__` now rejects `decode_precision_scaled=True` with a
  full-covariance family on the linear-decode path (the elementwise `mu/sigma` would shape-crash at
  the first decode).
- **DBG-2** (`train.py`): gradient accumulation is now token-weighted (each microbatch's mean loss is
  weighted by its valid-token fraction `n_mb/n_tot`); byte-identical to the old uniform `1/K` on the
  default unpadded loader (equal counts), correct under uneven ignore-padding.
- **DBG-3** (`train.py`): `parameter_report` records the probe exception (`probe_error`) instead of
  swallowing it; the init banner now prints `[live/dead probe failed]` so a probe failure is
  distinguishable from a genuinely all-live config.
- **PERF-1** (`train.py`): the per-step `train_ce` D2H sync is skipped on silent (non-logged) steps;
  the loss sync stays (it feeds the NaN-skip guard and the return value).
- **PERF-2** (`prior_bank.py`): the three chunked-CE kernels use a device-side
  `(.../ valid.sum().clamp_min(1))` masked mean instead of a Python branch on `valid.sum()==0`,
  removing a per-step host sync (value/gradient identical).
- **PERF-3** (`model.py`): the dense-decode CE uses the same branchless masked mean (sum-reduction /
  clamped count) instead of `if (flat_targets != -100).any()`.
- **PY-1** (`gauge_optim.py`): `GaugeNaturalGradAdamW.step` now rejects a non-`None` closure
  (`NotImplementedError`) and calls `super().step()` with none, making the latent double-step footgun
  explicit (no caller passes a closure today).
- **REF-2** (`transport.py`, `model.py`, `e_step.py`): `register_transport` carries `needs_mu`/
  `needs_sigma` state-routing metadata; `_REGIME_NEEDS_MU/SIGMA` and the e_step literals now derive
  from it instead of hard-coded mode-name tuples (add-by-registering contract; derived sets verified
  byte-identical to the old literals).
- **REF-4** (`prior_bank.py`, `config.py`, `model.py`): `register_decode` carries `is_full`/`chunked`
  metadata; the config rank cross-check and the model's fused-CE dispatch read `_FULL_DECODERS`/
  `_CHUNKED_DECODERS` instead of literal name tuples.
