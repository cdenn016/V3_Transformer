# July 12, 2026 edits

## GitHub README mathematics compatibility

The repository landing page contained twelve unresolved math-renderer elements. GitHub reported ten instances of the unsupported `\operatorname` macro, one double-subscript diagnostic, and one extra-close-brace-or-missing-open-brace diagnostic. The latter two diagnostics were secondary effects of Markdown interpreting paired literal asterisks in superscripts before the expressions reached the math renderer. Live validation of the task branch also exposed one set delimiter written as `\left\{...\right\}`: Markdown removed the brace escapes and left the math parser with an invalid `\left` delimiter.

The README now uses GitHub-compatible roman operator labels through `\mathrm`, with the conventional abbreviation `\mathrm{im}` for the image of the exponential map. Every mathematical star, including pushforward subscripts and superscripts denoting stationary or optimized values, now uses `\ast` instead of a literal asterisk. Set expressions use `\lbrace` and `\rbrace`, including the scalable `\left\lbrace...\right\rbrace` form, which avoids Markdown escape removal. These substitutions change only the TeX spelling and rendered typography; the definitions, derivations, symbols, and mathematical claims are unchanged.
