# July 13, 2026 edits

## Scope

This change set resolves the 41 findings that the two July 13 audits investigated, reproduced, and retained as confirmed findings. The work was performed from `origin/main` commit `1b58d4f53ca7786f45b27ad4e70ab7fd85df9c01` in the isolated branch `codex/fix-validated-7-13-audits`. The design and implementation plan are recorded in `docs/superpowers/specs/2026-07-13-validated-audit-remediation-design.md` and `docs/superpowers/plans/2026-07-13-validated-audit-remediation.md`.

## Artifact, reporting, and tooling changes

Run finalization now isolates individual figure failures and closes figures opened by a failed renderer, while provenance hashes large files and token tensors incrementally. Scaling resumption binds cached cells to corpus identities, and shuffled training loaders use explicit per-cell generators whose cache identity includes the actual data seed. The scaling bootstrap now applies the same estimator acceptance rule to every replicate that it applies to the reported point estimate. Nonfinite wall times are excluded from Pareto input, policy output is ignored by Git, package discovery matches only `vfe3` and its subpackages, and the stale audit verifier node was repaired.

The duplicated path sanitizers were replaced by one allowlist-based, length-bounded filesystem slug helper. Manual figure generation now honors an explicit large-figure override or inherits the saved run policy. UMAP report generation retains one isolated worker for multiple embeddings, uses a writable worker cache, publishes status atomically, and still keeps the library import outside the parent process. The obsolete free-energy-descent figure registry entry and unused imports were removed.

## Runtime and state changes

The training and model paths now log head-mixer diagnostic failures, avoid unnecessary host synchronization in conjugate-gradient diagnostics, share identical model-channel encodings, and reuse one first-sequence post-evaluation diagnostic snapshot for beta and gamma maps. Single-run reporting uses the same captured belief and static/refined model-channel state across its ten same-token extractors instead of replaying belief or model-channel inference. EFE scoring rejects an empty term set and preserves tensor semantics. EMA loading reconciles its shadow with the live trainable parameter set, and EMA copies recompute the barycenter-derived hyper-prior state before evaluation or checkpointing. The gamma prior uses the same exact `log_softmax` entropy form as the belief channel, and the belief-cache regression covers all three supported causal priors.

Configuration now rejects nonfinite or nonpositive covariance trust radii and Metropolis temperatures on every active reflection path. The documented `sigma_max=None` pure path reaches metrics and figures without numeric-ceiling assumptions. Linear decoding warns when `decode_tau` is inert, and query-adaptive temperature is identified and warned as an opt-in noncompact-GL(K)-breaking baseline while its disabled pure path remains unchanged.

## Geometry and numerical changes

Regime II retains an explicitly supplied connection under the trivial gauge. The diagonal covariance trust region now uses the same whole-tangent ball geometry as the full-covariance Frobenius constraint. Diagonal Gaussian KL and Renyi calculations retain float64 inputs, failed safe-Cholesky masks propagate instead of producing finite partial-factor results, and covariant edge log determinants use the guarded Cholesky helper. ALiBi slopes now follow the reference non-power-of-two construction.

Full-covariance transport batches equal irrep blocks and preserves the heterogeneous fallback. Mahalanobis normalization uses an exact float64 solve when possible and labels its regularized fallback as approximate. Dispersion transport is routed through the selected family, so Laplace scale follows a degree-one law rather than a Gaussian variance law. Belief-spectrum diagnostics retain negative eigenvalues, emit an infinite condition number and a positive-definiteness flag for invalid covariance, and clamp only the effective-rank calculation. Sigma-gate evaluation now fails closed before producing small-sample NaNs and writes strict JSON.

## Regression and test-policy changes

New regression files cover the artifact/tooling, runtime/state, geometry/numerics, and integration corrections. CPU model fixtures were reduced below `K=6` wherever the representation itself did not require a larger dimension. Representation-specific SO(3), symplectic, isotypic, coupling-graph, and data-only cases retain their mathematically required dimensions. Golden values affected only by this fixture reduction were recaptured from the unmodified baseline commit before the fixes were applied.
