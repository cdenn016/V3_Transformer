# Memo — debate-expert-ml-engineer — BLUE — round 1 — blocks-k48-gauge-block-scaling-axis

## Verdict
The empirical core is defensible and strong: at fixed `embed_dim=48` and **fixed 245.76M tokens**, enlarging the GL gauge block lowers CE monotonically and seed-robustly (PPL 124.6 -> 92.2, per-label std <= 1.08), and the added capacity lives in a token-indexed table that contributes one active row per token — the recognized total-vs-active pattern of sparse / conditional-computation models. I must **narrow** two sub-clauses: "publishable *scaling exponent*" (axis-dependent on a 3.62x range — not a physical exponent) and the word "parameter-efficient" (it holds on active-params/compute, not on total params, and the cross-sweep frontier comparison inherits a 2x token confound).

## Argument (steelman + evidence)

**1. The within-sweep effect is clean, large, and confound-free on data.** All five `blocks_K48` cells train on the *same* 245.76M tokens, so the monotone PPL descent 124.6 -> 112.4 -> 106.2 -> 99.8 -> 92.2 is not contaminated by any token-budget difference — the half-token confound bites only the *cross-sweep* comparison to `grow_K_GL10`, not the internal axis. Per-label std <= 1.08 sits far below every adjacent gap. The GL3 -> GL8 segment is Pareto-improving on the *empirical* frontier: PPL drops 124.6 -> 106.2 while wall-time *falls* 6366s -> 4657s. Better loss at lower wall-clock on equal data is an efficiency signal, not a metric artifact.

**2. The extra parameters are a lookup table, one active row per token — the canonical total-vs-active pattern.** `phi_embed` is a `(V, n_gen)` parameter (`vfe3/model/prior_bank.py:167`) read by token-indexed row lookup (`prior_bank.py:682`); the closed-form active working set `5·V·K + 2·K + n_gen = 12,061,776 + n_gen` moves by only +1,008 (+0.008%) across the whole sweep (`vfe3/run_artifacts.py:616-620`). This is exactly the architecture the sparse-model canon legitimized:
  - *Shazeer et al. 2017, "Outrageously Large Neural Networks" (ICLR 2017):* a MoE "with up to 137 billion parameters," where "Conditional computation, where parts of the network are active on a per-example basis, has been proposed ... as a way of dramatically increasing model capacity without a proportional increase in computation," yielding ">1000x improvements in model capacity with only minor losses in computational efficiency." A vocab-indexed table read one row per token is the degenerate limit of this pattern.
  - *Fedus, Zoph & Shazeer 2021, "Switch Transformers" (JMLR 2022):* trillion-parameter models "with outrageous numbers of parameters -- but a constant computational cost," giving "7x increases in pre-training speed with the same computational resources." Decoupling total-param count from per-token compute is the whole point; `active_params_per_token` staying ~12.06M while total params grow 3.62x is that decoupling, not a bug in the metric.

**3. Canonical scaling-law practice already tells us `n_params` is the wrong x-axis here.** *Kaplan et al. 2020, "Scaling Laws for Neural Language Models":* the clean parameter power law is fit against the *non-embedding* parameter count, precisely because vocab/embedding tables do not scale like compute and muddy the trend. `phi_embed (V, n_gen)` is a vocab-size table; by Kaplan's own convention it belongs in the excluded-from-N bucket. So the "vertical column in CE-vs-active/tok" is the *expected* signature of a well-behaved conditional-computation axis, and the matched-active / matched-wall comparison (GL24: CE 4.523 at active 12.06M vs grow K50: CE 4.544 at active 12.56M) is the correct frame — on which GL24 sits on/below the grow frontier *despite half the tokens*.

**4. The axis is genuinely new and orthogonal to the published one.** `blocks_K48` holds `embed_dim` fixed and moves the gauge partition; `grow_K_GL10` holds the block fixed and grows width. The former is absent from both manuscripts. A complementary knob has scientific value independent of whether its headline exponent is reportable. And one confound is *ruled out* by the data: the head mixer *shrinks* 16x16 -> 2x2 as blocks enlarge (`01_evidence.md:34`), opposite to the loss gain, so it is not the driver.

## Concessions / limits
- **Total-parameter efficiency: conceded.** At matched *total* `n_params`, `blocks_K48` is *less* efficient than `grow_K_GL10` by ~+4.8 to +12.8 PPL at all five matched points. "Parameter-efficient" is only defensible on the active/compute axis, and must be stated that way.
- **"Publishable scaling exponent": conceded as the weakest clause.** The offset alpha is 0.929 vs `n_params`, 0.181 vs `n_gen`, degenerate (R^2=0.17) vs analytic-FLOPs — axis-dependence on identical CE data, a property of a compressed 3.62x x-range, not a physical exponent (Kaplan: a data-/range-limited fit is a slice, not the infinite-data exponent). Report a *curve/ablation*, not an exponent.
- **Cross-sweep frontier claim inherits a 2x token confound.** *Hoffmann et al. 2022 (Chinchilla):* `L(N,D) = E + A/N^alpha + B/D^beta`; "for every doubling of model size the number of training tokens should also be doubled" (~20 tokens/param), and Chinchilla 70B on 4x data beats Gopher 280B. A 2x gap in D is first-order for any absolute-loss comparison, so "sits on/below the grow frontier" is unproven until matched at 491.52M tokens.
- **Gauge-structure-per-se not isolated:** no non-gauge `V x m` table of matched size was trained, so raw-capacity vs gauge-block is unresolved.

## Falsification conditions
The claim sinks if: (a) a **matched-token-budget (491.52M) `blocks_K48` run** erases or substantially shrinks the GL3 -> GL24 gap; (b) a **non-gauge control** — a plain `(V x m)` learned table (or matched-param dense readout) of matched total size — reproduces the CE curve, showing raw added capacity, not gauge structure, is the driver; (c) on a **calibrated wall-clock/FLOP frontier at equal tokens**, GL24 (the slowest run) sits *above* the grow frontier; or (d) rescaling `phi_scale ∝ 1/sqrt(n_gen)` to hold the phi-row init variance fixed collapses the gain, implicating init conditioning rather than gauge capacity.

## Newly-discovered canon (for 01b_extended_evidence.md)
- **Shazeer et al. 2017, "Outrageously Large Neural Networks: The Sparsely-Gated Mixture-of-Experts Layer," ICLR 2017 (arXiv:1701.06538).** MoE up to 137B params; conditional computation increases capacity "without a proportional increase in computation," ">1000x ... model capacity with only minor losses in computational efficiency." Anchors the total-vs-active legitimacy of `phi_embed`'s one-row-per-token read.
- **Fedus, Zoph & Shazeer 2021, "Switch Transformers: Scaling to Trillion Parameter Models with Simple and Efficient Sparsity," JMLR 2022 (arXiv:2101.03961).** "Outrageous numbers of parameters -- but a constant computational cost"; 7x pretraining speedup at fixed compute. Anchors decoupling of total-param count from per-token compute and the matched-compute comparison frame.
- **Kaplan et al. 2020, "Scaling Laws for Neural Language Models" (arXiv:2001.08361).** Clean power law is fit on *non-embedding* parameter count; data-/range-limited fits are slices, not exponents. Supports both the "active/compute is the right x-axis" defense and the "compressed-range exponent is not physical" concession.

## Confidence
MEDIUM-HIGH on the within-sweep empirical effect and the total-vs-active framing; LOW on "publishable scaling exponent." A matched-token-budget `blocks_K48` run plus a non-gauge matched-size control would move me to HIGH or sink the efficiency claim outright.
