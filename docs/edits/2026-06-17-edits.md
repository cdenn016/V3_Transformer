# V3_Transformer edits — 2026-06-17

## Per-head reliability for precision-weighted attention (`precision_attention_per_head`, default OFF)

Follow-up to the 2026-06-16 `precision_weighted_attention` toggle, which empirically helps a bit
(while `decode_precision_scaled` slightly hurts -- so `Sigma` carries signal in inference, not at the
readout). The base toggle uses a GLOBAL per-key reliability `-log(b0 + tr Sigma_j)` (trace over all K
coords), identical for every head. This adds a per-HEAD variant: the reliability is computed per gauge
block, `-log(b0 + tr_h Sigma_j)`, so each head down-weights keys by its OWN block uncertainty --
better aligned with the per-head attention structure, and plausibly a larger gain.

- `vfe3/config.py`: new `precision_attention_per_head: bool = False`. `__post_init__` warns it is
  inert when `precision_weighted_attention=False`.
- `vfe3/model/model.py`: `_precision_key_bias` gains an optional `irrep_dims` arg -- when given it
  returns the per-block traces `(B, N, H)` (via `sigma.split(irrep_dims)`) instead of the global
  `(B, N)`. `forward` branches: per-head reshapes to `(B, H, 1, N)` (per head, query-broadcast, per
  key); global stays `(B, 1, 1, N)`. Both fold (detached) onto the position `log_prior` and broadcast
  against the per-head energy `(B, H, N, N)`; the closed-form belief kernel stays exact.
- `train_vfe3.py` + `ablation.py`: toggle added to both config dicts (default OFF); `ablation.py`
  also gets a `precision_attention_per_head` sweep (`[False, True]`, requires
  `precision_weighted_attention=True`) so global-vs-per-head is one runnable cell.

Behavior: uniform-over-coords `Sigma` (e.g. at init) -> per-head block traces equal across heads/keys
-> bias constant -> softmax-absorbed (no effect), so OFF and the uniform case are byte-identical to
the global path. Only coordinate-varying `Sigma` makes per-head differ from global.

TDD: `tests/test_precision_toggles.py` (+6 tests: defaults, per-block split of the helper, no-effect
at uniform sigma, differs-from-global when coords vary, inert warning, grad-flow). Full suite:
**1081 passed, 1 skipped, 1 xpassed, 0 failures** (junit tests=1083).

## Per-head attention-temperature sweep in `ablation.py` (`kappa_beta_per_head`)

`kappa_beta` per-head tau (tau_h = kappa_beta[h]*sqrt(d_head)) was already fully wired through the
beta-build kernel (`free_energy.attention_tau` emits a `(H,)` tau when `kappa_beta` is a list); it was
just never exposed as a sweep. Added a `configs`-shape sweep (the only shape that handles list values
+ gives clean labels): 5 arms, all length-2 lists for the baseline n_heads=2 block_glk, **mean held at
1.0** so it isolates per-head ASYMMETRY from the global-temperature axis the existing scalar
`kappa_beta` range sweep already covers. Arms: uniform_1.0 [1,1] (control), split_0.8_1.2,
split_1.2_0.8, split_0.6_1.4, split_1.4_0.6. Verified: each arm builds a valid VFE3Config;
`uniform_1.0` -> tau [3.162, 3.162] byte-identical to the scalar kappa=1 baseline; `split_0.6_1.4` ->
tau [1.897, 4.427] (per-head differs). No `requires` (baseline block_glk already supports the list).
Run via `CONFIG["sweep"]="kappa_beta_per_head"`. No production code touched.

## EMA / Polyak weight averaging (`use_ema`, default OFF)

New default-OFF toggle averaging the trained tables (prior/decode params + learned scalars). New
module `vfe3/ema.py`: `EMA` class tracking `requires_grad` params, `s <- decay*s + (1-decay)*theta`
(`update`), with `store`/`copy_to`/`restore` for the eval-time weight swap and `state_dict`/
`load_state_dict` for resume persistence.

- `vfe3/config.py`: `use_ema: bool=False`, `ema_decay: float=0.999`; `__post_init__` requires
  `0 < ema_decay < 1` only when `use_ema=True` (inert/unvalidated when off).
- `vfe3/train.py`: build `ema` before the resume load (so a resume restores the shadow, not re-seeds
  it); `ema.update(model)` after each `train_step`; in the `do_eval` block `store`+`copy_to` then
  `restore` (so val PPL, `best_model.pt`, samples and diagnostics all use the AVERAGED weights, and
  the live SGD weights return before the next step); a final `copy_to` so the returned model is the
  averaged one. EMA reads params only -- no RNG, no grad/optimizer touch -- so the SGD trajectory (and
  the `losses` history) is byte-identical to the OFF path.
- `vfe3/run_artifacts.py`: `save_checkpoint`/`load_checkpoint` gain an `ema` kwarg and persist/restore
  `ema_state` (mirroring `scaler`); old/EMA-off bundles carry `None` and skip restore.
- `train_vfe3.py` + `ablation.py`: `use_ema`/`ema_decay` added to both config dicts (default OFF);
  `ablation.py` gets a `use_ema` sweep ([False, True]) and an `ema_decay` sweep
  ([0.99, 0.999, 0.9995], requires use_ema=True).

TDD: `tests/test_ema.py` (+15: shadow tracks only requires_grad params, convex-blend update math,
store/copy_to/restore round-trip, state_dict round-trip, config defaults+validation, training
trajectory unperturbed (losses identical on/off), final model changes, eval-swap restores, and a
resume-equivalence test pinning the shadow round-trips through a checkpoint). Full suite:
**1105 tests, 0 failures, 0 errors, 1 skipped**.
