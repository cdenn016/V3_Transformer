## Fixes applied

- **M1 (Medium)** — decode boundary is fixed at alpha=1 KL while the docstring claimed it tracks the
  configured divergence. (a) `vfe3/model/prior_bank.py`: rewrote the class docstring to state the
  decode is a FIXED alpha=1 KL on a hardcoded Gaussian family (honored only at covariance-structure
  granularity), matching the code (`reference_decode`/fused kernels call `divergence.kl`, never read
  `cfg.alpha_div`/`cfg.divergence_family`). (b) `vfe3/config.py` `__post_init__`: added a UserWarning
  when `use_prior_bank=True and (alpha_div != 1.0 or divergence_family != 'renyi')`, so the
  inference/decode divergence mismatch is no longer silent (mirrors the existing decode_bias warning).
- **L1 (Low)** — `vfe3/model/model.py` `_amp_context` comment claimed fp16 is rejected at
  construction; corrected to state both 'bf16' and 'fp16' are accepted (`config.py` `_require`) and
  fp16 is loss-scaled by the GradScaler in `train.py`. Comment-only; code was already correct.
- **L5 (Low)** — `vfe3/families/base.py` `_renyi_from_log_partition`: gave the generic Renyi-from-A
  path the same fp32 catastrophic-cancellation guard the Gaussian closed forms have. Added module
  constant `_RENYI_KL_BAND = 1e-2` and a float64 island for `|alpha-1| < band` (outside the `<1e-6`
  KL switch). Pure-path correctness for a future custom exponential family. Probe near alpha=1: rel
  err 1e-2 -> ~3e-8 vs a float64 reference; outside the band unchanged.
- **L6 (Low)** — `vfe3/families/base.py` `squared_hellinger`: range doc `[0, 1)` -> `[0, 1]` (return
  comment + docstring) with an fp32-saturation note (`1 - exp(-50)` rounds to 1.0 at `kl_max=100`).
  Doc-only; math unchanged.
- **L7 (Low)** — `vfe3/families/gaussian.py` `DiagonalGaussian.renyi_closed_form` docstring: documented
  the `eps` contract — the kernel variance floor MUST equal the upstream belief sigma floor
  (`cfg.eps`), a deliberately looser, different floor than `natural()`/`entropy()`'s 1e-12. Doc-only
  (aligning the floors in code would decouple the kernel from the live retraction floor — rejected).
- **L8 (Low)** — `vfe3/numerics.py` `safe_spd_inverse`: rewrote the batched-cholesky try/except (which
  fell the WHOLE batch to `pinv` if any one element was non-PD) to per-element `cholesky_ex` with
  retry on the still-failed mask only, mirroring `safe_cholesky`. Latent (zero non-test callers).
  Probe: mixed batch keeps the well-conditioned element's exact Cholesky inverse, non-PD element
  finite via per-element pinv. Round-0 eps ridge preserved (byte-identical on well-conditioned input).
- **L2 (Low)** — `vfe3/geometry/rope.py` module docstring: narrowed the RoPE-as-gauge framing —
  `R(theta)` coincides with a structure-group gauge element only on full-`SO(d)` blocks; on higher
  `so_n`/`sp_n` irrep blocks it is a generic `SO(d)` element outside `rho(G)` (R enters only as the
  outer orthogonal similarity; config.py already warns at build). Doc-only.
