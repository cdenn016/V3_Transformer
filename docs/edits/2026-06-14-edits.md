## First non-Gaussian belief family — `DiagonalLaplace` (`laplace_diagonal`)

Registers the first genuinely-non-Gaussian belief family, the factorized Laplace (location `mu`, scale
`b`; exponential L1 tails, excess kurtosis 3, sufficient statistic `|x-mu|`). Chosen and corrected via
an adversarially-verified design workflow (info-geometry proposal → gauge-theory + numerical-analysis
critiques). **Honest finding:** the natural non-Gaussian candidates (Laplace, Student-t) are NOT
exponential families when the location varies (`|x-mu|` depends on `mu`), so there is no joint
log-partition `A(theta)` and the generic Bregman/Rényi-from-`A` path cannot serve them; the family
ships via the closed-form `renyi_closed_form` override seam instead (the dispatch already prefers it).
The generic A-path therefore remains exercised only by test fixtures — a true non-Gaussian EF would be
needed to close that sub-gap.
- `vfe3/families/laplace.py` (new): `DiagonalLaplace(BeliefParams)`, `cov_kind='diagonal'`. Implements
  the abstract surface; `natural()`/`log_partition_at()` raise `NotImplementedError` (non-EF, required
  so the class is instantiable and the generic-path error is explicit). Closed-form per-coordinate KL
  `log(b_p/b_q) + s/b_p + (b_q/b_p)exp(-s/b_q) - 1` and Rényi on `alpha in (0,1]` via the two-region
  exponential integral, with the removable singularity at `alpha* = b_q/(b_p+b_q)` handled by a float64
  island + `where` limit branch (the cancellation is per-coordinate/data-dependent, unlike the
  Gaussian's fixed band around `alpha=1`; float64 also covers the `alpha→1` outer cancellation).
  `alpha>1` non-convergent blend → NaN → `kl_max` (Gaussian non-PD policy). `renyi_per_coord`,
  `entropy = sum_k log(2 b_k e)`, `stack`, `block`, `broadcast_over_keys`.
- `vfe3/families/__init__.py`, `vfe3/divergence.py`: import `laplace` so `@register_family` runs
  (mirrors the gaussian registration). Config-selectable with zero validator edits (add-by-registering).
- Transport: rides the existing family-agnostic primitives unchanged (transport acts on raw `mu`/`sigma`
  before family construction). Distributionally exact only under permutation/sign gauge; under
  non-permutation rotation it is a marginal-scale projection — the same projection the diagonal Gaussian
  already incurs (only full-cov Gaussian KL is congruence-invariant under the live groups) plus a
  shape-level break. Documented in the module docstring; not claimed as exact equivariance.
- Routing: not kernel-eligible (`uses_kernel_route` requires `gaussian_diagonal`), so the belief
  gradient runs through the autograd-of-F oracle (the general path); decode stays the fixed Gaussian
  readout (the pre-existing M1 decode decoupling), a documented limitation for a belief-family swap.
- Closed forms validated to ~1e-10 against deterministic float64 trapezoidal integration in scratch

- Tests: `tests/test_laplace_family.py` (13) — registration/config-selectable; `natural`/`log_partition_at`

### Hardening pass + gradient bug fix (FD-of-F caught it)

Added gradient-correctness and device/edge coverage for the new family, per CLAUDE.md's FD-vs-oracle
discipline. The finite-difference check **found a real training-correctness bug** in the Rényi path.
- **Bug**: the `term2 = (e_q - e_p)/(c_p - c_q)` of the Rényi integral is a 0/0 at the removable
  singularity `c_p == c_q`. The float64 island kept the VALUE accurate, but the GRADIENT (quotient
  rule adds another `1/d`) catastrophically cancelled — so any pair with `b_q ≈ b_p` got a spurious
  belief gradient. The diagonal SELF-pair always sits there (`omega_ii ≈ I` ⇒ `d ~ 1e-7` fp noise,
  above the old `1e-12` limit threshold), and `alpha* = b_q/(b_q+b_p) ≈ 0.5` there, so it bit at
  `renyi_order ≈ 0.5` — i.e. also `bhattacharyya` / `squared_hellinger` on Laplace. The self-pair's
  true gradient is 0 (FD ~1e-7); autograd gave ~5e-3. KL (`alpha=1`, separate discrete branch) was
  unaffected, which is why only the Rényi FD test exposed it.
- **Fix** (`vfe3/families/laplace.py`, value-preserving): rewrote `term2` exactly as
  `exp(-c_m s) · s · sinhc(d s/2)` (`c_m=(c_p+c_q)/2`, `sinhc=sinh/x`) with the Taylor `sinhc`
  (`1 + x^2/6 + x^4/120`) for small `|x|` — stable in value AND autograd through the singularity, no
  `1/d`; the direct ratio is kept for larger `|x|` (where `sinh` would overflow), and `x` is zeroed
  off-branch so the dead Taylor branch never sees a large argument. Algebraically identical, so the
  divergence golden tests are unchanged.
- Tests (`tests/test_laplace_family.py`, +7, one CUDA-skipped here): FD-of-F vs the autograd oracle
  for `mu` and `sigma` at `alpha=1` and `alpha=0.5` (4th-order central stencil, the check that caught
  the bug); `alpha>1` divergent-blend → `kl_max` and convergent → matches reference; `kl_max`
  saturation; `s=0` cusp + sub-`eps` scale floor finite; and a CUDA-vs-CPU agreement + GPU model
  train step (gated on `torch.cuda.is_available()` — **skipped in this CPU-only run; not executed on
  the RTX 5090 here**).
