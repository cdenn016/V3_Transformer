# Edits ‚Äî 2026-06-11

## Early-morning pass ‚Äî VRAM sink fixes (flat / block_glk training path)

Follow-up to the 2026-06-10 evening VRAM investigation (15-agent verified sweep; record in
`docs/audits/audit-2026-06-11-vram.md`). At the user's operating point (K=60, 6 heads, N=128,
B=64) the training step held ~20.4 GB; the verified dominant sinks were the BCH positional
composition (~4.0 GB of bracket tensors pinned in the unrolled graph, 22 of them duplicate
broadcast copies of the same unbatched positional element), the dense linear-decode CE
(~3.3 GB of logits plus log-softmax retained for backward), the kernel's `beta_coord`
broadcast (one full B¬∑N¬≤¬∑K copy), and the float64 exp island firing on full K for
block-diagonal groups (~0.4‚Äì0.9 GB of f64 transients per build plus fp64-throughput
`matrix_exp` on a consumer GPU). All fixes are exact reassociations or dtype-island re-keying;
no formula changes, all pure paths intact.

- `vfe3/geometry/transport.py` ‚Äî `compute_transport_operators` and `build_factored_transport`
  now pass `exp_dim=max(block_dims)` to `stable_matrix_exp_pair`, keying the float64-island
  decision on the dimension actually exponentiated (the per-head block), mirroring the
  regime_ii edge exp and VFE_2.0's always-fp32 per-block exp. Conditioning argument: the phi
  retraction bounds the coordinate norm at `max_norm=5`, so each block is far inside fp32
  `matrix_exp`'s exact regime. `stable_matrix_exp_pair` itself is unchanged (its default
  full-K keying and the per-block bit-equivalence pin stand).
- `vfe3/geometry/lie_ops.py` ‚Äî `compose_bch`: (a) operands are broadcast to a common shape
  once before the Dynkin cascade, so all bracket matmuls save one shared storage instead of
  22 broadcast-materialized copies of the embedded positional element (bitwise-identical
  values); (b) new optional `block_dims`: for a multi-equal-block embedding the whole
  commutator cascade runs on the `(H, d, d)` diagonal-block stacks (extracted and scattered
  back with the `_blockwise_matrix_exp` view trick) ‚Äî identical values, 1/H the bracket
  memory, H¬≤ fewer matmul FLOPs. The cascade itself was factored into `_bch_dynkin_correction`
  (shared by the dense and blocked branches). `compose_euclidean`/`compose_phi` accept and
  forward `block_dims` (euclidean ignores it).
- `vfe3/model/positional_phi.py` ‚Äî `apply_positional_phi` passes `block_dims=group.irrep_dims`
  into `compose_phi`, activating the blocked cascade for block_glk / tied_block_glk / equal
  irrep towers (single-block and cross-coupled bases report `[K]` and keep the dense cascade).
- `vfe3/gradients/kernels.py` ‚Äî the diagonal kernel no longer consumes a materialized
  `(‚Ä¶, N, N, K)` `beta_coord`: `belief_gradients` passes the masked beta in its compact
  per-head form and the new `_pair_contract` contracts it against a head-shaped view of the
  pair difference (single-block and equal-block branches are view-only; unequal towers fall
  back to `_beta_to_coordinate`, which stays for that case and its test).
- `vfe3/model/prior_bank.py` ‚Äî new `decode_ce_linear_chunked`: the fused chunked-vocab CE for
  the `use_prior_bank=False` linear decode (`logits = mu @ W·µÄ (+ b)`), the same
  checkpointed streaming-logsumexp + target-gather contract as `decode_ce_diagonal_chunked`,
  so the `(B, N, V)` logits and cross-entropy's log-softmax copy are never materialized.
- `vfe3/model/model.py` ‚Äî the fused-chunked gate now covers both decode families:
  `decode_mode='diagonal_chunked'` with targets routes prior-bank to the diagonal fused CE
  and the linear ablation to the new linear fused CE (logits `None` on both, as before).
- `vfe3/config.py` ‚Äî dead-knob disclosure: `phi_precond_mode != 'none'` with `e_phi_lr == 0`
  and `m_phi_natural_grad=False` warns that no preconditioner code runs (its only consumers
  are the E-step phi substep and the natural-gradient M-step); resolves the user-observed
  pullback-vs-killing "difference" as confounded. Stale comment fixed: the linear decode now
  reads `decode_mode` for exactly the chunked-CE selection.
- Tests ‚Äî `test_perf_equivalence.py`: exp_dim wiring spy, blocked-vs-dense `compose_bch` at
  orders 1‚Äì4 (values and gradients), `_pair_contract` vs the legacy `beta_coord` einsum on all
  three branches. `test_chunked_decode.py`: linear chunked CE vs dense (values at 1e-5, grads
  to mu/W/bias, ignore_index, all-ignore zero, end-to-end forward parity with `logits is
  None`). `test_config.py`: dead-knob warning fires and stays silent when a gate is open.
- `train_vfe3.py` ‚Äî operating-point toggle (not committed with the code changes):
  `decode_mode = "diagonal_chunked"` so the linear-decode run actually takes the fused CE path.
