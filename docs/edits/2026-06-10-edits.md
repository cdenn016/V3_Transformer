## vfe3/geometry/cg.py
- `atol` joined the intertwiner cache key — two calls with different tolerances are different
  solves and no longer alias (CR1). New `clear_cg_cache()` public helper (CR5/PP5/RF6).
- Equivariance verify gate now scales with the null-space cut: `max(1e-7, 10*atol)`; identical
  behavior at the default `atol=1e-8` (PP2).
- `cg_selection` pre-checks the 5000 construction-size guard via the closed-form dims and skips
  an oversize triple with a warning instead of raising mid-enumeration and blocking the whole
  tower (F25).

## vfe3/model/cg_coupling.py
- Structurally dead paths pruned at enumeration: a self-pair (same source copy twice) through a
  swap-antisymmetric intertwiner slot is identically zero, so its weight could never train
  (F10/F13/F20). Note this changes `path_weights` length for affected towers — a CG checkpoint
  written before today would fail a strict load (flagged in the audit's User Decisions).
- Forward batched by (intertwiner slot, target block): one stacked outer product, one weighted
  sum, one intertwiner einsum per group instead of the per-path Python loop (F2/DB2/RF3).
- Grad-free identity short-circuit (`not torch.is_grad_enabled() and self.is_identity()`):
  zero-cost on no-grad/eval paths, and deliberately NOT under autograd, where it would sever the
  zero-init weights from the graph (F1/F14/F16/F30 with the CR2 verifier correction).
- `path_types` documented as introspection-only (RF4); means-only/no-congruence note added (F11).

## vfe3/model/head_mixer.py
- `A` now cast to the input dtype/device in forward — fixes the loud crash on any module/input
  dtype mixture; CGCoupling already did this (DB1).
- Same grad-free identity short-circuit as the coupling (CR3 with the gradient-flow caveat).
- Construction warnings: non-adjacent same-label blocks (mixed per contiguous run only — no
  cross-copy mixing) and all-distinct-label towers (Schur-forced per-block scalar gains)

- `_dense_m` documented as deliberately uncached (differentiable through the deltas; a cache
