## Gauge M-step: `m_gauge_update_rule` toggle (`heavy_ball` | `adam`)

Added an opt-in moment rule for the natural-gradient gauge M-step so the preconditioned
(natural) gradient can be driven by Adam's m/v/bias-correction instead of heavy-ball momentum.
Motivation: ablations (`vfe3_ablation_results/gauge_mstep_optim`) show the heavy-ball nat-grad
path freezes `phi_embed` (`weight_norm_phi` flat at ~190 across 15k steps) and lands on the
ungauged baseline (~272 PPL) while plain AdamW trains the frame (144 PPL). The raw phi gradient
is tiny (~0.06) and heavy-ball has no per-coordinate magnitude normalization; the Killing metric
inverse shrinks it further. `adam` restores `1/sqrt(v)` normalization while keeping the metric
direction. See `docs/analysis/2026-06-24-mstep-phi-natgrad-vfe2-vs-v3.md`.

- `vfe3/config.py`: new field `m_gauge_update_rule: str = "heavy_ball"` (default keeps current
  behavior byte-identical) + `_require(...,("heavy_ball","adam"),...)` validation.
- `vfe3/gauge_optim.py`: `GaugeNaturalGradAdamW.__init__` takes `gauge_update_rule`; `step()`
  branches â€” `adam` runs Adam moments ON `nat = G^{-1} grad` (betas/eps from the AdamW group,
  state keys `gauge_m`/`gauge_v`/`gauge_step`), `heavy_ball` unchanged. `__setstate__` docstring
  generalized (gauge state now also covers the adam keys; still never carries `"step"`).
- `vfe3/train.py`: `build_optimizer` forwards `gauge_update_rule=cfg.m_gauge_update_rule`.
- `tests/test_gauge_optim.py`: `+2` tests â€” (1) `adam` + `precond_mode='none'` reproduces plain
  `torch.optim.AdamW` to atol 1e-10 (the identity-metric equivalence; also why conformal killing
  collapses to AdamW); (2) `adam` moves phi >20x more than `heavy_ball` on a tiny gradient.

Verification: `tests/test_gauge_optim.py` 5/5 pass; end-to-end smoke (build â†’ train_step â†’ phi
moves, Adam state created); validator rejects bad rule; `test_checkpoint_resume.py` /
`test_phi_weight_decay.py` pass. (Repo `.venv` lacks numpy/matplotlib/pytest â€” ran via
`uv run --with pytest --with numpy [--with matplotlib]`; the only test "failures" were that
missing-dependency env gap, which clear when the deps are present.)

Default unchanged (`heavy_ball`); not committed (left for user). To sweep:
`m_phi_natural_grad=True`, `phi_precond_mode='pullback_per_block'`,
`m_gauge_update_rule âˆˆ {'heavy_ball','adam'}` vs the `adamw` arm (`m_phi_natural_grad=False`).

## Console banner: actual (live) param count via grad probe

receive NO gradient â€” `mu_embed`/`sigma_log_embed` (rerouted to the s-tables by
`prior_source='model_channel'`) and `decode_log_scale` (the linear decode discards `tau_eff`). A
`requires_grad` filter does NOT catch them (they stay `requires_grad=True`). Added a config-agnostic

  untouched, verified), `.grad` cleared afterward. Best-effort `try/except` â†’ falls back to
  `live=trainable` if the probe can't run.

  `15,153,002 params (13,142,721 live, 2,010,281 dead)` plus a

  when nothing is dead).

Verification (real code via `uv run --with numpy`): banner renders the live/dead split exactly;

## `scaling.py` user-friendliness + two new GL-block scaling routes

- Self-contained baseline (user request): removed `from train_vfe3 import config as BASELINE`; the
  runner now binds `BASELINE = config` (the local ~250-line `config` dict in `scaling.py`, previously
  dead/unused). Editing that dict now actually sets the scaling operating point. Docstrings/comments
  that referenced the train_vfe3 import updated (top docstring, the `max_steps` knob, `_baseline_value`).
- New route `route_grow_k_fixed_block(ks, block)` â€” grow K at a FIXED gauge block GL(`block`), so
  `n_heads = K/block` scales with K (each head is one GL(block) frame). `block_glk` n_gen = block*K

- New route `route_vary_block_fixed_k(embed_dim, blocks)` â€” fixed K, vary the block GL(`b`)
  (`n_heads=K/b`); the `route_blocksize` idea expressed in GL(b) terms, with GL-labels, `kl_max=8*K`,
  and single-block mixer-off handling.

  `CONFIG["routes"]` per request. Added a plain-language ROUTE MENU comment above `ROUTES` documenting
