# 2026-06-24 edits

## Gauge M-step: `m_gauge_update_rule` toggle (`heavy_ball` | `adam`)

Added an opt-in moment rule for the natural-gradient gauge M-step so the preconditioned
(natural) gradient can be driven by Adam's m/v/bias-correction instead of heavy-ball momentum.
Motivation: ablations (`vfe3_ablation_results/gauge_mstep_optim`) show the heavy-ball nat-grad
path freezes `phi_embed` (`weight_norm_phi` flat at ~190 across 15k steps) and lands on the
ungauged baseline (~272 PPL) while plain AdamW trains the frame (144 PPL). The raw phi gradient
is tiny (~0.06) and heavy-ball has no per-coordinate magnitude normalization; the Killing metric
inverse shrinks it further. `adam` restores `1/sqrt(v)` normalization while keeping the metric
direction. See `docs/analysis/2026-06-24-mstep-phi-natgrad-vfe2-vs-v3.md`.

- `vfe3/config.py`: new field `m_gauge_update_rule: str = "heavy_ball"` (default keeps current
  behavior byte-identical) + `_require(...,("heavy_ball","adam"),...)` validation.
- `vfe3/gauge_optim.py`: `GaugeNaturalGradAdamW.__init__` takes `gauge_update_rule`; `step()`
  branches — `adam` runs Adam moments ON `nat = G^{-1} grad` (betas/eps from the AdamW group,
  state keys `gauge_m`/`gauge_v`/`gauge_step`), `heavy_ball` unchanged. `__setstate__` docstring
  generalized (gauge state now also covers the adam keys; still never carries `"step"`).
- `vfe3/train.py`: `build_optimizer` forwards `gauge_update_rule=cfg.m_gauge_update_rule`.
- `tests/test_gauge_optim.py`: `+2` tests — (1) `adam` + `precond_mode='none'` reproduces plain
  `torch.optim.AdamW` to atol 1e-10 (the identity-metric equivalence; also why conformal killing
  collapses to AdamW); (2) `adam` moves phi >20x more than `heavy_ball` on a tiny gradient.

Verification: `tests/test_gauge_optim.py` 5/5 pass; end-to-end smoke (build → train_step → phi
moves, Adam state created); validator rejects bad rule; `test_checkpoint_resume.py` /
`test_phi_weight_decay.py` pass. (Repo `.venv` lacks numpy/matplotlib/pytest — ran via
`uv run --with pytest --with numpy [--with matplotlib]`; the only test "failures" were that
missing-dependency env gap, which clear when the deps are present.)

Default unchanged (`heavy_ball`); not committed (left for user). To sweep:
`m_phi_natural_grad=True`, `phi_precond_mode='pullback_per_block'`,
`m_gauge_update_rule ∈ {'heavy_ball','adam'}` vs the `adamw` arm (`m_phi_natural_grad=False`).
