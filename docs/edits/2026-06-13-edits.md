## 4. f-divergence registry build-out

`squared_hellinger` already existed (registered, with an efficient Gaussian closed form via
`H┬▓ = 1 ŌłÆ exp(ŌłÆD_┬Į/2)` reusing the pinned `renyi` at ╬▒=0.5). Added two more f-divergences that
likewise reuse the `renyi` Gaussian closed form (╬▒Ōēż1, numerically safe) and become config-selectable
via `divergence_family` automatically (validated against the functional registry; routed through
`get_functional` in `free_energy`):

- **bhattacharyya** (`vfe3/families/base.py`): `D_B(qŌĆ¢p) = D_┬Į(qŌĆ¢p)/2 = renyi(q,p,╬▒=0.5)/2`.
  Symmetric, zero iff `q==p`.
- **jeffreys** (`vfe3/families/base.py`): symmetrized KL `J(qŌĆ¢p) = KL(qŌĆ¢p) + KL(pŌĆ¢q) =
  renyi(q,p,1) + renyi(p,q,1)`. Symmetric, zero iff `q==p`.

Both re-exported from `vfe3/divergence.py`. Tests added to `tests/test_divergence.py` (definitional
identities, symmetry, self-is-zero and non-negativity over both Gaussian families, and an
end-to-end model forward+backward under each, routing through the autograd oracle like
`squared_hellinger`). (Chi-squared / R├®nyi-2 deferred: the ╬▒>1 blend hits the documented
`torch.where`+NaN gradient edge ŌĆö a separate numerically-careful addition.)

## 7. Ultra-deep multi-expert audit fixes

Branch `vfe3-ultradeep-fixes-2026-06-13` (off the learnable-r tip, so the audit's line numbers match
the tree). A 7-expert workflow swept 35 modules; every critical/high was adversarially challenged
(skeptic+defender+adjudicator). Result: **0 critical, 0 high, 4 medium, 21 low**, no missing/broken
pure path. Report: `docs/audits/audit-2026-06-13-ultradeep.md`. Tests:
`tests/test_ultradeep_fixes_2026_06_13.py` (TDD red->green per finding).

The 4 mediums:
- **M1** (`config.py`): `s_e_step=True` now rejects a full-covariance family at construction (the s/r
  channel is diagonal by construction; previously accepted then crashed deep in a kernel).
- **M2** (`families/gaussian.py`): the Renyi logdet term (three near-equal logs over a tiny
  `(alpha-1)`) is evaluated in a **float64 island** for `|alpha-1| < 1e-2` in all three kernels
  (diagonal closed/per_coord, full closed). Out-of-band fp32 path byte-identical (golden-pinned).
- **M3** (`geometry/transport.py`): the regime_ii soft cap now bounds the **embedded matrix**
  Frobenius norm `||delta.G||_F` (squared-norm form, grad-safe at zero), not the coordinate norm --
  the edge exp is the EXACT operator for non-orthonormal towers (so_n/sp_n), not the clamped
  surrogate. Byte-identical for orthonormal `block_glk` (Gram=I, the default).
- **M4** (`geometry/transport.py`): the full-covariance congruence sandwich `Omega Sigma Omega^T`
  (squares cond) is evaluated in a **float64 island**; the diagonal hot path is untouched. Extreme
  cond remains fp32-storage-limited (documented).

Lows landed: **L7** renyi warning gated to the blend branch (`families/base.py`); **L10**
`log_partition_at` routes through `safe_cholesky` (NaN, not raise, on non-PD); **L11**
`retract_spd_full`/`retract_logeuclidean` use a spectrum-relative `gap_eps` (`_rel_gap_eps`) so the
eigh-adjoint is accurate for meaningful gaps near the variance floor; **L13** `numerics.condition_number`
handles a diagonal `(...,K)` spectrum (the default family) instead of raising; **L15** the
`effective_rank` metric routes a full covariance through `_spectrum`; **L16** the `free_energy_terms`
metric wrapper requires `tau` (no silent `tau=1.0`); **L20** the CG antisymmetry prune is tied to the
solver `atol` (byte-identical for shipped towers). Documentation-honesty fixes (comments must not
lie): **L1** Mahalanobis diagonal-subgroup invariance, **L2** dead pos_phi slk-gate comment, **L3**
weight-decay-on-None, **L12** Killing-lift spectral assumption, **L17/L18** s_e_step EM gating +
mean/sum reduction convention, **L21** rope odd-block inert channel, **L5** closure-scan size-skip
limitation (+ the `close_basis=True` verified path), **L6** `check_admissible` is a test tool, not a
build guard.

Not landed (with rationale): **L4** the rope means-only incoherence is already surfaced by an existing
config warning (escalating to an error keyed on an internal consumer would over-restrict). **L9** the
audit wanted `FullGaussian.entropy()` to return NaN on a non-PD Sigma, but that conflicts with the
