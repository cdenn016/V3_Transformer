## 4. f-divergence registry build-out

`squared_hellinger` already existed (registered, with an efficient Gaussian closed form via
`H² = 1 − exp(−D_½/2)` reusing the pinned `renyi` at α=0.5). Added two more f-divergences that
likewise reuse the `renyi` Gaussian closed form (α≤1, numerically safe) and become config-selectable
via `divergence_family` automatically (validated against the functional registry; routed through
`get_functional` in `free_energy`):

- **bhattacharyya** (`vfe3/families/base.py`): `D_B(q‖p) = D_½(q‖p)/2 = renyi(q,p,α=0.5)/2`.
  Symmetric, zero iff `q==p`.
- **jeffreys** (`vfe3/families/base.py`): symmetrized KL `J(q‖p) = KL(q‖p) + KL(p‖q) =
  renyi(q,p,1) + renyi(p,q,1)`. Symmetric, zero iff `q==p`.

Both re-exported from `vfe3/divergence.py`. Tests added to `tests/test_divergence.py` (definitional
identities, symmetry, self-is-zero and non-negativity over both Gaussian families, and an
end-to-end model forward+backward under each, routing through the autograd oracle like
`squared_hellinger`). (Chi-squared / Rényi-2 deferred: the α>1 blend hits the documented
`torch.where`+NaN gradient edge — a separate numerically-careful addition.)

## 5. s_e_step=True data/figures

The model-channel refinement was invisible in the figure pipeline (`diagnostics()` folds the refined
s into the belief anchor and gates `hyper_prior`/`gamma_coupling` on `not s_e_step`). Added a
faithful s-channel diagnostic that replays the same two steps `forward`/`diagnostics` take:

- `vfe3/viz/extract.py::s_channel_refinement(model, tokens)`: returns `None` when `s_e_step=False`;
  else replays `encode_s` (static s0) → `_refine_s` (refined s1) under the frozen gauge frame and
  returns per-position `mu_delta`, `logsigma_delta`, and `KL(s0‖r)`/`KL(s1‖r)` to the frozen
  hyper-prior centroid `r` (the consensus the s E-step descends toward).
- `vfe3/viz/figures.py::plot_s_channel_refinement`: Panel A = per-position KL to `r` before/after
  (a healthy channel pulls `KL(s1‖r) ≤ KL(s0‖r)`); Panel B = per-position refinement magnitude.
- Wired into `vfe3/viz/report.py::generate_figures` as `s_channel_refinement.png`, guarded so it is
  emitted only when `s_e_step=True` (the `None` extractor skips it otherwise).
- Tests in `tests/test_report.py`: the extractor is present iff `s_e_step`, and the figure is
  emitted iff `s_e_step`.
