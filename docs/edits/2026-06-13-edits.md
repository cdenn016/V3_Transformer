## 2. 2026-06-13 deep-audit punch-list (the safe, confirmed findings)

- **#7 registry-bypass validation** (`vfe3/config.py`): `phi_precond_mode` now validates against the
  live `phi_preconditioner._PRECOND` registry, `phi_retract_mode`/`pos_phi_compose` against
  `lie_ops._COMPOSE` — restoring the add-by-registering contract.
- **#8 dead constants** (`vfe3/config.py`): removed the unused `_VALID_ENCODE_MODES`,
  `_VALID_DECODE_MODES`, `_VALID_PHI_PRECOND_MODES`, `_VALID_PHI_RETRACT_MODES`,
  `_VALID_POS_PHI_COMPOSE` (encode/decode were already registry-validated; the other three now are).
- **#6 invariant_families** (`vfe3/geometry/groups.py`): `("gaussian", "gaussian_full")` so
  `invariant_for` matches the real registry family key (diagonal correctly stays non-invariant).
- **#18 t5 guard** (`vfe3/config.py`): `t5_num_buckets >= 2` (was `>= 1`); prevents the
  `max_exact = nb//2 = 0` division-by-zero in the bucket function.
- **#12 attention_tau** (`vfe3/free_energy.py`): reject malformed (≥2-d) kappa tensors.
- **#17 metrics tau** (`vfe3/metrics.py`): the diagnostic entropy term uses `_broadcast_tau` so a
  per-head `(H,)` tau aligns to the head axis (so_n/sp_n diagnostics).
- **Closure diagnostic restored, size-gated** (`vfe3/geometry/lie_ops.py`,
  `vfe3/geometry/phi_preconditioner.py`): re-enabled `warn_if_basis_not_closed` in `compose_bch` and
  `_structure_constants` with a `max_elements` gate — small non-closed cross_couplings chains are
  scanned, large direct-sum bases (block_glk K=140, the prior O(K⁴) OOM) are skipped as
  closed-by-construction. Resolves both the OOM concern and the disabled-diagnostic finding.
- **Type contracts**: `_as_coeff` annotated (`vfe3/model/block.py`); `_retract_core`
  `trust_region`/`max_norm` → `Optional[float]` (`vfe3/geometry/lie_ops.py`); `b0`/`c0` →
  `'float | torch.Tensor'` in `free_energy_value`/`e_step_iteration` (`vfe3/inference/e_step.py`).
- **DA-P3a** (`vfe3/train.py`): deprecated `run_training` now moves the model to CUDA when available
  (was deriving CPU from a fresh param).

### Deferred (with rationale — NOT landed)

- **#13/#15** per-forward `(B,N,K,K)` zero-allocs (`lie_ops._from_equal_diag_blocks`,
  `transport._blockwise_matrix_exp`): transient (caching-allocator-reused), marginal vs the
  already-fixed dominant sinks; a value-equivalent fix needs a block-native `compose_bch`/
  `extract_phi` rewrite with golden tests (a dedicated perf pass).
- **#14** `exp(r_sigma_log)` recompute: a `(K,)=140`-element op per forward — caching is
  over-engineering for no measurable gain.
- **#4** `pos_phi_free` hard norm clamp: per-block norm at the operating point is ~1.2 (conditioning
  ~11, benign) and `pos_phi_free` already has `phi_weight_decay=0.05` soft bounding; a hard clamp at
  the operating scale would perturb active runs.
- **#5** NaN-rate monitor on `safe_kl_clamp`: observability the codebase deliberately keeps off the
  hot path (host-sync cost).
- **#11** `e_step` `**kwargs` typo-rejection: a correct allowlist must track the
  `e_step → e_step_iteration → belief_gradients/free_energy_value` chain (e.g. `mass_phi` is not a
  `free_energy_value` param); fragile maintenance burden, low real-world risk (`VFE3Config` already
  rejects unknown fields at construction).
