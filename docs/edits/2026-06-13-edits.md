## 4. f-divergence registry build-out

`squared_hellinger` already existed (registered, with an efficient Gaussian closed form via
`H² = 1 − exp(−D_½/2)` reusing the pinned `renyi` at α=0.5). Added two more f-divergences that
likewise reuse the `renyi` Gaussian closed form (α≤1, numerically safe) and become config-selectable
via `divergence_family` automatically (validated against the functional registry; routed through
`get_functional` in `free_energy`):

- **bhattacharyya** (`vfe3/families/base.py`): `D_B(q‖p) = D_½(q‖p)/2 = renyi(q,p,α=0.5)/2`.
  Symmetric, zero iff `q==p`.
- **jeffreys** (`vfe3/families/base.py`): symmetrized KL `J(q‖p) = KL(q‖p) + KL(p‖q) =
  renyi(q,p,1) + renyi(p,q,1)`. Symmetric, zero iff `q==p`.

Both re-exported from `vfe3/divergence.py`. Tests added to `tests/test_divergence.py` (definitional
identities, symmetry, self-is-zero and non-negativity over both Gaussian families, and an
end-to-end model forward+backward under each, routing through the autograd oracle like
`squared_hellinger`). (Chi-squared / Rényi-2 deferred: the α>1 blend hits the documented
`torch.where`+NaN gradient edge — a separate numerically-careful addition.)

**A1 — per-coordinate Bhattacharyya / Jeffreys.** `state_dependent_per_coord` was locked to renyi.

(`grad = alpha*(D)·dD`, no product-rule residue) for all four divergences. The user's note is the

**A3 — stale NaN docstring.** `oracle_unroll_grad`'s "expect NaNs there" for `gaussian_full` predated

## Per-run observability rollout — adversarial-review follow-ups

A 4-reviewer adversarial workflow over the Tier-1/2/3 diff (verdict: needs-fixes) surfaced issues in
the DIAGNOSTIC VALUES (none crash training -- all metrics are off-graph). Fixed:
- HIGH `guard_saturation` read `sigma_floor_frac`=0 on a FULL covariance whose SPD floor binds,
  because the eigvalsh error (~eps_machine*lam_max) exceeds the tight rtol*eps window. Widened the
  spectrum-boundary tolerance to that noise floor on the full-cov path only (diagonal path unchanged).
- HIGH `group_gauge_invariant` was frame-dependent for sp/sp_n (the singular-value squeeze is
  invariant only under ORTHOGONAL conjugation; Sp is not orthogonal). Switched to the eigenvalue-
  modulus squeeze max log|lambda| - min log|lambda| (conjugation-invariant).
- HIGH dropped `elbo_ce_gap`: it subtracted a CE from d["total"], which is complexity-only (no
  -E_q[log p] data term), so it had no canonical meaning. `free_energy_total` and `val_ce` are both
  already columns.
- MED `generalization_gap` flipped to val_ce - train_ce (standard: positive = overfit); documented
  the seq-0-train vs val-set-mean caveat.
- MED the FD gradient check now probes a DECODE parameter (output_proj_weight, else decode_log_scale)
  whose gradient does not pass through the detached E-step adjoint, so a healthy model reads ~1e-4 and
  a real bug has headroom (mu_embed sat at the detached-oracle's ~10-25% plateau, no headroom).
- MED scoped the `vertex_cond_max`^2 bound comment to the flat path (regime_ii's edge factor
