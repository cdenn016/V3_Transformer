## Head mixer auto-disables on a single head (config ergonomics)

`VFE3Config.__post_init__` (`vfe3/config.py`): `use_head_mixer=True` with a head-block group
(`block_glk`/`tied_block_glk`) and `n_heads < 2` no longer raises -- it now AUTO-DISABLES the mixer
(`self.use_head_mixer = False`) and emits a `UserWarning`. This is a single-HEAD scan artifact (one
gauge block, nothing to mix), so a head/K sweep can leave `use_head_mixer=True` set without a manual
toggle-off at `n_heads=1`. A genuinely single-BLOCK group (`glk`/`so_k`/`sp`) still RAISES -- choosing
a non-block group cannot mix regardless of head count, an independent static misconfiguration.

Tests (`tests/test_audit_fixes_2026_06_10.py`): `test_config_rejects_head_mixer_on_single_block_groups`
narrowed to the `glk` single-block raise; new `test_config_auto_disables_head_mixer_on_single_head`
pins the `block_glk`/`tied_block_glk` `n_heads=1` auto-disable + warning. Full suite: **1049 passed,
1 skipped, 1 xpassed, 0 failures/errors** (junit `tests=1051`).
