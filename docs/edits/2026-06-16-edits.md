# V3_Transformer edits — 2026-06-16

## Head mixer auto-disables on a single head (config ergonomics)

`VFE3Config.__post_init__` (`vfe3/config.py`): `use_head_mixer=True` with a head-block group
(`block_glk`/`tied_block_glk`) and `n_heads < 2` no longer raises -- it now AUTO-DISABLES the mixer
(`self.use_head_mixer = False`) and emits a `UserWarning`. This is a single-HEAD scan artifact (one
gauge block, nothing to mix), so a head/K sweep can leave `use_head_mixer=True` set without a manual
toggle-off at `n_heads=1`. A genuinely single-BLOCK group (`glk`/`so_k`/`sp`) still RAISES -- choosing
a non-block group cannot mix regardless of head count, an independent static misconfiguration.

Tests (`tests/test_audit_fixes_2026_06_10.py`): `test_config_rejects_head_mixer_on_single_block_groups`
narrowed to the `glk` single-block raise; new `test_config_auto_disables_head_mixer_on_single_head`
pins the `block_glk`/`tied_block_glk` `n_heads=1` auto-disable + warning. Full suite: **1049 passed,
1 skipped, 1 xpassed, 0 failures/errors** (junit `tests=1051`).

## metrics.csv: validation columns are eval-cadence (blank between evals)

`metrics.csv` was carrying the most-recent validation FORWARD into every `log_interval` row, so
`val_ce/val_ppl/val_bpc`, `generalization_gap`, and the held-out probes repeated on every log line
between evals. Now they are written only on a step where the periodic eval actually ran; the
log-interval rows in between leave those cells BLANK -- matching VFE_2.0's `metrics.csv` (one
validation per `eval_interval`, empty otherwise). Train-side columns still log every `log_interval`.

`vfe3/train.py` (`train` do_csv block): `val_ce/ppl/bpc`, `generalization_gap`, and the
`_VAL_DIAG_KEYS` probes now write their value iff `do_eval`, else `float("nan")` (instead of
`last_val.get(...)` carry-forward). `vfe3/run_artifacts.py` (`log_metrics`): NaN cells render as an
empty string in the CSV file, while `self.history` keeps the raw NaN float so the figure pass
(`math.isfinite` filtering) is unchanged.

Test (`tests/test_report.py::test_metrics_csv_logs_at_log_cadence`): updated from the carry-forward
assertions to assert blank cells on non-eval rows and fresh values on eval rows. Affected suites
(report, run_artifacts, run_diagnostics, model_channel_diagnostics): **53 passed, 0 failures/errors**
(junit `tests=53`).
