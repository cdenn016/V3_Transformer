## Isotypic label-grouped HeadMixer (Task 2)

  parameter. Without labels, behavior is byte-identical to the prior implementation (equal-dims
  required, single-component). With labels, the constructor builds one `A_t = I + Delta_t` per
  maximal run of equal-labeled blocks via a `nn.ParameterList` (`self.mixer_deltas`). The

  Back-compat `mixer_delta` property retained for single-component callers. The error message for
  the unlabeled unequal-dims case now matches the test's `"equal-size blocks"` regex. Module
  docstring updated to cover the label-grouped form. Import line extended to include `Optional`.

## Numerical Clebsch-Gordan intertwiner solver (Task 5)

- `vfe3/geometry/cg.py` (new): numerical CG solver. `cg_intertwiners(N, *, algebra, label_a,
  label_b, label_c)` builds the Gram matrix of the stacked Sylvester operator
  `kron(I_dc, rho_ab^T) - kron(rho_c, I_D)` over every algebra basis element, finds its null
  space via `eigh`, and returns all independent intertwiners as an `(n_mult, d_c, d_a*d_b)`
  float64 tensor (empty leading axis when multiplicity is zero). Build-time equivariance
  residual check (raise on > 1e-7). Results are cached in `_CG_CACHE`. Cost guard raises
  `ValueError("construction size")` for `d_c * d_a * d_b > 5000`. `cg_selection(N, *,
  algebra, labels)` enumerates all admissible triples with canonical `a <= b` ordering.
- `tests/test_cg.py`: four new tests — SO(3) selection rules (l1 x l1 = l0+l1+l2 with
  multiplicities 1; l1 x l2 contains no l0); group-element equivariance check to 1e-10;
  `cg_selection` admissibility and canonical ordering; cost guard ValueError.
- Full 5-test suite: 5 passed in 0.07s. Broader regression (test_cg + test_son_irreps +
  test_head_mixer_isotypic): 36 passed, 0 failed.

## CGCoupling between-block module (Task 6)

- `vfe3/model/cg_coupling.py` (new): `CGCoupling(nn.Module)` — bilinear CG coupling over the
  blocks of a labeled irrep tower. At construction, `cg_selection` enumerates all admissible
  triples and `cg_intertwiners` is called per triple; buffers registered as `cg_<t>` in float64
  to preserve equivariance precision when the module is cast to double. One learned scalar
  `path_weights[p]` per (source-pair, target-copy, multiplicity-slot) path, zero-initialized
  so step-0 is byte-identical to the coupling-off path. `forward` computes
  `delta[..., sc:sc+dc] += w_p * einsum("cd,...d->...c", C[m], xy)` where
  `xy = (x.unsqueeze(-1) * y.unsqueeze(-2)).reshape(..., da*db)` (row-major kron order
  matching `cg_intertwiners`). Sigma passes through untouched (means-only phase). Raises at
  construction when no admissible paths exist or `irrep_labels` is None. Deviation from plan:
  buffers stored as float64 (plan said float32) — required to pass the 1e-12 equivariance
  tolerance; float32 truncation introduced ~1e-8 residual. The `.to(dtype=mu.dtype)` cast in
  forward handles float32 training paths without precision loss on the stored intertwiner.
- `tests/test_cg.py`: three new tests appended — zero-init exact passthrough (torch.equal);
  equivariance to 1e-12 with random nonzero weights and a random group element; selective
  path activation (l1 x l1 -> l2 only: l2 block updates, l0+l1 blocks unchanged).
- Full suite: 9 passed (test_cg.py). Broader regression (test_cg + test_son_irreps +
  test_head_mixer_isotypic): 40 passed, 0 failed.
