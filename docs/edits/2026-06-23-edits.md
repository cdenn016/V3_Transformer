Fixed a full-covariance crash in precision-weighted attention (branch `fix/fullcov-precision-attention`, off `origin/main`). Running the `gauge_equivariance` sweep (also `cg_coupling`; any `family='gaussian_full'` arm with the baseline `precision_weighted_attention=True`) crashed in `VFEModel._fold_precision_bias` with `The size of tensor a (N) must match the size of tensor b (K) at non-singleton dimension 4`. Root cause: the per-key reliability `-log(b0 + tr Sigma_j)` reduced the covariance via `sigma.sum(-1)`, correct only for the diagonal family `(.., N, K)`; under `gaussian_full` the belief covariance is `(.., N, K, K)`, so the sum collapsed a covariance row and left a spurious `K` axis that broadcast-crashed the `log_prior` fold. Fix: in `_fold_precision_bias`, when `not cfg.diagonal_covariance`, reduce `sigma` to its per-coordinate variances `sigma.diagonal(dim1=-2, dim2=-1)` before the trace, so `tr Sigma_j` is the matrix trace.
