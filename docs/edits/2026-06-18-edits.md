# 2026-06-18 edits

## Vocabulary next-token probability figures (new)

Added four vocabulary-probability figures plus their extractors, single-run wiring, a cross-run
comparison driver, and tests. All additive; no existing path changed. Pure no-NN path untouched.

- `vfe3/viz/extract.py`: `vocab_prediction_stats()` (one inference pass -> Seq x top-k display
  sub-grid + per-token calibration accumulators + a confusion sample; entropies in nats) and
  `decode_readout()` (the linear-decode `output_proj_weight` matrix; returns `None` on the
  `use_prior_bank=True` KL-decode path where no `W` exists). Helper `_vocab_display_panel()`.
- `vfe3/viz/figures.py`: four registered plotters, each taking a LIST of arms (1 arm = single run,
  2 arms = side-by-side): `plot_vocab_probability_heatmap` (rows = top predicted tokens, green box =
  true next token), `plot_vocab_calibration` (mean pred prob vs empirical unigram, log-log, with the
  `y=x` marginal-predictor line and a context-gain `Huni - Hpred` annotation), `plot_vocab_confusion`
  (row-normalized next-token category confusion via the existing `_token_category_labels`),
  `plot_decode_readout` (the `W` heatmap). Helper `_tok_label()`.
- `vfe3/viz/report.py`: wired the four single-arm figures into `generate_figures` (every run now
  emits them; `vocab_confusion` needs a decoder, `decode_readout` is skipped on the KL-decode path),
  and added `vocab_comparison_figures(run_dirs, out_dir, ...)` for the cross-run case.
- `compare_vocab_figures.py`: click-to-run K70-vs-K120 driver.
- `tests/test_viz.py`: 5 tests (extractor shapes/invariants incl. mean-prob sums to 1; readout
  present only off prior-bank; single + two-arm plotter saves; confusion-needs-decoder; registry).

### Auto-run by train_vfe3.py (confirmed; no new wiring needed)

`train_vfe3.py:488` -> `finalize_run` -> `run_artifacts.py:671` calls `generate_figures(...)` when
`cfg.generate_figures=True` (default), so the four single-run vocab figures are produced at the end
of every training run into `<run_dir>/figures/` (on the test split). Default `use_prior_bank=False`
means `decode_readout` is present by default; `vocab_confusion` appears when the dataset has a real
tokenizer (wikitext-103 -> gpt2). Changes here:
- `vfe3/viz/report.py`: corrected the stale module docstring that claimed `generate_figures` is
  "never invoked by train/finalize_run" (it IS, via `finalize_run`).
- `tests/test_report.py`: added the three decoder-free vocab figures to the `robust` set in
  `test_generate_figures_drives_live_model`, and asserted them in `test_finalize_autoruns_figures`
  (pins that the train_vfe3.py autorun emits them).

Verified: `tests/test_viz.py` + `tests/test_report.py` = 50 passed (plus the new pins). Rendered the real K70-vs-K120
comparison from the trained checkpoints over wikitext-103 validation (CPU, batch 8, 48 sequences) to
`vfe3_runs/_vocab_compare/`. Quantitative collapse signal: K70 gain = 1.995 nats (Hpred 4.222),
K120 gain = 1.839 nats (Hpred 4.378), same Huni = 6.217 — K120 predictions are measurably less sharp
and use less context, directionally consistent with its worse ppl but a modest gap, not catastrophic
collapse (supports the compute-optimal-turnover read over a hard prior-collapse).
