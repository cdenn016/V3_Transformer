## Per-layer diagnostics + figures (new)

The stack (`vfe_stack`) returns only the final block, so `diagnostics()`/`converged_state()` and all
95 metrics.csv columns describe layer 2 only. Added a per-DEPTH view, reusing the `attention_maps`
block-replay pattern (no hot-path / no pinned-`diagnostics()` changes):

- `vfe3/model/model.py::VFEModel.diagnostics_per_layer`: replays the block loop one block at a time
  and, at each converged belief, recomputes the same belief-channel quantities `diagnostics()` uses
  (transport, energy, beta, self-divergence vs THAT block's own handoff prior, alpha) and the same
  `metrics.*` reductions. Returns L-length lists for self/belief/attention-entropy/total free energy,
  self_divergence, holonomy (Frobenius + Wilson), gauge trace/invariant spread, effective_rank,
  attn_entropy, belief_cond_median, phi_norm_mean. Model-channel blocks (hyper-prior, gamma) are a
  single coupling, deliberately excluded — `total` is the belief-channel F at that depth.
- `vfe3/viz/figures.py::plot_per_layer_diagnostics` (registered `per_layer_diagnostics`): 2x2 summary
  (free-energy budget / holonomy / gauge action / belief geometry) vs layer index.
- `vfe3/viz/report.py`: drives `model.diagnostics_per_layer`, emits `figures/per_layer_diagnostics.png`,
  and writes `metrics_per_layer.csv` (rows = layer) to the run dir. Both best-effort.

Scope note: per-layer is a SNAPSHOT of the best/final model at figure time (it answers "which layer
carries the mass; does curvature/rank build with depth"), NOT a per-step time series in metrics.csv —
that would need hot-path changes and was intentionally not added.
