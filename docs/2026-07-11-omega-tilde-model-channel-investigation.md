# Investigation: a separate model-channel transport ~Omega_ij for the s-channel

Date: 2026-07-11
Status: investigation memo + design (no code changes; nothing in this memo has been implemented)
Question: apropos of the full theory (PIFB2.tex), should the transformer build out a separate model-fiber transport `~Omega_ij` — an independent frame field `~phi_i` for the gamma / s-channel — and should we expect it to improve learning and performance?

## Verdict in brief

The theory licenses it, the code seams anticipate it, and the sister codebase already implements it — but the expected performance benefit is unproven in either direction. The recommendation is a staged buildout: two zero-training-cost diagnostics on the current checkpoint, then a gamma-content ablation to establish the baseline the whole question depends on, then two cheap theory-sanctioned interpolations (a learned intertwiner `M`, then a low-rank per-token frame offset) before committing the full +10.05M-parameter independent frame table, which must beat a width-matched control (K = 27) to be adopted. The strongest mechanistic argument in favor is not "more capacity" but a redundancy defect specific to the current wiring: under `prior_source=model_channel` the beta and gamma attention distributions are computed from nearly the same Gaussians in the *same* transported geometry, so the `gamma_as_beta_prior` fold mixes two strongly correlated distributions and the model channel contributes less independent relational information than the functional's two-channel structure intends. A separate `~Omega` is the theory-canonical way to decorrelate them — and the mm_exact E-step, the best-performing update path to date, is exactly where that decorrelation has its largest effect, because under mm_exact the beta weights are not a gradient tilt but the convex-combination weights of the prediction itself (Section 5).

## 1. Sources and a provenance note

This investigation is grounded in the current WIP `Research/manuscripts/PIFB2.tex` (3,557 lines, 2026-07-04 build) together with `GL(K)_attention.tex` / `GL(K)_supplementary.tex` (2026-07-09 revision), the `vfe3` codebase at its current working state, the research wiki ([[VFE Transformer Program]], [[VFE Transformer Research Directions (2026-06-21)]]), and the `MAgent_Model` reference implementation. The repo's `Manuscripts-Theory/PIFB.tex` is the older mirror; on every passage relevant here (independent-frame configuration, envelope gradients, cross-scale shadows, model-channel functional) PIFB2 preserves the same content at new line numbers, so citations below give PIFB2 lines with PIFB.tex equivalents where useful.

One provenance correction to carry forward: PIFB2's three "independent-frame configuration realized by the reference implementation" claims (PIFB2:236, 409, 1584) refer to `MAgent_Model/gauge_agent/`, *not* to the transformer. The GL(K) papers say the opposite of the transformer — "using the shared frame phi" (GL(K)_attention:2147, 2174, Algorithm 1 at 2206; GL(K)_supplementary:897) — and PIFB2 itself severs the two codebases explicitly. The manuscripts are internally consistent once the referent is fixed, but PIFB2:907's shared-frame sentence ("equals Omega_ij as a group element") sits in unresolved tension with 236/409/1584 about which configuration is canonical; Section 8 below lists the manuscript edits an adoption decision would trigger.

## 2. What the theory licenses

PIFB2 defines two associated bundles over the context manifold: the state bundle E_state carrying (q_i, p_i) and the model bundle E_model carrying (s_i, r_i). The canonical functional couples the slow channel by gamma_ij KL(s_i || ~Omega_ij s_j) with its own entropy term and its own softmax stationary point ("An analogous expression holds for the model-channel weights gamma*_ij, with ~Omega_ij, s_j, and pi^(s)_ij in place of Omega_ij, q_j, and pi_ij" — PIFB.tex:1308), and its own temperature: tau specializes per channel as kappa_beta = kappa sqrt(K_q) on the belief fiber and kappa_gamma = kappa sqrt(K_m) on the model fiber (PIFB.tex:1265).

Two configurations of the model-fiber transport are admitted (PIFB2:236, the load-bearing passage):

1. **Tied / shared frame.** ~Omega_ij is the rho_model-representation image of the same group element as Omega_ij. For G = GL(K_q) the manuscript fixes K_m = K_q with the fundamental representation, under which ~Omega_ij = Omega_ij numerically. This is what the transformer implements.
2. **Independent frame.** "The framework admits giving the model fiber its own principal bundle with structure group GL(K_m) and an independent frame field ~phi_i, so that ~Omega_ij = exp(~phi_i) exp(-~phi_j) is a transport in its own right rather than the image of Omega_ij under a representation." Under this configuration "even the structure-group identification is absent, leaving the cross-scale shadows as the only inter-channel coupling" (PIFB2:409).

The envelope-gradient discussion states the optimization consequence directly: "For shared frame variables phi_i, both channels contribute only when the same frame field induces both Omega_ij and ~Omega_ij; with independent frame fields the belief-channel derivative is taken with respect to phi_i and the model-channel derivative with respect to ~phi_i" (PIFB2:775; PIFB.tex:1434, 4007). Tied frames route two channels' gradients into one table; independent frames decouple them. The meta-agent machinery is already written for both cases: under independent model frames the model fiber carries its own barycenter with its own coherence weights w^mod solved by an independent fixed point (PIFB2:1566).

Constraints an independent ~Omega inherits automatically or must satisfy:

- **Cocycle and flatness.** A vertex-local ~Omega_ij = exp(~phi_i) exp(-~phi_j) satisfies ~Omega_ij ~Omega_jk = ~Omega_ik identically with vanishing holonomy, by the same algebraic collapse as the belief transport (PIFB.tex:923, thm:vanishing_holonomy). No new theory is needed; the s-fiber gets a Regime-I flat transport of its own. This same rigidity is also the safeguard against "gauge cheating": N frames cannot zero N^2 pairwise divergences, so the gamma term retains content under an independent frame just as the beta term does.
- **Gauge covariance.** The invariance theorem (thm:glk_invariance) is fiber-agnostic — it holds for any GL(K_m) acting by simultaneous congruence pushforward on both KL arguments — so gamma_ij KL(s_i || ~Omega_ij s_j) is invariant under an independent diagonal action on E_model. What is *not* proved is anything at K_m != K_q: the supplementary restricts the uniqueness/invariance package to matched fibers (GL(K)_supplementary:910). A first buildout should therefore keep K_m = K_q.
- **Diagonal-family projection.** The witness/closure caveat for diagonal Gaussians under general congruence (GL(K)_supplementary:912) applies equally to ~Omega on s_i; the existing projection machinery covers it, and the same "projected, not exact" scope statement must be attached.

The theory's stated purpose for the slow channel is timescale-separated identity: the imposed adiabatic hierarchy eta_q : eta_s : eta_phi ~ 1 : eps : eps^2 (PIFB.tex:1617), consensus on generative models as the content of the gamma term, and the slow pair (s, phi) as the carrier of persistent structure. Both papers concede the transformer route does not realize a slower model timescale ("The recorded update rates do not establish a slower model timescale" — GL(K)_attention:2147). A shared frame makes that failure structural: the s-channel's relational geometry is dragged at whatever rate the belief channel trains phi_embed. An independent ~phi with its own learning rate is the only way the transformer can realize the per-fiber timescale hierarchy the theory imposes.

## 3. What the code does today

The s-channel is fully built *except* the frame. It has its own per-token tables `s_mu_embed`, `s_sigma_log_embed` (prior_bank.py:247-248, each V x K), its own softmax gamma over transported model divergences with its own temperature `kappa_gamma` (model.py:1547, 590-594) and its own attention prior `gamma_attention_prior` (model.py:1541-1543), its own coupling weights `lambda_gamma`, `lambda_h`, and its own E-step `_refine_s` (model.py:631-687, eta_s_mu = 0.85, eta_s_sigma = 0.1). Under the live config (vfe3_runs/20260710-180301, K = 20, block_glk, H = 2) the channel is not a side loss but the backbone of prediction, through three routes: `prior_source='model_channel'` makes the refined s^(1) both the belief seed and the prior and the decode bank (prior_bank.py:448-474), q_i^(0) = p_i = s_i^(1) exactly as eq:model_channel_seed states; `s_e_step` replaces the beliefs with s^(1) before the belief stack (model.py:813-820); and `gamma_as_beta_prior` folds the detached gamma posterior into the belief attention log-prior at weight 0.5 (model.py:828-836, 1867-1913). The dormant tables on this path are the *belief* embeddings, not the model ones.

Every one of those routes transports s with the belief frame. `_refine_s` builds its BeliefState with `phi = phi0` (the shared encode-plus-positional frame) under hardcoded `transport_mode="flat"` (model.py:632, 664-676); `_gamma_energy` calls `build_belief_transport` on the belief phi (model.py:1530-1533); and `share_refine_s_transport=true` (the live setting) passes one prebuilt FactoredTransport to both channels (model.py:786-811) — reuse that is only coherent because the frames are identical. The codebase names the gap itself: "no independent model-fiber frame phi_tilde: Omega_tilde reads the belief frame out.phi, so transporting r against it is a rho_model != rho_state category error" (model.py:1363-1364), and config.py:1948-1956 warns that "the s-fiber has no non-flat transport law." No `rho_model`, `K_m`, `phi_tilde`, or `s_omega` identifier exists in code. The seams for adding one are unusually ready: BeliefState carries reserved `s`/`r` fields (belief.py:29-30), the optimizer already handles `gauge=True` and `omega=True` parameter groups generically (gauge_optim.py:202-346), the omega_direct storage/lookup machinery (prior_bank.py:294-387) is a ready template for a per-token frame table, and the phase-3 s-channel frame-fidelity work (docs/superpowers/plans/2026-07-08-omega-direct-phase3-schannel.md) already threaded a frame argument through every s-channel call site.

The `MAgent_Model` reference implementation confirms the theory's claims about itself: each Agent carries separately trainable `omega` and `omega_model` group elements with `phi` / `phi_model` log views, a `tied_gauge_frames` toggle collapsing them to the shared-frame configuration, `step_in_lie_algebra(fiber='model')` updates, and a separate model-frame barycenter (agents.py:346-430; meta_agents.py:1013-1019). The shipped species configuration ties the model fiber across agents (`gauge_fixed_models=True`, PIFB2:129), so nontrivial model-fiber alignment is reachable but not yet exercised there either — the deferred gamma != 0 untied multi-seed rerun is the sister experiment to what is proposed here. Building ~Omega in the transformer would give the program its second realization of the independent-frame configuration and remove the current asymmetry in which only one of the two codebases matches PIFB2's description.

## 4. The mechanistic case for ~Omega_ij

**Redundancy of gamma with beta under the live wiring.** Under s_e_step the fold threads the refined model state into the gamma computation — `s_belief=s_refined`, and the docstring is explicit that "the folded gamma reads the SAME s1 the caller anchors the belief to" (model.py:1867-1913) — while the belief seed is q^(0) = s^(1) and n_e_steps = 1 means beta is computed exactly once, from that seed. Both channels' pairwise energies are therefore KL divergences between the *same* Gaussians under the *same* transport Omega_ij built from the same phi. The gamma and beta distributions differ only through temperature bookkeeping (kappa_gamma vs kappa_beta, both 1 in the live config), the precision-reliability bias folded into the belief prior but not the gamma prior, and the static-prior mixture itself (both priors are causal_alibi_noself). The fold pi = 0.5 softmax(B) + 0.5 gamma consequently mixes the static prior with a distribution close to the very softmax the belief channel is about to compute — approximately a prior-sharpening, not a second information channel. This is a wiring-level observation, not a conjecture; the residual empirical question (D1 below) is only how much the precision bias and temperature differences decorrelate the two in practice. The functional's two-channel structure intends gamma to carry relational information the belief channel does not; the shared frame plus the shared s^(1) source make that structurally impossible in the current configuration. An independent ~Omega breaks the degeneracy at the transport, the one factor of the energy the two channels need not share; the same docstring's deferred "UNDETACHED variant" would break it at the gradient instead, and the two are complementary rather than competing fixes.

**Gradient cross-talk on two-thirds of the parameters.** phi_embed is V x n_gen = 10.05M parameters, 66% of the model. Under the live config it receives gradients from both channels: through the belief stack, and through the attached `_refine_s` transport (the phase-3 design deliberately keeps omega_s/phi attached there so the frame trains "symmetrically"). The envelope-gradient passage (PIFB2:775) is explicit that only independent frames separate these derivative sectors. Whether the two gradient streams actually conflict is an empirical question with a cheap probe (Section 7, D2); if they do, the shared frame is paying an optimization tax on the largest table in the model, and the theory's prescribed fix is precisely ~phi.

**Type-level vs token-level relational geometry.** The s tables are per-vocabulary-type and refined once, target-blind; q is per-position and context-driven. The linguistic reading in both manuscripts assigns grammar/ontology to the s-channel and in-context interpretation to the q-channel. One shared frame forces paradigmatic (type-type) and syntagmatic (token-token) alignment to share a single relational geometry. Separate frames let the model channel learn, for example, part-of-speech-like alignment structure while the belief frame handles contextual transport — the "shared ontologies" role the theory assigns to gamma (PIFB.tex:1458) without conscripting the belief geometry to carry it.

**Timescale realization.** An independent ~phi table with its own `m_s_phi_lr` (and heavy-ball inertia) is the only mechanism by which the transformer can realize eta_s < eta_q at the frame level, converting the papers' concession about timescales into a configurable property.

## 5. Interaction with the mm_exact path

The live config runs `e_step_update='mm_exact'` with `mm_damping=1.0` and `skip_belief_sigma_update=true`, and this path has produced the best results to date, so the ~Omega question has to be answered against it specifically rather than against the gradient route the earlier sections implicitly assumed. Three facts about the wiring, all verified in source, change the weight of the arguments.

**mm_exact applies to the belief channel only.** block.py:94 threads `e_step_update=cfg.e_step_update, mm_damping=cfg.mm_damping` into the belief stack, but `_refine_s` (model.py:596-687) never passes either knob, so the s-refine always runs the gradient route at `e_s_mu_lr=0.85, e_s_sigma_lr=0.1` (e_step.py:540 default) even when the belief E-step is the closed-form fusion. "mm_exact produces the best results" therefore means: exact anchored fusion on q, damped gradient consensus on s. This asymmetry matters for the test design below — the s-refine's update rule and the s-refine's frame are orthogonal toggles, and threading mm_exact into `_refine_s` is a separate, frame-independent experiment that should not be conflated with any ~Omega arm.

**Under mm_exact, beta weights *are* the prediction, and the fold reweights an exact minimizer.** The fused mean is mu*_i = (a_i mu_p/sigma_p + sum_j w_ij mu_t/sigma_t) / (a_i/sigma_p + sum_j w_ij/sigma_t) at frozen beta, with w_ij = lambda_beta beta_ij and mu_t = Omega_ij mu_j (kernels.py `mm_exact_update`; anchored since the 2026-07-10 prior-anchor fix). Under prior_source=model_channel with n_e_steps=1 this is a precision-weighted convex combination of model-channel states — the anchor mu_p = s^(1)_i and the transported neighbors Omega_ij s^(1)_j — and, with sigma skipped, that one fusion *is* the E-step output that decodes. Two consequences. First, the frame is now the exact value transport of the prediction, not just an attention-logit geometry: frame quality and frame gradient conflict (D2) have first-order effect on mu*. Second, the gamma_as_beta_prior fold acts on beta, and beta under mm_exact sets the convex weights of the output directly rather than tilting a gradient step scaled by a learning rate. The redundancy defect of Section 4 is therefore *more* costly on the mm_exact path — mixing the prior with a near-duplicate of beta wastes exactly the mechanism (weight shaping) that mm_exact amplifies — and a decorrelated gamma from an independent ~Omega would be a correspondingly stronger lever here than on the gradient route. The case for ~Omega is strongest on precisely the path that currently performs best.

**~Omega does not break mm_exact eligibility.** The kernel-route gate (e_step.py:664-679, `uses_kernel_route`) requires the filtering kernel, diagonal Gaussians, Renyi order 1, attention entropy, and flat transport with coupled value gauge. All three proposed s-frame modes preserve this: a vertex-local ~Omega_ij = exp(~phi_i) exp(-~phi_j) is flat by the same algebraic collapse as the belief cocycle, the conjugation M Omega M^-1 is a vertex cocycle in ~U = M U M^-1, and `mm_exact_update` consumes the transport as an argument rather than rebuilding it, so any prebuilt s-frame transport slots into the same fusion math should the s-refine ever be moved onto mm_exact. The belief channel's own mm_exact fusion keeps using the belief Omega untouched under every option, since the s-frame modes alter only the gamma machinery and the s-refine. The 2026-07-10 anchor-fix history also flags the one sensitivity worth a test: mm_exact was discontinuous exactly at the q0 == p initialization state the model always occupies. An s-frame change does not move that state (q0 = p = s^(1) comes from prior_source, not from the frame), and zero-init ~phi / M = I keeps step-0 byte-identity on the mm_exact path, but the E1-E3 test sets should include the anchor-at-init and continuity pins from tests/test_mm_exact_prior_anchor.py rerun under each s-frame mode.

One further observation from the same reading: the anchor fix restored the live mu_p -> mu* -> decode path, which under model_channel means the s tables already receive value-path gradient through the mm fusion anchor. An independent ~phi leaves that gradient topology unchanged (the fold stays detached, the s-refine stays attached), so none of the E1-E3 arms perturbs the mechanism the fix repaired.

## 6. The case against, and the honest performance prior

**Parameter cost at the canonical design.** A full block_glk-shaped ~phi table costs V x n_gen = 10,051,400 parameters at the live config — a 66.7% increase (15.08M to 25.13M), making frame tables about 80% of all parameters. An omega_direct-style stored element table costs the same in compact block form and double that dense.

**The matched-parameter bar.** Any +10M-parameter mechanism must be judged against spending the same parameters on the axis with the best evidence behind it. At matched total parameters the alternative to Option A is width K ≈ 27 (V(5K + K^2/2) = 25.13M at H = 2), and the audited width-sweep fit PPL ≈ 1728 K^-1.049 + 63.96 predicts roughly 138.6 → 118.4, a ~20-PPL improvement, as the bar the independent frame must clear. (The fit is descriptive, from a heterogeneous development sweep at fixed GL(10) blocks; it is indicative, not a law — but the live K = 20 run at ~142.8 sits close to it.)

**No gamma baseline exists.** All 36 retained sweep runs and the live config have the model channel active; there is no gamma = 0 or fold-off ablation anywhere in the record. Until the shared-frame gamma channel's marginal value is measured, a ~Omega result is uninterpretable — if gamma-off costs nothing today, the first-order problem is the channel's wiring, not its frame.

**Anchoring and tuning surface.** Nothing in the current objective references a model-fiber frame. ~phi would train only through CE via the attached `_refine_s` route (the fold is no_grad; the scored gamma/hyper-prior terms are gated off under s_e_step). It shares phi's tolerated global flat direction, has no weight decay on the geometric optimizer path (gauge_optim.py:82-84), and `mass_phi` is 0 in the live run — so an independent ~phi starts life *less* anchored than phi (which at least gets two gradient streams). The token-uniform frozen r anchors s in the computational frame but does not pin ~phi. Practically this argues for zero init, a small learning rate, an optional `mass_s_phi` term, and heavy-ball inertia — and it doubles the frame-optimizer tuning surface the m_phi_lr sweep is currently mapping (0.005/0.010/0.0125/0.016; sweep_results.csv).

**Opportunity cost.** The wiki's priority tier still ranks the multi-seed variance floor and the gauge on/off/frozen causal test above any new mechanism; the sigma-collapse finding (2026-06-29) shows the covariance pathway is inert for reasons no new frame fixes. A ~Omega buildout competes with experiments that gate the program's central causal claim.

On net: the sign of the performance effect is unknown. The redundancy and cross-talk mechanisms are real and specific enough to justify cheap probes; the parameter cost, the width-matched bar, and the missing gamma baseline justify refusing to start with the full table.

## 7. Staged plan with decision rules

Every arm below runs on the live mm_exact configuration (e_step_update='mm_exact', mm_damping=1.0, skip_belief_sigma_update=true), since that is the path any adoption decision applies to; the s-refine's update rule is held at its current gradient route across all arms, with the mm_exact-for-`_refine_s` question (Section 5) parked as a separate experiment.

**D1 (diagnostic, zero training).** On the current checkpoint, log per-row overlap between the gamma distribution and (a) the beta distribution and (b) the static causal-ALiBi prior — mean JS divergence or total-variation, per head. The wiring analysis of Section 4 predicts JS(gamma, beta) near zero; the measurement quantifies how much the precision bias and temperature differences decorrelate them in practice. If gamma turns out already far from beta, the shared frame is not the binding constraint and the case for ~Omega weakens sharply; if it duplicates beta, the fold is confirmed as prior-sharpening and the headroom is real.

**D2 (diagnostic, zero training).** Instrument phi_embed with two backward hooks separating the `_refine_s`-routed gradient from the belief-stack-routed gradient; log the cosine between the two streams over ~100 steps. Systematically negative or near-orthogonal cosines confirm the cross-talk tax; consistently positive cosines mean frame-sharing is synergistic and Option A would likely *hurt*.

**E0 (baseline ablation, 1 seed then 3).** gamma-content arms at the live config: (i) `lambda_gamma=0` (s-refine toward r only), (ii) `gamma_as_beta_prior=false` (gamma alive in the s-refine, fold off), (iii) live. This is the missing denominator for everything else. Runs are cheap at K = 20 (~20 min to 15k steps on the 5090 per the current logs).

**E1 (Option C: learned intertwiner, ~200 params).** ~Omega_ij = M Omega_ij M^-1 with one learned M per irrep block, init M = I (byte-identical at step 0). This is the *shared-frame* configuration with a nontrivial learned rho_model — the mildest theory-sanctioned way to give the model fiber a different measurement basis. Cocycle and covariance survive by conjugation (the model fiber transforms under the conjugated representation); the diagonal family needs the same projection already used for general congruence. If D1 shows redundancy and E1 moves PPL at essentially zero parameter cost, the "different geometry helps" hypothesis is confirmed cleanly, uncontaminated by parameter count.

**E2 (Option B: low-rank frame offset, +0.40M at r = 8).** ~phi_i = phi_i + sum_a c_i^a G_a with a small per-block generator basis G_a (learned or fixed) and a per-token coefficient table c in V x r, zero-init (byte-identical at step 0). A true partial-independent frame: vertex-local, flat, covariant, interpolating tied → independent as r grows. Gives the dose-response curve.

**E3 (Option A: full independent ~phi, +10.05M) — only on monotone evidence from E1/E2.** Full V x n_gen table, zero-init, own optimizer group and `m_s_phi_lr`, `share_refine_s_transport` forcibly disabled. Control arms: (i) width-matched K = 27 baseline without ~phi, (ii) the E0 winner. Decision rule per program convention: adopt only with non-overlapping ±1 SD bands over ≥3 seeds against *both* controls. Marginal compute is small — one extra factored exp build per forward (~N H d^3) plus loss of the shared-transport amortization; the N^2 energy grids are unchanged — parameter memory plus Adam moments is roughly +120MB, comfortable on the 5090.

Option E (omega_direct-style stored `s_omega_embed`) is deliberately deferred: it matches Option A's cost without the zero-init byte-identity story, and the phase-3 gate history shows the filtered-diagnostics seam (e_step.py:384-395) would need the same treatment again on a second stored-element table.

## 8. Implementation sketch (for the E1-E3 arms)

A single config knob `s_frame_mode` in {"tied" (default), "conjugation", "offset_lowrank", "independent"}, with `s_frame_rank` for Option B and `m_s_phi_lr` / `mass_s_phi` for Option A. Validation: raise on `s_frame_mode != "tied"` combined with `share_refine_s_transport=true` (the reuse premise dies with the shared frame), raise on combination with `omega_direct` until the stored-element variant is threaded (phase-3 precedent), and keep the flat-only warning pointing at the new knob. Storage lives in PriorBank next to its siblings: `s_frame_M` (per-block K x K, init I) for conjugation; `s_phi_coeff` (V x r, zero-init) plus `s_frame_basis` for offset_lowrank; `s_phi_embed` (V x n_gen, zero-init) for independent, registered in the existing `gauge=True` optimizer group. The frame swap happens at exactly the three seams the phase-3 work already made frame-aware: `_refine_s` (BeliefState phi slot), `_gamma_energy` (its build_belief_transport call), and the shared-transport block in forward_beliefs (which under any non-tied mode simply builds two transports). encode_s grows an optional third return, mirroring `_omega_lookup`. Tests, following the phase-3 template: byte-identity of the default path and of zero-init non-tied modes at step 0; a differential frame-use test per mode (hold phi fixed, perturb the s-frame object, assert gamma energies and refined s move — and the converse); gauge invariance with independent co-transformation of the s tables and s-frame under the model-fiber action; cocycle property of the composed transport for conjugation and offset modes. All tests at K < 6 per the CPU rule.

## 9. Manuscript obligations on adoption

If any non-tied mode ships and is retained in runs, the GL(K) papers' shared-frame sentences (attention 2147/2174/2206, supplementary 897) need the corresponding qualifier, and the transformer would then instantiate the independent-frame configuration PIFB2 currently attributes only to MAgent_Model — resolving the referent split of Section 1 in the cleaner direction. PIFB2:907's single-symbol convention paragraph should gain the same tied-vs-independent qualifier it already carries elsewhere (236, 409, 1584), and the kappa_gamma = kappa sqrt(K_m) distinction becomes live rather than degenerate. The matched-fiber restriction (K_m = K_q) should be stated as a scope condition of the implementation since the uniqueness package is only proved there. The `barycenter_r_` closed form's documented "drops the model-fiber transport" shortcut (prior_bank.py:437-439) becomes incorrect under a real ~Omega and needs revisiting if r ever moves off the token-uniform special case.

## 10. Recommendation

Build toward it, but do not start with it. Run D1/D2 this week on the existing checkpoint (hours, no training); run E0 to establish the gamma baseline the record is missing; then E1, which answers the actual question — does a different model-fiber geometry help at all? — for 200 parameters. Commit to the full ~phi table only if E1/E2 trend positive and E3 clears the width-matched control under the 3-seed rule. If D1 shows gamma already decorrelated from beta, or D2 shows synergistic gradients, write the negative result to the wiki and spend the 10M parameters on width, which the current evidence says is worth ~20 PPL at this scale.
