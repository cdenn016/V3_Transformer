## Controlled semantic-probe evaluation

Added the design specification for preregistered English semantic probes in the controlled belief-geometry reporting path. The design keeps UMAP display-only, evaluates semantic structure in native channel charts and controlled HDBSCAN labels, records exact GPT-2 token resolution, and preserves explicit unavailable records for non-English or decoder-free datasets.

Added the task-by-task TDD implementation plan at `docs/superpowers/plans/2026-07-15-semantic-probe-evaluation.md`. The plan pins manifest validation, exact decoder resolution, native-space pair and field metrics, noise-aware HDBSCAN co-membership, controlled sidecar integration, cross-run comparison fields, and machine-readable verification before the repository lifecycle is completed.

Baseline verification before implementation used `python -m pytest --junitxml=C:\tmp\vfe3-semantic-probes-baseline-20260715.xml`. The JUnit suite recorded 2,857 tests, 0 failures, 0 errors, and 31 skipped tests in 142.433 seconds.

Implemented `vfe3/viz/semantic_probes.py` with the versioned `english_gpt2_semantic_probes` manifest. The manifest contains eight semantic fields, exact single-token GPT-2 forms, close pairs such as `king` to `queen` and `father` to `mother`, cross-field controls, and descriptive pairs such as `king` to `father`. Inline review replaced `king` to `blue` as a control with `king` to `jump` because royalty has genuine associations with blue. Validation rejects duplicate concepts, duplicate field membership, unknown or identical pair endpoints, overlapping accepted forms, and invalid expectation classes. Decoder resolution records accepted forms, token IDs, decoded strings, occurrence indices, occurrence counts, and explicit reasons for missing concepts.

The evaluator averages contextual occurrences into one equal-weight centroid per resolved concept in the supplied native feature chart. It reports Euclidean centroid distance, empirical distance percentile, directional partner rank, mean reciprocal rank, hit at five, semantic-field within and between distances, field silhouette, and semantic-field adjusted mutual information against non-noise HDBSCAN labels. HDBSCAN co-membership uses per-concept cluster probabilities with every occurrence in the denominator and excludes the noise label from the shared-label sum, so shared noise cannot create positive evidence. Descriptive pairs are reported but excluded from close-versus-control aggregates.

Controlled belief and model-channel sidecars now contain a stable `semantic_probes` section. English runs with a decoder evaluate the manifest; non-English runs and decoder-free calls serialize explicit unavailable records. The evaluation consumes the same native features and controlled HDBSCAN labels already produced by the report and never consumes UMAP coordinates. The controlled sidecar schema is now version 2.

Cross-run validation now treats semantic manifest name and version as comparison-contract fields. Comparison JSON adds semantic-field silhouette, semantic-field adjusted mutual information, close-pair distance percentile, mean reciprocal rank, hit-at-five rate, HDBSCAN co-membership, and control-to-close distance ratio. The metric-only comparison figure expands from four to six panels and does not overlay independently fitted UMAP coordinates.
