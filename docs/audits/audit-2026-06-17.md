Whole-repo deep multi-agent audit of V3_Transformer (VFE_3.0). Run via the `deep-audit` skill
orchestrated as a background `Workflow`: 12 investigators (5 base + the full 7-member `audit-*`

## Headline
The two substantive correctness defects both live in the freshly-committed precision feature
(`c1a69eb feat(decode+attn): precision-scaled decode and precision-weighted attention toggles`):

1. **`precision_weighted_attention=True` crashes the forward on every single-block gauge group**
   (`glk`/`so_k`/`sp`, and `block_glk` collapsed to `n_heads=1`). Four independent investigators
   (debugger, transformer-ml ×2, impl-engineer) converged on it. Default-OFF, so the pure path is
   safe — but `ablation.py:230` turns it ON in the baseline and `ablation.py:367-374` sweeps those
   exact single-block groups, so an ablation run crashes those cells. **CONFIRMED critical, UPHELD.**
2. **`decode_precision_scaled=True` is silently dropped on the fused/chunked training decode path**
   while honored at inference, producing a train/eval mismatch. **CONFIRMED medium.**

A clean positive: the gauge-theorist, info-geometer, and variational experts each returned **zero**
findings. The equivariance/transport/holonomy code, the KL/Fisher/f-divergence code, and the
free-energy / E–M-separation / attention-entropy assembly each survived an expert sweep with no
defects surfaced. (Absence of findings is not proof of correctness, but three independent expert
lenses on the theory core came back empty.) No confirmed finding breaks a theoretically pure path;
the impl-engineer confirmed the documented pure paths remain reachable end-to-end.

## Surviving Punch List (post-challenge, ranked)

### Critical
1. **[critical]** `precision_weighted_attention` crashes the forward on single-block gauge groups —
   `vfe3/model/model.py:692,695` (4-D `(B,*,1,N)` bias) vs headless `(B,N,N)` energy at
   `vfe3/free_energy.py:151-152`. Reachable via `ablation.py:230` (baseline True) +
   `ablation.py:372-374` (glk/so_k/sp sweep). *Fix:* emit `(B,1,N)` for a headless single-block
   energy and the 4-D shape only when `len(group.irrep_dims) > 1`; or reject the toggle on
   single-block groups in `__post_init__`. (ids 45/2/48 same root; 46 = its test gap.)

### Medium
2. **[medium]** `decode_ce_linear_chunked` ignores `decode_precision_scaled` — builds logits from
   bare `mu` (`vfe3/model/prior_bank.py:593`), no sigma passed (`vfe3/model/model.py:761`), while
   the dense `_decode_linear` (`prior_bank.py:810`) scales by `mu/(sigma+eps)`. Train (chunked) vs
   inference (dense) mismatch. *Fix:* apply the scaling in the chunked kernel, or guard the combo.
   (ids 0/10.)
3. **[medium]** No config guard for `decode_precision_scaled=True` + `use_prior_bank=False` +
   `*_chunked` decode_mode — the only guard at `vfe3/config.py:1267` covers `use_prior_bank=True`.
   *Fix:* warn/error on the silent-drop combo. (id 1.)
4. **[medium]** Single-block precision-attention path has **zero** regression coverage — the
   `_model` fixture hardcodes `n_heads=2` (`tests/test_precision_toggles.py:28`). *Fix:* add a
   `glk`/`n_heads=1` + varied `sigma_log_embed` case. (id 46.)
5. **[medium]** `ablation.py:73` `BASELINE_CONFIG` is a hardcoded dict literal, not the documented
   `from train_vfe3 import config` (docstring at lines 15-16 falsely claims the import;
   `scaling.py:45` does it correctly). Sweeps can silently drift from the training operating point.
   *Fix:* import the real config. (id 13.)
6. **[medium]** Non-PD Rényi blend returns a spurious `0.0` instead of `kl_max` on the full-cov
   `alpha>1` path — `safe_cholesky`'s escalating ridge rescues a negative-definite blend so the `ok`
   mask is True (`vfe3/families/gaussian.py:331,361`); fp64 slogdet drops the sign and
   `safe_kl_clamp` floors to 0. *Fix:* drive the `ok` mask off `eigvalsh(min)>0`, matching the
   diagonal arm. (Warned non-default `alpha>1` regime.) (id 38.)

## Fixes Applied (branch `vfe3-audit-fixes-2026-06-17`)

All behavioral fixes were TDD'd (failing test first, watched fail, minimal fix, watched pass).

**Fixed:**
- **[critical, id 45/2/48/46]** precision_weighted_attention single-block crash — `model.py` now
  branches on `len(group.irrep_dims)==1` and folds a head-axis-free `(B,1,N)` bias against the
  headless energy. +3 single-block regression tests (closing the id-46 coverage gap).
- **[medium, id 0/10/1]** `decode_ce_linear_chunked` now accepts `sigma_q` and applies
  `mu/(sigma+eps)` when `decode_precision_scaled`, matching the dense path; call site forwards sigma.
  No config guard needed (the silent drop is gone). +2 tests.
- **[medium, id 38]** alpha>1 Rényi `ok` mask now gates on the blend's eigenvalue sign, so a non-PD
  blend → `kl_max` not `0`. +1 test.
- **[medium, id 13]** ablation module docstring corrected (BASELINE_CONFIG is self-contained, not
  imported) — left as a hand-synced copy rather than forcing an import that would silently swap
  ablation-specific toggles.
- **[low, ids 39, 44, 47, 28, 31, 32, 34, 12, 16]** condition_number → +inf on non-PD (id 39, +1
  test); MahalanobisNorm dispatch on `mu.dim()+1` (id 44); T5 bucket self-clamp (id 47); `Sequence`
  import (id 28); device/`_require`/`NoReturn` type hints (id 31/32/34); banners print
  `len(group.irrep_dims)` (id 12/16).

**Deferred (low, with reason):**
- ids 41/42/43/14 (SPD-retraction trust calibration + sigma_max defaults): no live-path impact
  (step_size is always 1.0, the bare defaults are exercised only by golden tests); changing them
  churns golden-tested kernels for cosmetic gain. Both retraction arms are valid retractions.
- ids 18/19/23/20/24/25/27 (device syncs + dense-Omega allocations): all on eval/log/diagnostic
  cadence, not the training hot path; loop-perturbation risk outweighs the marginal gain.
- ids 7/11 (n_layers>1 M-step capture / intermediate allocations): the default `n_layers=1` is
  exact; this is a structural limitation of a non-default config, not a bug.
- ids 29/30/35 (bare-`Callable` registry typing): zero runtime effect and no mypy in CI; a separate
  typing pass, not a bugfix.
- id 40 (FullGaussian.entropy non-PD mask): a deliberate, previously-overruled decision (2026-06-13
  audit / memory 19625) — left as-is by design.
- id 26 (generate() KV-cache): documented correct-but-slow; a feature rework, out of audit scope.
- id 17/15 (duplicate `vfe3/train.py` `_banner`): the project rule is to leave pre-existing dead/
  duplicate code, not delete it; only the n_heads display (id 12) was corrected.
