## Surviving punch list (all severity **low**; grouped by relevance)

### Tier 1 — Real defects gated behind Route B (`regime_ii_covariant`), the path under active development

These are the most actionable findings. They are "low" only because Route B is OFF in the
live config — they would bite the moment it is enabled on the live `block_glk`/K=70 frame.

1. **`gauge_invariant_edge_features` is fp32-fragile on the live non-compact frame** — `transport.py:148-165`.
   Cholesky/solve run in input dtype; the sibling sandwich `transport.py:773-775` deliberately
   uses a float64 island "because the congruence squares cond(Ω)", but the edge-feature path has
   none. Probe (block_glk, K=70, φ-scale 1.5 ≪ retraction max_norm 5.0): cond(cov_kt)≈2e9, fp32
   features `[4400,43630,4.6]` vs fp64 `[2435,20473,5.5]` → **113% relative error**; at higher
   φ-scale fp32 Cholesky fails (non-PD). The claimed gauge invariance degrades numerically when
   the path is actually used. *Fix: run the whole congruence (lines 430/433) and the three
   Cholesky-solve invariants in a float64 island, matching `transport_covariance`'s M4 sandwich.*
2. **`gauge_invariant_edge_features` uses raising `torch.linalg.cholesky`, not `safe_cholesky`** — `transport.py:151-152`.
   A non-PD transported covariance aborts the whole forward; every other SPD Cholesky in the
   package routes through `safe_cholesky` (cholesky_ex + ok-mask). *Fix: use `safe_cholesky` and
   drive non-PD pairs to NaN via the ok mask, mirroring `families/gaussian.py`.*
3. **`_build_regime_ii_covariant` `AttributeError` if `sigma=None` while `connection_M` is set** — `transport.py:403` (`sigma.dim()` after the early-out). Contract-hardening guard; current callers always pass σ. *Fix: raise a clear `ValueError` after the early-out.*
4. **`_REGIME_II_CHUNK_ELEMS` budget underestimates peak ~5×** — `transport.py:325`. The chunk loop
   simultaneously holds 4–5 dense `(B,C,N,K,K)` tensors, not one; the OOM fix may not prevent OOM
   on smaller GPUs. *Fix: cut the budget ~5× or document the true multiplier.*

### Tier 2 — Diagnostic / figure-faithfulness consistency defects (never touch loss/training)

5. **`gamma_attention_maps` omits `connection_M` and `_fold_precision_bias`** — `model.py:1136-1147`.
   Diverges from sibling `diagnostics` (1284,1296) and `attention_maps` (1546,1562); under Route B
   the gamma figure converges its belief under flat transport, mis-framing the figure. Reported by
   3 agents. Dead under live flat config (`connection_M` never created). *Fix: add
   `connection_M=getattr(self,"connection_M",None)` and the `_fold_precision_bias` call to match siblings.*
6. **Headline curvature metric `holonomy_deviation` is the frame-dependent ‖H−I‖_F**, not the
   gauge-invariant Wilson trace — `metrics.py:91`, logged at `model.py:1404`. The gauge-invariant
   `holonomy_wilson` (Re Tr(H)/K) is *already computed* and in the diagnostics dict (`model.py:1409`),
   just not the logged primary. The wiki itself flags this exact bug class. Diagnostic-only. *Fix:
   report `1 − Re Tr(H)/K` as primary, ‖H−I‖_F as secondary.*
7. **`extract.py` has no Route B saturation diagnostic** — `extract.py:358` keys `regime_ii_delta_max_norm`
   on `transport_mode=="regime_ii"` only; under `regime_ii_covariant` it is silently absent. *Fix:
   add a `regime_ii_covariant_delta_max_norm` branch.*

### Tier 3 — Theory/numerical caveats on opt-in paths

8. **`sigma_max` eigenvalue clamp breaks SPD affine-invariance when active** — `retraction.py:187`
   (`gaussian_full` arm). Congruence residual jumps from ~1e-6 (clamp inactive) to ~2.6 at
   sigma_max=5. *Fix: document as a non-equivariant numerical ceiling; keep it well above the
   operating spectrum (live=100.0) or clamp a congruence-invariant scalar.*
9. **`spd_affine` diagonal vs full arms use inconsistent trust-region geometry** — `retraction.py:128`
   (per-coordinate L∞ box) vs `:176-177` (Frobenius L2 ball); differ by up to √K for the same
   tangent, though both are dispatched as "the same affine-invariant exp map". Full arm dead under
   live diagonal family. *Fix: clamp the L2 norm of the whitened tangent in the diagonal arm.*
10. **Decode boundary hardcodes α=1 KL on a hardcoded family** — `prior_bank.py` (`reference_decode`
    + fused kernels) never read `cfg.renyi_order`/`cfg.divergence_family`; the "registry behind
    every seam" claim is unmet at the *divergence* axis of the decode seam (it is a covariance-only
    seam). `use_prior_bank=False` routes around it. Documented choice; wiki-drift only.
