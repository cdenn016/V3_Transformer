## Theory-Shortcomings Assessment

The requested comparison of the implementation against the principled theory, consolidated
from the theory-gap analyst and the seven domain experts (31 raw entries, deduplicated to
twelve themes). These are shortcomings and scope boundaries, not defects, except where a
finding number is attached. A recurring positive result: documentation candor is unusually
high — nearly every gap below is already acknowledged in README.md, config docstrings, or
the manuscripts themselves, and the theory-gap analyst verified that the pure paths the
project's own standard requires do exist for every axis except the two flagged as findings
(N17, and the Route-A absence noted in theme 5).

**1. The objective boundary (largest gap).** The canonical F carries the data term
−E_q[log p(o|x)]; the trained scalar is next-token cross-entropy through the unrolled
beliefs, and the inner E-step descends the target-blind F (self-coupling + belief coupling
+ entropy). `free_energy()`'s `log_likelihood` argument is a gated stub with no production
caller (`free_energy.py:405-415,464`), and the default belief descent runs through the
closed-form kernel, which never calls `free_energy()` at all — so no executable path
optimizes one ELBO. The manuscripts declare this deliberately (GL(K)_attention:899-921:
the observation-inclusive functional "is not the implemented E-step objective"; README
"Objective boundary"). This is the single most consequential gap for any "the transformer
minimizes one free energy" research claim.

**2. Hierarchy channels are partial.** The h→s→p→q hierarchy is implemented same-scale
and piecewise: r is one frozen token-uniform (K,) Gaussian (learnable_r opt-in;
barycenter update exact only for the uniform-vocab objective); the gamma block is either a
detached loss regularizer (predictively inert — s never feeds q) when `s_e_step=False`, or
the s E-step descent direction gated off the loss when `s_e_step=True` (deliberate
no-double-count routing); s-channel terms are mean-reduced while the belief block is
sum-reduced (an implicit 1/(B·N) recalibration absorbed into lambda_h/lambda_gamma);
`_refine_s` hardcodes diagonal family and flat transport even when the belief channel runs
full covariance or regime_ii (`model.py:801,813`; warned at `config.py:2123-2137`). No
executable path realizes the joint canonical F with both channels live and coupled into q;
the cross-scale shadow prior r_i is explicitly assigned to MAgent_Model (PIFB:2334).

**3. Covariance geometry: the default is a projection, not a group action.** The
GL(K) invariance theorem covers full-Gaussian pushforwards; the default diagonal family
transports Sigma as diag(Omega Sigma Omega^T) (`transport.py:1643`), which is not closed
under general GL(K) — pair-score gauge invariance and cocycle path-independence fail on
the default route (exactly invariant only under diagonal-scaling/permutation subgroups).
The exact sibling exists and is reachable: `family='gaussian_full'` (fp64 congruence
island, exact Cholesky KL, full natural gradient, SPD retraction registry, full-cov
decode), with the caveat that full-cov E-step gradients route to the autograd oracle and
need `oracle_unroll_grad=True` for through-inference signal. Related precision asymmetry:
the full-cov divergence itself is fp32-only (finding N11) while the retraction has an fp64
island.

**4. E-step estimators optimize different objects than the theory gradient.** The default
`gradient_mode='filtering'` is a parallel mean-field own-row update map (key-side
appearances detached, `oracle.py:110-111`) — "an update-map iteration, not an exact
coordinate minimization," with no monotone global-F descent guarantee; the manuscript says
the same (GL(K)_attention:956-981). The pure smoothing gradient exists as an opt-in
(oracle route). `straight_through` drops second-order terms; `detach` severs everything;
on non-kernel routes the oracle returns a detached tangent unless `oracle_unroll_grad=True`
(runtime warning fires). `e_phi_lr` defaults to 0 — the frame is not descended during
inference on the default path, and outer AdamW on phi_embed is not the Fisher/pullback
geometry the supplementary derives (the geometric M-step is the opt-in
`m_phi_natural_grad` + `pullback_per_block`). `mm_exact` under state-dependent alpha
minimizes a frozen-alpha majorizer with a damped step, not a one-step exact argmin — and
its strict pair mask breaks the majorizer property outright (N18, documented in README but
contradicted by the config docstring).

**5. Transport and group scope.** The realized transport is a Frobenius-clamped exp image
strictly inside GL+(K): beyond ||M||_F=20 the operator is a surrogate for exp(M) (exact
within the clamp ball; skew groups are exact at any norm); the det<0 component is
reachable only via the opt-in reflection/Metropolis machinery. Regime II's learned
bilinear connection is gauge-invariant only at W=0 (sanctioned exception 3); Route B is
covariant only up to the eps=1e-6 Cholesky ridge and sees the belief pair only through
three scalar invariants. There is NO exactly-equivariant non-flat (Route A,
commutant-restricted) connection, and the canonical F carries no curvature/Wilson term —
holonomy exists only as a diagnostic (`metrics.holonomy_wilson_sampled`); Route A is a
marked TODO (`transport.py:724-729`). The flat Regime-I cocycle is the only
exactly-equivariant transport. The s channel is always transported flat regardless of
`transport_mode` (theme 2). Frame composition is BCH-truncated at Dynkin order 4 —
configurable for the positional compose (`bch_pe_order`) but hardcoded in the phi
retraction (N15) — with no exact matrix-log composition chart; the saving grace is that
the flat transport itself, exp(phi_i)exp(−phi_j), never goes through BCH and is exact.

**6. Positional structure.** Default gauge-RoPE is means-only: the mean transport is
rotated (R_i Omega_ij R_j^T) while the covariance sandwich stays on the un-rotated base,
so the transported pair is not a single congruence image (probe: Mahalanobis invariant
7.136 → 7.160). The coherent sibling `rope_full_gauge=True` exists and correctly requires
`gaussian_full` (diagonal cannot hold a rotated covariance without projecting). RoPE is a
gauge-fixing layer (position-fixed rotations don't commute with a global gauge element;
config warns, measured beta residual 0.44-0.61 vs 7e-6 without). The T5 relative bias is
one shared bucket table, not per-head (ALiBi does get per-head slopes); the per-head T5
axis does not exist yet.

**7. Attention block details.** The two-hop block uses detached beta@beta weights with
deliberately no entropy partner (W2 is a derived composition, not a variational row
distribution). The entropy-suppressed surrogate is a documented ablation toggle for which
softmax-beta is provably non-stationary (probe: 0.33 projected gradient vs 1e-8 canonical)
— correctly routed to the oracle, never the envelope kernel. The mixture-KL ELBO reading
holds only at tau=1, while the default tau = sqrt(d_h); registered non-KL divergences keep
the Gibbs row solution but lose the ELBO interpretation (README states this precisely).

**8. The structural self-attention sink.** Under the flat cocycle Omega_ii = I, so
E_ii = 0 and the self edge always carries the largest weight — a structural sink implied
by the theory itself. The manuscripts do not address it; the code counters it with ad-hoc
noself priors and mm_exact's strict pair mask, neither derived from the variational
principle. This is a theory gap surfaced only in code comments.

**9. EFE policy path.** The only live ambiguity estimator makes ambiguity equal predictive
entropy, so the epistemic term is identically zero and G reduces to pragmatic
cross-entropy to the preference; sigma-dependent MC ambiguity deliberately fail-closes;
multi-step rollouts score only the terminal predictive, not the per-step sum. All three
restrictions are stated in policy.py/config.py; the scorer is no-grad, opt-in, default OFF.

**10. Manuscript constructs with no code counterpart.** Base-manifold fields and presence
functions chi_i(c), connection one-forms and curvature F_munu, cross-fiber morphisms Phi_i
with independent K_q ≠ K_p, and the multiscale meta-agent tower with cross-scale shadow
priors. The manuscripts themselves scope the transformer as the zero-dimensional
single-scale limit (PIFB:1642) and assign the multiscale machinery to MAgent_Model.

**11. Code machinery with no theory counterpart.** The kl_max saturation clamp (freezes
gradients of saturated pairs and reshapes the effective landscape), eps variance floors
and the sigma_max cap, mean/sigma trust regions, the detached reliability bias and
detached gamma-posterior probability-space prior fold, the detached query-adaptive
temperature, the cross-block prior handoff recurrence, the learnable decode temperature,
z-loss, EMA, and default weight decay on log-sigma tables (config itself flags the last as
"an unintended lognormal prior pinning sigma to 1"). Most default to OFF or the pure
value; the clamps/floors have no off switch but are inert on the healthy pipeline.

**12. Decode boundary.** The KL-to-prior decode always scores −KL(q‖pi_v)/tau at fixed
alpha=1 KL on a hardcoded family, never reading `cfg.renyi_order`/`divergence_family` —
under a non-KL E-step objective the decode boundary silently stays KL (config-time warning
only; no divergence-matched decode kernel exists). Additional small drifts: PIFB's
"learnable per-coordinate b0/c0" claim vs fixed constants in code (N16); the manuscript
RoPE frequency ladder 10000^{-2n/d} vs the config default rope_base=100 (config value —
user-owned, not re-deliberated).

For completeness on the faithful side: the info-geometer confirms the code is faithful to
what the manuscripts actually claim on natural gradients (Fisher metric only — the
manuscripts do not claim Amari dual/alpha-connection geodesics, so their absence is a gap
against the broader IG framework, not against the stated theory); softmax-beta
stationarity, the envelope identity, the state-dependent-alpha envelope cancellation, the
per-head temperature, T5/ALiBi/causal priors, the flat cocycle's exact trivial holonomy,
and the full-covariance Cholesky KL island are all implemented exactly as derived.
