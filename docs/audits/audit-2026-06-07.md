## FIXED (this session)

### [HIGH] Full-covariance SPD retraction emitted 100% NaN gradients at the isotropic init `Sigma=I`
`vfe3/geometry/retraction.py` — `retract_spd_full` / `retract_logeuclidean_full`. The symmetric-eigh
adjoint's `1/(lambda_i-lambda_j)` gap terms diverge on a degenerate spectrum, and `Sigma=I` is the
**default `gaussian_full` prior init**, on the **default `unroll` estimator** (not gated behind the
opt-in oracle). The forward was fine; the first backward NaN-poisoned the optimizer step, so the
full-covariance pure path was unusable from its own default init.
**Challenge:** skeptic→DOWNGRADE-to-medium (opt-in non-default family), defender→UPHELD-at-high (it is
the *pure* full-cov gradient path, and the project's audit invariant is "the pure path must exist/work").
**Adjudication (re-read source): UPHELD at high** — a first-class registered family whose gradient is
100% NaN on its own default init is a broken pure path, exactly what the audit exists to catch; the
non-default-ness bears on priority, not severity.
**Fix (`f069a8a`):** custom autograd `_EighDamped` with a Lorentzian-damped gap `Delta/(Delta^2+eps)`
at all eigendecomposition sites; forward bit-identical, backward finite. Validated against stock
`torch.linalg.eigh` backward on well-separated spectra (independent oracle), FD gradcheck, and finite
backward at `Sigma=I` end-to-end. **Review note for the author:** changes gradients on the opt-in
`gaussian_full` family (small bounded bias near genuine degeneracies); `gaussian_diagonal` untouched.

### [HIGH→resolved] `min_lr_frac=0.01` default reddened two scheduler tests (suite was RED)
`vfe3/config.py:267`. The 2026-06-06 edit-doc's own Changes section states the intended default is
**0.0** ("keeps current behavior"); the committed `0.01` contradicted it and made the proportional
floor dominate the absolute `min_lr`, breaking `test_scheduler_floors_lr_at_min_lr` and
`test_scheduler_min_lr_zero_is_pure_cosine`. **Challenge:** both sides agreed it is a real red-suite
regression with a one-line fix and no correctness defect (severity MEDIUM, not high).
**Decision (#1 confirm-this):** restored `min_lr_frac=0.0` (minimal, trivially-reversible, leaves the
five just-authored LR tests untouched) — this matches the edit-doc intent and dissolves Finding C
(ablation now inherits pure cosine). If you actually intend a 1%-of-base default floor, instead set
`min_lr_frac=0.0` in the two tests and assert the combined `max(min_lr, min_lr_frac*base)`.

### [MEDIUM] ablation validation loader shuffled + dropped the tail (F1 fix not mirrored)
`ablation.py` `get_loader` called `make_dataloader` without the eval flags, so validation/test ran the
train-regime `shuffle=True, drop_last=True` (datasets.py:134-135, whose own docstring forbids it for
eval). **Challenge:** both sides DOWNGRADE high→medium (leaderboard ranking is safe — deterministic
reseed + no active sweep changes batch/seq — but absolute val-PPL is a ~97%-of-corpus shuffled measure).
**Fix (`ac19d2b`):** `get_loader` now threads `shuffle=(split=='train'), drop_last=(split=='train')`.

### [MEDIUM] ablation resume could serve a wrong-dataset cell as current
`ablation.py` `_cell_is_current` compared only the VFE3Config (which has no `dataset` field), so a rerun
on a different dataset with `resume=True` skipped the cell. **Fix (`ac19d2b`):** also compare the
persisted top-level `config.json["dataset"]` against the session dataset.

### [LOW, real crash] `cross_couplings` JSON round-trip crashed cold-start figure regen
`viz.report._load_config` rebuilds `VFE3Config(**json)`, where `cross_couplings=[(0,1)]` round-trips as
`[[0,1]]` (lists) and failed the `isinstance(pair, tuple)` gate. **Fix (`ac19d2b`):** coerce list pairs
to tuples in `config.__post_init__` before validating.

### [LOW] min_lr / min_lr_frac validators admitted NaN
`config.py` `v < 0.0` lets `NaN` through. **Fix (`ac19d2b`):** also reject `v != v` (NaN).

### [LOW] means-only RoPE docstring overstated invariance; train_vfe3 "EVERY toggle" claim false; stray `''`
**Fixed (`ac19d2b`):** tightened the `RopeTransport` docstring (mean uses rotated operator, covariance
the bare Omega → Mahalanobis invariant not preserved; coherent path is `rope_full_gauge`); surfaced
`min_lr_frac`+`amp_dtype` in the click-to-run dict and softened the "EVERY toggle" claim; removed the
stray `''` SWEEPS literal (ablation.py); deleted three atomic-write `*.tmp.*` scratch files from root
(tooling debris — the Edit tool itself leaves them; swept again at session end).

## NOT fixed (deliberate — rationale)

- **[MED] `amp_dtype='fp16'` has no GradScaler** (`config.py`). This is a **documented, deliberate
  deferred stub**: `config.py:288-291` states "fp16 TRAINING would need a GradScaler in the M-step — a
  deferred follow-up (this toggle is forward/inference-correctness scope)." Rejecting it would remove
  the author's intentional stub; adding a GradScaler is the deferred feature, not a bugfix. `bf16` is the
  supported AMP path. Left as-is (recommend: wire a GradScaler if/when fp16 *training* is wanted).
- **[MED] `straight_through` E-step silently freezes learnable alpha / connection_W / lambda_beta**
  (`e_step.py:370-371`). Real opt-in footgun (default is `unroll`), but the fix is a config-validation
  restriction on a toggle *combination* — a judgment call about intended use. Recommend a guard (raise
  or warn) when `e_step_gradient='straight_through'` is combined with a learnable alpha / `regime_ii` /
  `learnable_lambda_beta`. Left for the author's call.
- **[MED, off by default] Two dense-Omega hot spots** — `phi_alignment_loss` (`e_step.py:268`, only when
  `e_phi_lr>0`, default 0) and the `gamma_coupling` forward (`model.py:520`, only when `gamma_coupling>0`,
  default 0) materialize the dense Omega instead of the factored route. These are the known dense-Omega
  P0 surfacing on two more paths; both are perf-only and gated OFF by default. Belongs to the existing
  speedup roadmap (`docs/perf/2026-05-31-speedup-opportunities.md`), as a behavior-preserving refactor
  with golden equivalence — not bundled into this correctness pass.
- **[LOW] `attention_tau` uses one temperature across heads** (`free_energy.py:32`) — flagged by the
  gauge/variational lenses for unequal-irrep multi-block groups. This is adjacent to the settled
  per-irrep-block-beta design and the F8 banner-tau triage; needs a focused theory check against the
  manuscript before any change (likely by-design for the equal-block default). Left, flagged.
- **Pre-existing dead code / unused imports** (`run_training`+duplicate `_banner` train.py:499;
  `gauge_parameterization` field config.py:59 validated-but-never-dispatched; unused imports in
  metrics.py/datasets.py/closure.py/prior_bank.py/extract.py). Per CLAUDE.md "leave pre-existing dead
  code — mention it, don't delete it." Mentioned, not touched.
- **~30 LOW diagnostics/metrics findings** — registered `holonomy_deviation` uses the biased row-major
  estimator (not the sampled one); `effective_rank` passes raw sigma not eigenvalues under full-cov;
  `attention_entropy` registered metric returns Shannon H(beta) while the CSV column under the same key
  carries `tau*KL(beta||pi)`; `gauge_trace_spread` is identically 0 for unimodular SO/Sp; `fisher_trace`
