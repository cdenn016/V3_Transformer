## Surviving Punch List (post-challenge, ranked)

1. **[HIGH — repo health]** `main` is red: **10/490 tests fail.** Fix the suite before anything else
   (see § Test Suite for the proven root cause and the two clean resolutions). A red `main` makes every
   branch start red and masks real regressions.
2. **[medium — gauge correctness]** Per-block sl(K)/trace det-control silently over-subtracts by
   `n_heads` under `tied_block_glk` (`lie_ops.py:284-286`, `:302-307`; root `_block_trace_vectors:253`).
   The orthogonal-projection formula assumes the per-block trace functionals are independent, but the
   tied generators `kron(I_n, E_ij)` make all `n_heads` rows identical, so `project_phi_to_slk` /
   `clamp_phi_trace` drive `det Ω → 0.68` instead of 1.0. Reachable via `pos_phi_project_slk=True` +
   `tied_block_glk` (`model.py:267`→`positional_phi.py:134`). *Fix:* joint Gram solve
   `coeffs = pinv(V Vᵀ)(V·φ)`; add a config guard mirroring `killing_per_block` at `config.py:458`.
   *(Doubly-opt-in; the hot E-step det-control is unaffected.)*
3. **[medium — geometry consistency]** `sigma_max` means a variance ceiling on the diagonal retraction
   (`retraction.py:62`, LE `:268`) but an *eigenvalue* ceiling of `sigma_max²` on the full retraction
   (`:117`, LE `:213`). The shared dispatcher forwards one `sigma_max` to both arms, so the same
   physical quantity is bounded a factor `sigma_max` looser on the full family. *Fix:* clamp full-cov
   eigenvalues to `sigma_max` (not squared) for one variance convention across the `cov_kind` seam.

### Root cause — proven, framed neutrally

Nine of the ten failures share one cause: the config default `pos_phi` is **`"learned"`**
(`config.py:93`), but (a) its own comment one line above still says "BCH positional encoding
(**default-off**) … the pure path is `none`" (`config.py:89-92`), and (b) `test_pos_phi_defaults_off_and_validates`
asserts the default is `"none"` (`test_config.py:256`). `pos_phi="learned"` is default-ON, so a learned
positional-gauge parameter table is created for every default-config model, changing the forward and
the optimizer parameter groups — which breaks the frozen-oracle pin, the byte-identity / copy-equivalence
tests, the linearity tests, the learnable-alpha test, and the optimizer-group test.

**Proven by experiment:** temporarily setting the default back to `pos_phi="none"` and re-running the
nine non-`test_viz` failures gives **9/9 passed** (the diagnostic edit was reverted; `config.py` is
back at `main`'s state). The flip rode in on commit `52162a5` (`chore(sync): local working tree ->
remote`).

The evidence (a `chore(sync)` commit, a comment that still says "default-off," and nine tests encoding
`"none"`) leans toward an **accidental** flip during the sync rather than a deliberate default change.
Either way, the default, the comment, and the suite disagree and `main` is red. This audit does **not**
pick the value (per the user's "don't worry about config values" directive). The two clean resolutions:

- **If accidental:** revert `config.py:93` to `pos_phi = "none"` — restores green and matches the
  comment + tests.
- **If intended:** update the `config.py:89-92` comment (drop "default-off"), fix
  `test_pos_phi_defaults_off_and_validates`, and re-baseline the 8 forward-equivalence / byte-identity /
  optimizer-group tests against the `pos_phi="learned"` default.
