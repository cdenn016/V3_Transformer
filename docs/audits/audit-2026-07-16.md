## Executive conclusion

The July 16 change set was unusually broad: 118 files changed, with 19,824
insertions and 2,104 deletions. The reported Spyder reset was a confirmed native
runtime abort, not a Python exception and not a failure in `numpy.corrcoef`
itself. PyTorch and the Anaconda NumPy/SciPy stack had initialized incompatible
copies of Intel OpenMP in one long-lived Spyder process. The first later NumPy
kernel entering that runtime, here `plot_free_energy_codescent`, triggered OMP
Error 15 and called the native abort path. Python `try`/`except` blocks cannot
contain that failure.

Future training finalization with `generate_figures=True` now launches one
disposable figure interpreter.
Only that child receives `KMP_DUPLICATE_LIB_OK=TRUE`; the training process and
Spyder kernel do not. A child failure can no longer abort the training
interpreter, and numeric artifacts are published before the child starts. The
child also owns every Matplotlib render initiated directly by the two terminal
finalizers, the model-replay publication set, and the two saved-probe figures
that initially remained in the parent. Valid resumed
best checkpoints are compared through the selected-weight semantic config, and
a run without `best_model.pt` supplies a temporary self-bound snapshot of the
live terminal model. No manual environment variable or separate
`make_figures.py` invocation is required for figure-enabled future runs.

The audit originally confirmed one High scientific-integrity defect, twelve
Medium defects or operational risks, four Low defects or disclosure issues, and
one unresolved numerical risk. The current remediation closes every actionable
in-scope item in source. The EFE implementation and its former M-10 observation
are outside the user-directed scope and were neither evaluated nor changed in
this repair. The head-mixer and diagonal-covariance items remain explicit
scientific disclosures with config-selected pure paths rather than silent
claims of exact equivariance. Because the user stopped executable testing, the
closure status is implemented and independently source-reviewed, not
runtime-verified.

Deletion of the slow UMAP tests is accepted by the user and is not a finding in
this report.

## Method and scope

The audit used separate waves for implementation/runtime behavior, code quality,
debugging, Python correctness, performance, numerical analysis, differential
geometry, gauge theory, information geometry, variational inference, and
transformer architecture. A separate verifier reconciled reachability and
severity. The surviving High finding then went through an isolated skeptic and
defender challenge. The review followed executable paths and config wiring; code
comments were treated as context, not proof.

The Research vault's VFE program index was consulted for the program's existing
purity and evidence boundaries. The vault was not modified.

## Remediation closure

The upheld High ablation finding is closed by a versioned aggregation contract
that binds every admitted row to the invocation's code identity, corpus and
tokenizer identities, data order, token budget, baseline semantic
configuration, requested diagnostics, and exact checkpoint bytes. Publication
now requires every requested current cell to be successful and contract-bound.
Historical tack-on rows are admitted only when they reconstruct under the same
contract, their seed belongs to the declared design, and each base label has
exactly one row for every declared seed. Incomplete and duplicate seed panels
are excluded as a unit, so an old survivor cannot enter a new aggregate or make
an incomplete sweep appear complete.

The operational findings are also closed in source. One shared portable path
validator covers Python 3.10 through 3.14, Windows device names, Unicode and
case aliases, component lengths, and output-name uniqueness. Scaling and
ablation recomputation remove only runner-owned stale artifacts after proving
directory ownership. Ablation cells now require a versioned sentinel bound to
the exact sweep, expanded label, and seed; a nonempty unowned or mismatched
directory fails before any file is overwritten, while one exact regular
legacy marker can be promoted once. Long-lived Spyder processes bind the imported package
identity and fail closed when source changes on disk. Run discovery is
recursive but accepts only finalized, finite-metric, strict-checkpoint-compatible
runs. CPU test subprocesses have bounded fast and slow lane timeouts and are
contained in disposable process trees.

Figure generation is entirely child-owned. Windows workers enter a Job Object
before they can spawn their workload, POSIX workers use a new process group,
and interruption, timeout, or parent exit terminates and reaps descendants.
Figure directories reject symlinks, junctions, and reparse points before any
write. PNG and generated side-artifact publication uses unique sibling
temporaries and atomic replacement. On-demand generation derives reliability
from the current report split, accepts saved reliability only with matching
split provenance, prepares missing depth and phi probes from the already-loaded
model and retained batch, and rebuilds every dashboard supported by persisted
metrics. A strict-opt-out historical run cannot exactly reconstruct the
full-resolution loss curve because the per-step in-memory loss list was never
persisted; the driver and README now state that limit instead of promising a
complete replay.

The report path now budgets from materialized batch shapes and dtypes, clips an
oversized terminal batch to its actual cap, decodes each retained belief once,
shares CE and vocabulary reductions, and releases full-vocabulary worksets
before the next allocation. Corpus hashing and unigram counting share a bounded
pass and cache. Public full-SPD outputs receive strict lower and upper Cholesky
certificates in their represented dtype; only failing rows are repaired, the
fallback is recertified, and an interval without a representable interior fails
closed. The transport inverse uses the true stored group inverse, and the SPD
distance is labeled as the affine-invariant Riemannian metric rather than the
half-scaled Gaussian Fisher covariance distance.

## Reconstructed failure and permanent repair

The supplied crash stack reached
`vfe3/viz/figures.py:865`, in `plot_free_energy_codescent`, through
`vfe3/run_artifacts.py`'s end-of-run figure pass. The native message identified
duplicate `libiomp5md.dll` initialization and terminated the interpreter with
`Fatal Python error: Aborted`. Reproduction in the user's Anaconda interpreter
also aborted inside a NumPy/SciPy covariance kernel when the duplicate-runtime
bypass was absent.

The repair boundary is `vfe3/run_artifacts.py:465-560` and
`vfe3/viz/figure_worker.py`. The parent snapshots only the bounded report batches
needed by the publication pass, writes a short-lived JSON request, optionally
offloads the live CUDA model while the child is active, launches the same Python
executable as a module, restores the model in `finally`, and reports child
stderr without converting a figure failure into a training failure. Temporary
request, batch, and live-model files are same-directory unique files and are
removed after the worker exits.

The first repair still left two Matplotlib calls in the parent, compared a
resumed best checkpoint against the entire output-policy config, started CUDA
offload just outside its restoration guard, and could omit publication figures
when no selected checkpoint existed. The source-only follow-up in `46669c3`
closes those gaps. A final source check also excluded `force_large_figures` from
the selected-weight config projection because that allocation policy cannot
change trained weights. That follow-up was not re-run after the user instructed the
audit to stop testing, so its status is implemented and source-inspected, not
runtime-verified.

## Adversarial challenge of H-1

The skeptic first steelmanned the claim, then attempted to refute reachability and
severity. It conceded the finding because current-cell contract checks at
`ablation.py:2582-2591` govern cache reuse only; the later collector at
`ablation.py:2457-2474` reads all historical success markers without their
contracts. The skeptic's decisive falsification condition was an executable
pre-aggregation contract gate.

The defender traced the default click-run configuration through `main`,
`run_sweep`, the whole-directory collector, CSV publication, seed aggregation,
sensitivity plots, and winner selection. It showed that the omitted fields are
the exact comparison-defining identities already present in the current-cell
contract. It would drop the finding only if every admitted row were partitioned
by compatible code, corpus, tokenizer, data-order, token-budget, and normalized
common configuration, or if the executable workflow enforced a fresh output
root for every cohort.

Adjudication: **UPHELD at High**. Both sides identified
`ablation.py:2458` as the decisive source location. The finding is reachable
under the checked-in default reuse workflow and can silently change scientific
rankings rather than merely produce an operational error.

## Findings not sustained

The proposed Log-Euclidean gradient defect was refuted: the implementation uses
the Frechet derivative and a Loewner divided-difference map rather than treating
the matrix logarithm elementwise. The allegation that chart safeguards are
entirely absent was also dropped; base clamps and projections remain active,
while the extra chart-domain rejection and monitoring switches are intentional
opt-in diagnostics. No information-geometry, variational-objective, or
transformer-attention defect survived verification beyond the scale-label and
explicit head-mixer disclosures listed above.

## Disposition

The recommended repair sequence above was executed in the same order: cohort
integrity first, then portable path and artifact ownership, bounded report and
corpus work, process containment, numerical certification, and theory-label
cleanup. No actionable in-scope finding remains open in this audit ledger. M-10
is retained only as a historical observation and is explicitly excluded from
this task because the user directed the audit to ignore EFE. M-12 and L-3 are
configuration-dependent disclosures with documented pure paths; changing the
user's selected regime was not part of the repair.

The next step is a fresh whole-codebase ten-expert audit from the merged repair
commit. That review must treat this ledger as historical context rather than
assuming closure, exclude EFE, follow executable code paths rather than
comments, and report any newly confirmed defect in a separate dated audit
artifact.
