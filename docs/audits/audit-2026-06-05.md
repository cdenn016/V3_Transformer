# Audit 2026-06-05 — Is the Rényi divergence working? (alpha_div ablation)

Four expert investigators (implementation, information-geometry, numerical-analysis,
variational) swept the Rényi/alpha-divergence path after the user reported two symptoms:
(1) `alpha_div < 1` trains roughly 2.5x faster than `alpha_div = 1`, and (2) the
`vfe3_ablation_results/alpha_div` sweep shows Rényi runs (alpha != 1) at validation
perplexity ~259-273 against ~159 for KL (alpha = 1). All four converge on the same
conclusion, with reproductions.

## Verdict

The Rényi divergence itself is correct and is not the problem. The `alpha_div` ablation
is **confounded**: `alpha_div = 1` and `alpha_div != 1` take different gradient code
paths, so the sweep measures gradient-estimator truncation, not divergence order. Both
reported symptoms are produced by that single asymmetry.

## Root cause — kernel-vs-oracle gradient asymmetry

`belief_gradients` (`vfe3/gradients/kernels.py:179-194`) uses the analytic hand kernel
only for filtering + gaussian_diagonal + renyi + `abs(alpha_div - 1) < 1e-9` + entropy.
Every `alpha_div != 1` falls back to the autograd oracle `belief_gradients_autograd`.
The hand kernel computes the gradient analytically on the LIVE belief and priors, so the
unrolled-through-inference signal reaches the prior tables regardless of any toggle. The
oracle keeps a differentiable gradient only when `use_live = create_graph and
mu.requires_grad and sigma.requires_grad` (`vfe3/gradients/oracle.py:87`); otherwise it
returns `grad_mu.detach(), grad_sigma.detach()` (`oracle.py:118`). `create_graph` is
`oracle_unroll_grad and e_step_gradient == "unroll"` (`vfe3/inference/e_step.py:357`), and
`oracle_unroll_grad` defaults to `False` (`vfe3/config.py:247`, plumbed through
`vfe3/model/block.py:61` as `cfg.oracle_unroll_grad`). Every saved ablation config has
`oracle_unroll_grad=false`, `e_step_gradient="unroll"`, `use_prior_bank=false`,
`e_phi_lr=0.0`, `n_e_steps=1`.

Consequence under that exact config: for `alpha_div != 1` the belief gradient returned to
the E-step is DETACHED, so the outer backward never traverses the divergence/transport for
the prior. This (a) drops the backward graph through the unrolled E-step (the speedup) and
(b) starves the learned prior and gauge-frame tables of the through-inference signal (the
worse perplexity). With `e_phi_lr=0.0`, the only route from the gauge-frame tables
(`phi_embed`, `pos_phi_free`) to the loss is through that belief gradient, so under Rényi
they receive `grad = None` and stay frozen at init.

Reproductions against the ablation config:

- Per-call `belief_gradients`: KL analytic kernel = 1.44 ms; Rényi (a=0.5) autograd oracle
  = 2.73 ms — the oracle is ~1.9x SLOWER per call. The observed 2.5x SPEEDUP therefore
  cannot come from a cheaper divergence; it comes from the dropped backward graph. The
  "faster" is the tell.
- Prior-table gradient through the E-step: KL (kernel) = 651; Rényi a=0.5,
  `oracle_unroll_grad=False` = 0; Rényi a=0.5, `oracle_unroll_grad=True` = 356 (restored).
- Gauge-frame tables: `phi_embed.grad`/`pos_phi_free.grad` present (0.0026) at alpha=1.0,
  `None` at alpha=0.99 and 0.5, restored (0.0008) with the toggle on.
- Wall-time with `oracle_unroll_grad=True` flips Rényi to ~1.08x KL (slower than KL, as a
  correctly-unrolled autograd path should be — never faster).

The cleanest falsifier is the sweep's own `alpha_div=0.99` row: perplexity jumps 159 ->
273 and wall-time 1672s -> 695s discontinuously at exactly the kernel-gate boundary
`abs(alpha_div - 1) < 1e-9`. Rényi is continuous in its order at alpha = 1 (it limits to
KL), so a genuine divergence-order effect must give PPL(0.99) ~ PPL(1.0). A discontinuity
at a code-path switch can only come from the kernel-vs-oracle asymmetry.

## The divergence math is correct (verified)

The diagonal closed form (`vfe3/families/gaussian.py:88-99`), the full-covariance form
(`:230-256`), and the per-coordinate form (`:101-131`) match the canonical multivariate
Gaussian Rényi divergence D_alpha(q||p) = (alpha/2) Delta^T Sigma_a^{-1} Delta -
1/(2(alpha-1)) ln(|Sigma_a| / (|Sigma_q|^{1-alpha}|Sigma_p|^alpha)), Sigma_a =
(1-alpha)Sigma_q + alpha Sigma_p (van Erven & Harremoës 2014; Gil/Alajaji/Linder 2013),
term-for-term, with correct q||p orientation. Executed checks pass self-divergence-zero,
non-negativity, monotonicity in alpha, the alpha -> 1 KL limit, full-vs-diagonal agreement,
and per-coordinate summation. The configured alpha genuinely reaches `renyi_closed_form`
for both the attention energy and the belief gradient; the alpha=1-only geometric-mean
kernel is correctly gated off for alpha != 1, so Rényi is not silently running as KL. For
alpha in (0,1) the blend is a convex combination, hence always SPD — the well-behaved
regime where the user's runs sit. The autograd gradient there is finite and matches finite
differences to fp32 precision, and Rényi saturates the `kl_max` clamp LESS often than KL,
so there is no conditioning, cancellation, or saturation pathology in the alpha<1 regime.

## Variational status (honest qualification, not a bug)

Softmax-beta remains the F-stationary attention under Rényi (verified: the beta-block
gradient equals the `-tau log Z` envelope gradient to ~1e-7 at both alpha=0.5 and 1.0),
because the row-Lagrangian needs only linearity of the energy in beta plus the entropy
term, not that the energy is a KL — disclosed at `GL(K)_supplementary.tex:1084`. EM
separation is intact. What is lost for alpha != 1 is the variational-bound interpretation:
F becomes an entropy-regularized consensus functional, not an evidence bound, since the
alignment block is no longer the `KL[q_i || Omega_ij q_j]` of the mixture free energy. The
manuscript discloses this (forward KL is the unique f-divergence preserving exp-family
closure and the dual attention interpretation, `GL(K)_attention.tex:771`; the closed-form
stationary belief is alpha=1-specific, `GL(K)_supplementary.tex:1096`). This is a
heuristic generalization by design, not a defect, and does not by itself force worse
perplexity.

## Secondary real effects (would prevent exact parity even after the fix)

Two genuine, well-behaved consequences of the divergence choice (not bugs) mean a corrected
rerun may still not land Rényi exactly on KL:

1. Self-coupling magnitude shift. alpha^(k) = c0 / (b0 + D^(k)) with D^(k) the
   per-coordinate self-divergence (`vfe3/alpha_i.py:116-135`). Since Rényi D_{alpha<1} <=
   D_KL, alpha^(k) is systematically larger for alpha<1 (measured mean 0.557 vs 0.424),
   a stronger pull of the belief toward its prior that changes the fixed point. Falsifier:
   an alpha<1 run with `alpha_mode="constant"` (ignores D) reproducing the gap would
   exonerate this.
2. Softer attention. D_{0.5} < D_KL pointwise, so at fixed tau = kappa sqrt(K) the logits
   compress and attention softens (measured entropy 1.79 vs 1.24). If kappa/tau is not
   retuned when switching from KL to Rényi, perplexity moves for that reason alone.

## Latent bugs found (not the user's alpha<1 issue)

1. Diagonal alpha>1 blend masking. `sigma_blend = (...).clamp(min=eps)`
   (`vfe3/families/gaussian.py:89`, and the per-coord twin `:123`) turns a non-PD blend
   (possible for alpha>1) into a tiny positive variance and a WRONG finite divergence with
   a nonzero gradient, instead of NaN -> kl_max. The full-covariance path handles this
   correctly via `safe_cholesky` + an `ok` mask (`:237-255`). Probe (alpha=2, sigma_q=1,
   sigma_t=0.1): diagonal returns 4.61 (blend -0.8 floored to eps), full-cov returns 100.0
   (kl_max). Medium severity, latent: only reachable on the diagonal family at alpha>1, a
   regime the user does not currently run. Fix: in the alpha != 1 diagonal branch, set
   elements with raw blend <= 0 to NaN before `safe_kl_clamp` (mirror the full-cov mask).
2. KL-branch threshold mismatch. The kernel routes KL at `abs(alpha_div - 1) < 1e-9`
   (`vfe3/gradients/kernels.py:183`) but the closed form switches to its exact-KL branch
   at `< 1e-6` (`gaussian.py:82`, `:117`). For alpha within (1e-9, 1e-6) of 1 the slow
   oracle runs while the formula computes exact KL. Low severity, no config sits there.
   Fix: use one tolerance for both.
3. Float32 hard-switch near alpha=1. `logdet_term / (alpha - 1)` loses precision in a
   ~1e-6..1e-3 band around alpha=1 with no Taylor branch (the manuscript mentions one,
   `supplementary:1096`). Low severity, no config sits there.

## Recommendation

This is a confounded-ablation / dangerous-default issue, not a missing-pure-path one: the
theoretically pure live-gradient path EXISTS via `oracle_unroll_grad=True` (config-exposed,
test-pinned at `tests/test_e_step.py:306`). For a fair `alpha_div` sweep, force the live
oracle on the non-kernel rows by adding `"requires": {"oracle_unroll_grad": True}` to the
`alpha_div` sweep entry (`ablation.py:493-496`), mirroring how `n_heads` forces
`use_head_mixer=False`. At alpha=1.0 this is a no-op (the kernel ignores the toggle), so
the KL baseline is unchanged; at alpha != 1 it restores the through-inference gradient and
the comparison becomes apples-to-apples. Then rerun the sweep. Expect Rényi to be slower
than KL (autograd vs analytic kernel), not faster, and the perplexity gap to shrink
substantially (residual differences then attributable to the two real effects above:
self-coupling magnitude and attention softness / kappa tuning).
