## Investigator Findings

### 1. code-reviewer

#### Resume checkpoint loading opts into arbitrary pickle execution

**Location:** `vfe3/run_artifacts.py:272`  
**Severity:** high  
**Evidence:** `load_checkpoint()` calls `torch.load(Path(path), map_location=map_location, weights_only=False)`.
`vfe3/train.py` reaches this loader when `resume_path` is set from `resume_from` or `cfg.resume_from`.  
**Fix:** Make resume checkpoints compatible with `weights_only=True`, or require an explicit
trusted-checkpoint opt-in before allowing `weights_only=False`.

#### Full-covariance precision-scaled linear decode is accepted but cannot run

**Location:** `vfe3/model/prior_bank.py:596`  
**Severity:** medium  
**Evidence:** linear decode uses `mu_q / (sigma_q + eps)` in both chunked and dense linear paths,
while full covariance passes a rank-4 covariance tensor rather than a diagonal variance vector.  
**Fix:** Reject `decode_precision_scaled=True` for full-covariance families unless the code implements
the full-covariance natural parameter, such as solving with `Sigma^{-1} mu`.

### 2. debugger/theory

#### Skipped M-steps still advance training state

**Location:** `vfe3/train.py:399`  
**Severity:** medium  
**Evidence:** a non-finite step can skip `_scaler.step(optimizer)`, but `barycenter_r_()` and
`scheduler.step()` still run after the skip branch.  
**Fix:** Gate `barycenter_r_()` and `scheduler.step()` on whether an optimizer update actually occurred.

#### Gradient accumulation reweights padded batches

**Location:** `vfe3/train.py:341`  
**Severity:** medium  
**Evidence:** each microbatch loss is backpropagated as `loss_mb / grad_accum_steps`, while valid-token
counts are only recorded later as `grad_accum_tok_spread`.  
**Fix:** Accumulate summed losses over valid tokens and divide by the total valid-token count, or reject
uneven valid-token counts before backpropagation.

#### Live-parameter probe masks its own failure

**Location:** `vfe3/train.py:993`  
**Severity:** medium  
**Evidence:** the report initializes `live=trainable` and `dead=0`; a broad `except Exception: pass`
leaves those values in place with `probed=False`.  
**Fix:** Return or log the exception and mark live/dead counts unknown when `probed=False`.

#### Scaling resume accepts incomplete cached runs

**Location:** `scaling.py:617`  
**Severity:** medium  
**Evidence:** cache currentness checks only summary/config existence and config equality; the cached
return reads `scaling_point` fields with `.get()` and sets `error_kind=None` even when required metrics
are missing.  
**Fix:** Validate the summary payload inside `_cell_is_current`; retrain when required metrics are absent
or non-finite.

### 3. refactoring-specialist

#### Active path keeps dead parameters optimizer-grouped

**Location:** `vfe3/train.py:96`  
**Severity:** medium as reported; verifier suggests low/medium  
**Evidence:** active `train_vfe3.py` uses `use_prior_bank=False` and `prior_source="model_channel"`,
while optimizer grouping still adds `pb.mu_embed`, `pb.sigma_log_embed`, and `pb.decode_log_scale`.
Model-channel accessors return `s_*` tables on this path.  
**Fix:** Gate allocation or optimizer grouping by `prior_source` and `use_prior_bank`, or explicitly
freeze/exclude inactive parameters.

#### Transport registry acceptance outruns propagation

**Location:** `vfe3/config.py:750`  
**Severity:** high as reported; verifier suggests medium unless new transports are active work  
**Evidence:** config validation accepts every registered transport from `_TRANSPORTS`, but callers still
route state tensors through hard-coded literal mode checks in `vfe3/inference/e_step.py` and
`vfe3/model/model.py`.  
**Fix:** Move transport capability metadata into the registry and let callers pass state according to
registered requirements.

#### Model channel ignores active transport and RoPE routes

**Location:** `vfe3/model/model.py:616`  
**Severity:** medium  
**Evidence:** the q-channel forwards connection and RoPE arguments, while s/gamma paths force
`transport_mode="flat"` and do not forward those arguments.  
**Fix:** Either label these as q-channel-only routes or thread active transport/RoPE arguments through
the model-channel paths with tests.

#### Decode registry lacks rank and chunk metadata

**Location:** `vfe3/config.py:1327`  
**Severity:** medium  
**Evidence:** decoder names validate through `_DECODERS`, but full-rank checks and fused chunked CE
routing are literal-name tests for `("full", "full_chunked")` and
`("diagonal_chunked", "full_chunked")`.  
**Fix:** Register decoder capabilities such as covariance rank and chunked support, then consume registry
metadata instead of literal name lists.

#### Registered `gauge_fixed` encoder is unreachable

**Location:** `vfe3/model/prior_bank.py:663`  
**Severity:** low  
**Evidence:** `gauge_fixed` is registered, but implementation and config construction both raise
`NotImplementedError`.  
**Fix:** Remove it from the public registry until implemented, or register it with explicit unavailable
metadata.

#### Ablation width sweep leaves derived `kl_max` stale

**Location:** `ablation.py:352`  
**Severity:** medium  
**Evidence:** the `embed_dim` sweep overrides only `embed_dim`; `_cell_cfg_dict()` merges overrides
without recomputing `kl_max`, while other entry points derive `kl_max = 8 * embed_dim`.  
**Fix:** Recompute derived config fields after sweep overrides or add explicit `kl_max: 8*K` overrides
to the `embed_dim` arms.

### 4. performance-engineer

#### Hot-loop CUDA sync from training scalars

**Location:** `vfe3/train.py:326`  
**Severity:** medium  
**Evidence:** loss and CE CUDA tensors are converted to Python floats inside the training step and each
gradient-accumulation microbatch.  
**Fix:** Keep losses as detached tensors during the step and convert to Python scalars only at logging
or return boundaries.

#### Chunked CE synchronizes on rare all-ignore branch

**Location:** `vfe3/model/prior_bank.py:441`  
**Severity:** medium  
**Evidence:** the chunked CE path computes `n_valid = valid.sum()` and branches in Python with
`if n_valid == 0`.  
**Fix:** Use a device-side expression such as `(ce_per_pos * valid).sum() / valid.sum().clamp_min(1)`.

#### Dense decode CE also branches on CUDA tensor

**Location:** `vfe3/model/model.py:797`  
**Severity:** low  
**Evidence:** dense CE branches in Python on `(flat_targets != -100).any()`.  
**Fix:** Compute masked CE fully on device with a clamped valid-token denominator instead of branching
in Python.

#### Full-covariance transport silently upcasts to float64

**Location:** `vfe3/geometry/transport.py:808`  
**Severity:** medium  
**Evidence:** full-covariance transport casts operators/covariance blocks to double and casts results
back to the original dtype.  
**Fix:** Put FP64 stabilization behind an explicit precision toggle, or document the full-covariance
path as an intentional FP64 precision island with tests and user-facing config language.

### 5. python-pro

#### Optimizer closure can double-step gauge parameters

**Location:** `vfe3/gauge_optim.py:188`  
**Severity:** medium  
**Evidence:** gauge parameters are updated and their grads cleared before `super().step(closure)` is
called with the original closure. If the closure repopulates grads, base AdamW can update them again.  
**Fix:** Type the closure explicitly and either reject non-`None` closures or evaluate the closure before
the custom gauge update, then call `super().step(None)`.

#### Generation sampler lacks runtime contract checks

**Location:** `vfe3/model/model.py:1204`  
**Severity:** medium  
**Evidence:** `temperature`, `top_k`, and `top_p` are used directly in division, `topk`, nucleus
filtering, and multinomial sampling.  
**Fix:** Validate `temperature > 0` when not greedy, `1 <= top_k <= vocab_size` when set, and
`0 < top_p <= 1` when set.

#### Figure driver advertises generic modules but requires VFEModel

**Location:** `vfe3/viz/report.py:76`  
**Severity:** low  
**Evidence:** the signature accepts `Optional[torch.nn.Module]`, then dereferences `model.cfg` and
VFE-specific methods such as `attention_maps()` and `diagnostics_per_layer()`.  
**Fix:** Introduce a `Protocol` or use the concrete `VFEModel` type for the `model` parameter and the
extractor functions it calls.

#### Figure registry uses untyped callable contracts

**Location:** `vfe3/viz/figures.py:417`  
**Severity:** low  
**Evidence:** public registry and getters use raw `Callable`; `umap_embed()` has an untyped `features`
parameter and no return annotation.  
**Fix:** Define typed figure callable aliases or Protocols, annotate registered functions fully, and make
`get_figure()` return the narrowed callable type.

## Confirmed Punch List

1. **[high]** Resume checkpoint loading uses unsafe pickle semantics on a configurable resume path,
   `vfe3/run_artifacts.py:272`. Make checkpoint resume use `weights_only=True` where possible, or
   add an explicit trusted-checkpoint opt-in before `weights_only=False`.
2. **[medium, reported high]** Transport registry validation accepts new registered modes while state
   propagation still depends on hard-coded mode-name sets, `vfe3/config.py:750`,
   `vfe3/inference/e_step.py:136`, and `vfe3/model/model.py:36`. Move transport capability metadata
   into the registry before adding new stateful transport modes.

## Cross-Cutting Themes

Two themes dominate the confirmed findings. First, several registry surfaces validate names but do
not carry the metadata that call sites need, so extension still requires editing literal-name gates.
Second, hot paths leak across the host/device boundary through Python scalar conversions and
tensor-boolean branches, which can force CUDA synchronization. The verifier found no contradicted
findings, but several confirmed findings are extension-risk or diagnostics-quality issues rather
than immediate default-path failures.
