# Improvement ideas from six parallel expert reviews (2026-07-05)

Six independent expert agents (variational family/geometry, E-step dynamics, attention/coupling structure, objective/decode/recipe, numerics/compute, literature scout) each inventoried the actual code paths and proposed ideas verified absent from the codebase. Scale knobs (n_steps, n_layers, K, T, N) were excluded by instruction. This document merges, dedupes, and ranks their output. Where multiple agents independently converged on the same idea, that is flagged — convergence across lenses is the strongest signal here. File references come from the agents' reads of the code; spot-check before building.

A recurring structural observation anchors many of the top items: at T=1 with q0 = p (model.py:687), the self-coupling gradient is exactly zero at the start point, so the entire per-token computation beyond encode is a single interpolation of mu toward attention-transported keys, and sigma moves by roughly exp(±e_q_sigma_lr) per step — effectively frozen. Several ideas below attack that under-convergence without raising T.

## Tier 1 — convergent, high expected impact

### 1. Closed-form MM/coordinate-exact E-step for (mu, sigma) at frozen beta
Independently derived by three agents (VI, dynamics, numerics). At fixed beta, transported keys, and alpha, the per-token F is convex in the diagonal Gaussian coordinates with an exact stationary point: precisions add, means fuse precision-weighted,

    Lambda_i* = (alpha_i Lambda_p + lambda_beta sum_j beta_ij Lambda_t,ij) / (alpha_i + lambda_beta),
    mu_i*     = Lambda_i*^{-1} (alpha_i Lambda_p mu_p + lambda_beta sum_j beta_ij Lambda_t,ij mu_t,ij),

with the sigma component being the harmonic barycenter one gets by zeroing the kernel's sigma gradient (kernels.py:137-142). Because the surrogate sum_j beta0 KL majorizes the canonical F (the log-partition envelope), this is a monotone MM step: exact block-coordinate minimization in one shot, at the same O(N^2 K) cost as the current single gradient step. It replaces the global lr*lambda interpolation coefficient with the F-optimal per-token, per-coordinate prior-vs-attention weighting, and it delivers a real per-token sigma* in one iteration — currently the KL decode's precision channel is dead capacity because sigma never leaves sigma_init. Damped variant theta <- (1-eta) theta + eta theta* is mirror descent in natural coordinates and subsumes line search. Pure (parameter-free, same canonical F), differentiable in the tables, gauge status identical to the existing kernel. Slot: new `e_step_update` registry key ('gradient' default, 'mm_exact') dispatched in e_step_iteration, sibling to e_step_mu_precond. Cheapest test: kernel route only (gaussian_diagonal/flat/KL), 2k steps at T=1, watch val CE and the estep_f_drop diagnostic. Verified absent: e_step.py:506-571 is gradient-step only; grep fixed_point/barycenter matches only the hyper-prior centroid.

### 2. Unigram log-prior in the decode (Bayes-correct class prior)
Convergent (VI + objective agents). The live decode is a uniform-categorical-prior Bayes rule: logits = -KL(q_i||p_v)/tau (or mu @ W^T + zero-init bias). The Zipfian marginal must therefore be encoded geometrically in the tables, burning belief-space capacity on frequency instead of semantics — expensive at K=100 and about one epoch of data. Fix: logits_iv = kappa log pi_v - KL(q_i||p_v)/tau_eff with pi_v the corpus unigram frequency (fixed data statistic, kappa=1 pure); on the linear path, init decode_bias to log pi_v instead of zero. This is the exact Bayesian posterior over the vocabulary and supplies the unigram CE component for free. The cheapest experiment in this document: swap the decode on an existing checkpoint and re-eval — no retraining needed to see the effect. Slot: register_decode("diagonal_unigram") in prior_bank.py; counts from data/datasets.py. Verified absent: no additive prior in any registered decode; decode_bias exists only on the linear path and is zero-init (prior_bank.py:188-190).

### 3. Compute-reclamation package: 2-4x step time, reallocate to T=2 or more steps
The numerics agent's shape-derived FLOP skeleton puts the factored pair transport einsum at roughly two orders of magnitude above everything else (~130 TFLOP/step with backward and both channels), so decode-side sampling ideas are irrelevant (<1% of step). Four stacked fixes, all exactness-preserving:

(a) Per-head transport_mean. exp_phi is stored dense (B,N,K,K) with exactly-zero off-blocks, yet the mean path (transport.py:996) contracts the full K axis — H times the necessary work. The covariance path already slices per head (transport.py:1078-1120); mirror that loop. ~4x cut of the dominant GEMM at H=4. Probe: time full-K vs per-head einsum at live shapes, allclose 1e-6.

(b) fp64 matrix-exp island fires permanently at d_head=25. stable_matrix_exp_pair upcasts to float64 when d_eff >= 20 (transport.py:794-795, 962) — tuned when d_head was 10. The conditioning argument is a norm argument, not a dimension argument; at the observed block norms (~1.5 under the clamp) fp32 matrix_exp is accurate to ~1e-7 while the 5090 runs fp64 at 1/64 rate. Key the island on the clamped Frobenius norm (or a config threshold) instead. Probe: measure max block norm on a checkpoint, compare fp32-vs-fp64 exp error, time both.

(c) Share the FactoredTransport between _refine_s and the belief E-step — both build the identical flat transport from the same phi per forward (e_step.py:662-669; model.py:670-696). Hoist once, thread as _prebuilt_omega. Exact.

(d) Regional torch.compile on the pair chain (kernels.py:127-142 pointwise chain over (B,N,N,K) intermediates; static shapes; keep matrix_exp/eigh outside). No compile usage exists anywhere. Expect 1.5-2x on the non-GEMM residual. Also: bf16 autocast for the E-step is wired with fp32 islands but never validated for training quality (tests are wiring/finiteness only) — the pair stage is bandwidth-bound, so a seed-matched 2k-step A/B of CE-vs-wallclock is worth one run.

The point of all four is the user's own framing: better loss at equal wallclock by buying T=2, larger N, or more optimizer steps with freed compute.

### 4. Expected-likelihood (convolution) decode kernel
Convergent (VI + objective agents, identical math). Replace the KL readout with the exact Gaussian predictive: logit_iv = log integral q_i(x) N(x; mu_v, sigma_v) dx = -1/2 sum_k [ (mu_q-mu_v)^2/(sigma_q+sigma_v) + log(sigma_q+sigma_v) ]. Variances add rather than ratio, so the readout is robust to sigma_q miscalibration, mass-covering over the vocabulary, closer to the manuscript's -E_q[log p(o|x)] than the KL surrogate (which penalizes small-sigma_v banks through the trace term even at perfect mean match), and it delivers sigma-adaptive per-position flattening automatically through the log-det. Note the naive alternative E_q[log p_v] is softmax-identical to the current decode (differs by the v-independent H(q)), so this is the only genuinely different sigma-aware readout nearby. Cost caveat: the (sigma_q+sigma_v) coupling breaks the 2K-fused-matmul separability, needing a chunked broadcast (~1.7 GB transient at Vc=512 — runnable). Pure. Slot: register_decode("expected_likelihood_chunked") + fused CE twin. Eval-only swap on an existing checkpoint is the first test. Verified absent: decode registry is diagonal/full/linear (+chunked) only, all hardcoding alpha=1 KL.

### 5. Randomized-T training with truncated backprop and KL-based halting
Literature agent, high confidence (arXiv:2502.05171 Huginn-3.5B; arXiv:2502.17416; Ouro 2025 — replicated across three groups). Train with T sampled per batch from a log-normal Poisson, backprop only through the last 1-2 iterations (the detach_e_step machinery is the hook), then at eval run T large with a native halting rule — KL(q_i^t || q_i^{t-1}) is closed-form for Gaussians. Randomized-T training is what buys path independence so that extra eval iterations help rather than diverge. This is the principled version of "more E-steps": the training cost stays near T~1-2 in expectation while eval-time compute becomes elastic. Fully pure — only more F-minimization. Pairs with idea 1 (each iteration then does maximal work) and idea 3 (freed compute funds the expected-T). The dynamics agent's Anderson/extragradient accelerators and per-token top-p refinement (score rows by the whitened step norm already computed as delta_mu, re-update only the top quartile against frozen keys — the partial-row machinery exists in belief_cache._appended_belief_step) become live once T >= 2.

## Tier 2 — strong single-lens ideas, cheap to test

### 6. Make the gamma model channel predictive: gamma as the prior for beta
Attention agent's top idea. Currently lambda_gamma=0.75 shapes s-tables that never touch logits (model.py:940, "predictively INERT"). Set pi_ij := softmax_j(log pi_s_ij - E_s_ij/tau_g), i.e. the model-channel posterior becomes the belief channel's attention prior: beta = softmax(log gamma - E/tau). This is exactly the manuscript hierarchy h -> s -> p -> q acting on attention — models tell beliefs where to attend; beliefs refine within that prior. Detached variant keeps the kernel exact (same treatment as the existing precision bias); zero new parameters; gauge-invariant per edge. Slot: a _fold_gamma_prior sibling of _fold_precision_bias in model.forward, reusing _gamma_energy. One ~10-line fold plus one training run.

### 7. Two-hop coupling via the exact cocycle (depth without layers)
Attention agent. Add F_2 = lambda_2 sum_ik (beta beta)_ik KL(q_i || Omega_ik q_k). In Regime I the composed transport is exact and free: Omega_ij Omega_jk = Omega_ik, so no new operators are built — only the weight matrix beta@beta (per head, causal-closed, O(H N^2) extra). At L=1 this buys effective depth 2, the induction-head composition standard transformers need two layers for. Fully pure and exactly equivariant; lambda_2=0 is the current path. First cut with detached hop weights (Danskin covers the E term, not the weight dependence — detach is the honest start). Slot: optional term in free_energy.free_energy plus one extra masked _pair_contract.

### 8. Per-query adaptive temperature tau_i from belief precision
Convergent (VI + attention agents). precision_weighted_attention is key-side only; the query-side complement is absent (attention_tau depends only on kappa and block dims, free_energy.py:41-76). Set tau_i = kappa sqrt(d) f(tr Sigma_i) (detached): uncertain queries hedge across keys, confident queries commit. Softmax remains the row-stationary point of F with tau_i in its own entropy term; same detached-state-function footprint (including the same tr-Sigma gauge caveat under full GL) as the existing key bias. Requires extending _broadcast_tau to a row shape. Eval-only toggle on a trained checkpoint is a zero-training first read.

### 9. Decode-side hygiene trio (near-free, objective agent)
(a) Weight decay currently pulls sigma_log_embed toward 0 — an unintended lognormal prior pinning bank sigma to 1 while sigma_init=3 and the E-step barely moves sigma; the optimizer silently fights the configured prior on the pure decode path. Zero the decay on the sigma role group or reparameterize as offset from log(sigma_init). One group-dict change. (b) z-loss: loss += lambda_z mean(logsumexp_v logits)^2 (lambda_z ~ 1e-4). The free-energy reading is that a normalized log Z makes the decode a calibrated observation model, which -E_q[log p(o|x)] formally requires; logsumexp is already materialized in all three fused CE paths (prior_bank.py:461, 581, 643), so it is one extra term with zero extra memory. (c) Untied decode bank: clone decode tables from encode at init and register decode_mode='diagonal_untied'. This isolates the tied-vs-untied confound in the standing linear-beats-bank result (the linear ablation unties AND changes geometry simultaneously). Three arms at 2k steps: tied-KL, untied-KL, linear.

### 10. Self-edge attention sink probe
Numerics agent. Omega_ii = I exactly, so E_ii = KL(q_i||q_i) ~ 0 permanently, and causal_alibi gives the j=i slot zero bias: the self-pair holds the row's maximal logit through all of training while contributing an identically-zero pull. The codebase already masks the self-edge as degenerate in regime_ii (transport.py:352-359). Probe first (10 lines: mean diagonal beta mass per row on a checkpoint); if the mass is far above uniform share, register a causal_noself prior and A/B. Whether excluding i=j from the coupling sum is "pure" depends on the manuscript's sum convention — check before claiming purity.

### 11. Grad-clip bind-rate check (the kl_max bug class, again)
clip_grad_norm_(all params, 1.0) is global over a vector dominated by phi_embed (~80% of ~150M params). If the total norm sits persistently above 1, the clip binds every step and becomes a silent global LR rescale that couples the mu/sigma/decode effective LRs to phi-table gradient noise — the same silent-100%-bind pattern kl_max had. Zero-cost check: read the already-logged grad_norm columns from an existing metrics.csv. If binding, clip per role (roles are already tagged in build_optimizer). The same agent proposes a standing clamp-telemetry monitor (bind fractions for every clamp site to metrics.csv each eval, ~40 lines via numerics.register_monitor) — converts this bug class from "found by audit" to "read from CSV."

### 12. Dead sigma compute under the live linear-decode config
With use_prior_bank=False, norms off, T=1, L=1, mstep_self_coupling_weight=0, the numerics agent traces the belief-channel sigma update (one of the two big pair contractions plus retraction, ~15-25% of belief-block cost) into a tensor nothing consumes — and e_q_sigma_lr is then a phantom knob (only e_s_sigma_lr is live). Verify with the free probe: 200 steps at e_q_sigma_lr=0 vs 0.001 should be bit-identical; if so, add a config-conditional skip. Pure path (bank decode) untouched.

## Tier 3 — worthwhile, more effort or more speculative

Divergence-role split. Score beta with a bounded mass-covering divergence (Bhattacharyya / Renyi-1/2) while the coupling the beliefs descend stays KL — the coupling_energy seam for exactly this decoupling already exists with only the RoPE consumer (free_energy.py:340, 381-387). Convergent with the VI agent's per-term asymmetry (self vs coupling divergences; both self_divergence and pairwise_energy already accept independent divergence arguments — only the callers pass one shared key). Caveat: mixed orders leave the closed-form kernel route; accept the oracle cost or write one mixed kernel. Related cheap flyer: Renyi-order annealing 0.5 -> 1.0 over the first ~20% of training (mass-covering through the random-prior transient, canonical KL at convergence).

Alternative beta stationarity conditions, all pure and seam-shareable behind one beta_normalization registry: entmax/sparsemax (Tsallis entropy term; the closed-form kernel remains exact by Danskin with sparse beta plugged in — kills mandatory softmax leakage over all causal keys), Sinkhorn/doubly-stochastic (column marginals from the prior's key load; the repo already measures the rank-collapse metric this targets, metrics.py:303; modest evidence, nearly free), and stick-breaking attention (arXiv:2410.17980, ICLR 2025: better wikitext ppl at ~1B, no positional encodings needed — corresponds to a sequential stick-breaking prior over each beta row and could subsume the T5 channel).

Empirical-Bayes attention recursion. At E-step iteration t, mix last iteration's beta into the prior: B^t = log((1-w) softmax(B^0) + w beta^{t-1}.detach()) — posterior-becomes-prior for attention, the same move the stack makes for mu_p. Live only at T>=2; cheapest content-adaptive pi available.

Learnable per-head kappa. tau's Vaswani calibration is a dot-product-statistics argument that does not obviously transfer to KL energies whose scale is set by sigma_init and guards; an (H,) kappa parameter (init 1.0, step-0 identical, t5-exception family, no gauge contact) lets heads pick sharp-copy vs diffuse-context regimes. One optimizer group.

Mixture/hierarchical readout against the rank bottleneck. The literature agent flags K=100 against a 267k-type vocabulary as a classic softmax bottleneck (arXiv:1711.03953; MFS ACL 2022) and the objective agent independently proposes the two-level bank (C ~ sqrt(V) cluster Gaussians via the existing barycenter machinery, then members): shared structure for rare tokens plus ~100x fewer decode KLs per position. Multi-prototype bank entries (min/logsumexp-KL over M Gaussians per vocab item) are within the stated mixture-family mandate. Highest-effort decode idea; frozen-random-clusters speed test first.

Frequency-aware table treatment. Beyond the unigram bias (Tier 1): init bank sigma rows rarer = wider (sigma_log = log sigma_init + c(-log p_v)); mask AdamW decay/ghost-momentum updates to rows absent from the batch (dense AdamW actively shrinks rare rows between occurrences at ~1 epoch); GGN/Sophia-style diagonal curvature for the vocab tables (row curvature spans orders of magnitude with frequency; Adam's sqrt(v) conflates noise with curvature).

M-step variants with a VFE reading. IVON drop-in for AdamW (ICML 2024, GPT-2-scale parity at equal runtime; its per-parameter posterior variance is a principled initializer/regularizer for bank Sigmas — ties M-step uncertainty to the KL-decode geometry). iPC incremental EM (arXiv:2212.00720): small table updates at every inner E-step iteration once T>1. Prospective-configuration flavor (Nature Neurosci 2024): closed-form moment-matching of bank entries toward settled beliefs as a supplement to backprop-through-inference. Setting mstep_self_coupling_weight to the alpha envelope makes E and M descend one functional (the knob exists, apparently never swept >0) — arguably the purer path. EMA-teacher self-distillation (decay ~0.999) is the recipe-level cousin: the teacher is an averaged realization of the slow model channel; needs a chunked KL(p_T||p_S) since logits are never materialized.

Family/geometry upgrades. Low-rank-plus-diagonal Gaussians (Sigma = D + UU^T, rank 2-4, Woodbury KL at O(Kr^2); the U-factor transports exactly, so it keeps the correlations that the diagonal projection currently discards every step — strictly better transport fidelity than the current family at far below gaussian_full's O(K^3)). Bures-Wasserstein SPD retraction (registry sibling to spd_affine/log_euclidean; the W2 metric stays finite as sigma -> 0, unsticking the collapsed-coordinate freeze that Fisher geometry causes — one config-line A/B). Mixture-of-centroids hyper-prior ({r_m} with softmax responsibilities, closed-form per-cluster barycenters — adaptive James-Stein by token class instead of one global shrinkage target dominated by frequent tokens). Mixed per-head gauge tower (product group: GL heads for scaling/gating, SO/Sp heads for bounded rotation-like transport; registry-only except the per-block skew_symmetric flag). LieRE-style learned positional generator (exp((i-j)A) composed into the transport, A tied in the algebra — a strict RoPE generalization living inside the connection; t5-exception family, equivariant in the tied-tower gauges; arXiv:2406.10322).

Test-time compute on F. The hierarchy already licenses TTT/Titans-style adaptation purely: a few natural-gradient VFE steps on a context-local copy of s (or h) during evaluation is online empirical Bayes, no architecture change (arXiv:2407.04620 for the mechanism class). Similarly EBT-style decode-time descent (arXiv:2507.02092): a few mu descent steps including the readout term before scoring, or min-F candidate selection. Both pure, both elastic eval-compute.

Equilibrium solvers, for when T grows. Fixed-beta CG solve of the SPD mean-field system (matvec = two beta-weighted transport_mean applications; causal filtering even admits an exact forward-substitution scan) delivers the multi-hop equilibrium at fixed beta — the DEQ move; pairs with an implicit-function-theorem gradient estimator as a fourth e_step_gradient value (adjoint solve with the same operator, O(1) memory, tested against the autograd-of-F oracle). Extragradient beta-refresh (predictor mu half-step, recompute beta at the same hoisted Omega, corrector) fixes the fact that at T=1 attention never sees the query it shaped, at far less than a second full iteration. Within-loop tau annealing (diffuse early iterations, sharp last) rides along. Selective-attention redundancy masking of pi (arXiv:2410.02703, parameter-free) and a windowed+global-sink prior are low-priority at N=128 but become relevant as N grows. When layers stack, read muPC first (arXiv:2505.13124) — depth-wise rescaling rules that decide trainability for deep PC-style nets.
