## Sigma-validation gate result recorded (FAIL)

Recorded the v1 operating-point sigma-gate outcome in the pre-registration doc
(`docs/research/active-inference/2026-06-28-sigma-gate-prereg.md`, new "Result (2026-06-29)" section).
The gate was run on a prior-bank (KL-to-prior) decode checkpoint (`embed_dim=20`, 15k steps,
`use_head_mixer=False`, `use_prior_bank=True`) — the decode path on which the belief covariance enters
the readout. Outcome: FAIL. The Spearman correlation between `tr(Sigma_q)` and realized cross-entropy is
`-0.137` (CI `[-0.147, -0.127]`, permutation floor `0.008`): significant but with the wrong sign, so
higher belief uncertainty predicts lower surprise. The trace barely varies (`cv = 0.044`) and the
sigma-stratified cross-entropy trends downward (`mono_spearman = -0.43`). Sigma-binned ECE is `0.019`
(below `0.05`), so the model stays calibrated within strata, but the gate as a whole fails. Artifact:
`vfe3_policy_results/sigma_gate/wikitext103_ed20_15k.json`, bound to spec commit `c05b927`.

Consequence (per pre-registration and spec Section 4.5): the information-gain term stays inert; the
`sigma_mc`, epistemic-only, and shuffled-sigma arms remain reported-only; `policy_sigma_ambiguity_validated`
cannot be set (Guard 4 refuses this FAIL artifact). The EFE policy scorer is a validated pragmatic
preference reranker (risk + likelihood-entropy ambiguity), not an exploratory agent with a live epistemic
value. No code changes; documentation only.

## Observation-term no-leakage investigation recorded

Copied `docs/research/active-inference/2026-06-29-observation-term-e-step-no-leakage-investigation.md`
into this repo's `docs` tree from the clean investigation worktree. The report records a five-agent
read-only investigation into how an observation term could enter the E-step without target leakage.
It verifies that the current executable path is target-blind: `forward()` produces beliefs from
`token_ids` before `targets` enter CE, and the live E-step gradient has no target or likelihood
payload. Main conclusion: `targets[:, i] = token_ids[:, i + 1]` must never be used to update `q_i`;
once `x_{i+1}` arrives, it can become evidence for the next belief state, not retroactive evidence for
the belief scored against it. A safe implementation would derive an optional prefix/current-token
observation payload inside `forward_beliefs()`, thread it through the E-step, add raw observation
gradients before natural-gradient preconditioning, and mirror the same term in the scalar diagnostic
and oracle paths.

## Phase 3a: belief-prefix cache + efe_rollout (H>1) unlocked

Built the belief/key-value cache the EFE horizon rollout was gated on (spec Section 3.5), unlocking the
H>1 `efe_rollout` scorer. Branch `feat/efe-belief-cache-phase3a`.

New `vfe3/inference/belief_cache.py` (`cache_supported`, `rollout_predictive_cached`). Under a causal
attention prior the converged beliefs at context positions 0..N-1 are invariant to any appended token
(the future is masked to exactly zero weight) — the causal-invariance linchpin verified by the
cacheability audit `wf_a12bc02f-988` (both adversarial verifiers returned claim_holds with the
validity conditions below). So the cache computes only the APPENDED positions' E-step rows against the
shared context: the context-context attention block is built once, not Kp times (the scorer reads only
the last position, so that block was previously recomputed and discarded per candidate). It reuses the
live E-step primitives (`transport_mean`/`transport_covariance`, `pairwise_energy`,
`attention_weights`, the registered belief kernel, the family Fisher natural gradient, the SPD
retraction), so it reproduces the full per-candidate recompute to within float tolerance (NOT
byte-identity: a partial recompute reduces over a different key-axis length and float addition is
non-associative).

Scope guard `cache_supported(cfg)`: the fast path is golden-verified exact only for `n_layers=1`,
`n_e_steps=1`, the closed-form filtering kernel (`gaussian_diagonal` + KL + `include_attention_entropy`),
flat transport, a causal belief prior, frozen gauge (`e_phi_lr=0`), and none of model-channel
(`s_e_step`) / precision-bias fold / gauge-RoPE / head mixer / CG coupling / mean trust region / raw
mean preconditioner. Any other config falls back to the full recompute in `policy.py` (correct, slower).

`vfe3/inference/policy.py`: `_rollout_predictive` takes the cache fast path when `cache_supported` and
`N+L <= max_seq_len` (no sliding-window eviction, which would invalidate the prefix). Extracted the
shared `_efe_score` body; replaced the `efe_rollout` hard-error stub with a real H>1 scorer (a policy is
the H-action candidate sequence; q(o|pi) is read from the LAST rolled-out position, the environment
response never folded in). `efe_rollout` REQUIRES the cache to be active: on an unsupported config it
raises rather than silently paying the dishonest Kp*H full recompute. `vfe3/config.py`: added the
symmetric guard that `policy_mode='efe_rollout'` requires `policy_horizon>1`.

Tests: new `tests/test_belief_cache.py` — golden `cached == full` to float tolerance across decode path
(linear / KL-to-prior) x H in {1,3} x heads in {1,2}, the `cache_supported` predicate, and `efe_rollout`
end-to-end + guards. Updated `tests/test_efe_scorer.py` and `tests/test_policy_registry.py` to the new
`efe_rollout` contract (was: accepted-at-config / hard-error-at-dispatch; now: functional H>1 scorer
requiring `horizon>1` and the cache). Full suite 1347 passed, 1 skipped, 1 xpassed, 0 failures.

## Masked key-value retrieval task (H>=2 epistemic): foundation only, experiment NOT run

Scoping the matched-compute beam / best-of-N arms surfaced that they are inherently H>=2 (at H=1 they
reduce to the existing logprob / sampling arms) and that the ring cannot host them (single-transition
training makes a multi-action lookahead out-of-distribution; and goal-blind decoders collapse to chance
because the ring action token is trained independently of the goal). Their non-degenerate home is the
spec's deferred masked key-value retrieval task. Wrote the design note
`docs/research/active-inference/2026-06-29-masked-retrieval-task-design.md` and the task ENV / data /
training foundation `vfe3/inference/masked_retrieval.py` (vocab, table-with-masked-target render,
PROBE/ANSWER compound actions, random-action teacher-forcing so the model learns the probe->reveal and
answer-the-revealed mechanics but not the probe-choice policy, predictive-adequacy, train_checkpoint).
Sanity-checked (token ranges disjoint, target masked in context, teacher-forcing shift correct).

The closed-loop runner and the arms (efe_rollout + matched-compute beam / best-of-N) were NOT built and
the experiment was NOT run, by decision: the predicted outcome is a null result. Even with a goal
preference, efe_rollout scores every probe equally because in scoring there is no environment reveal
(spec 2.2), so P(correct) is flat across probes and only the epistemic information-gain term could break
the tie -- and that term is identically zero (sigma-gate FAILED, point belief; I == 0 even at H>=2
without marginalized belief spread) and structurally cannot carry contextual probed-vs-unprobed
uncertainty (no data-precision channel in grad_sigma; sigma anchored to a per-vocabulary prior table).
The foundation is committed as WIP for a future continuation if the epistemic term is ever made live
(the data-precision lever from the why-sigma-collapses investigation). This module is standalone (not
imported anywhere), so it does not affect the build or the test suite.

## Dense Regime II direct-link specification

Wrote `docs/research/2026-06-29-regime-ii-direct-link-spec.md` on branch
`docs/regime-ii-link-spec-20260629`, from a fresh `origin/main` worktree. The
spec uses four specialist read-only lenses: gauge theory, variational
free-energy structure, implementation wiring, and numerical analysis. It
incorporates the binding design clarification that Regime II should behave like
Regime I at the attention-graph level: set the configured sequence length and
construct all pairwise transports, with the existing attention prior deciding
the row weights. The chosen v1 target is therefore dense all-pairs direct link
variables `L_ij`, not a sparse or windowed connection.

Main decision: the canonical Regime II object is a group-valued direct link
`L_ij` over every ordered position pair, acting on Gaussian beliefs by
`mu_j -> L_ij mu_j` and `Sigma_j -> L_ij Sigma_j L_ij^T`, with the existing
canonical KL-plus-attention-entropy variational block preserved. The
vertex-frame sandwich `exp(phi_i) exp(delta_ij) exp(-phi_j)` is documented as a
possible chart, not the ontology. The spec rejects the raw bilinear `regime_ii`
route as a principled target because its gauge break at nonzero `connection_W`
is already pinned by tests, keeps `regime_ii_covariant` as a useful covariant
and amortized reference, and defines the future `transport_mode="regime_ii_link"`
contract, expected config/model/optimizer/reporting seams, dense all-pairs
performance rule, and acceptance tests. Documentation only; no production
source changed.
