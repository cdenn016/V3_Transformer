## Sigma-validation gate result recorded (FAIL)

Recorded the v1 operating-point sigma-gate outcome in the pre-registration doc
(`docs/research/active-inference/2026-06-28-sigma-gate-prereg.md`, new "Result (2026-06-29)" section).
The gate was run on a prior-bank (KL-to-prior) decode checkpoint (`embed_dim=20`, 15k steps,
`use_head_mixer=False`, `use_prior_bank=True`) — the decode path on which the belief covariance enters
the readout. Outcome: FAIL. The Spearman correlation between `tr(Sigma_q)` and realized cross-entropy is
`-0.137` (CI `[-0.147, -0.127]`, permutation floor `0.008`): significant but with the wrong sign, so
higher belief uncertainty predicts lower surprise. The trace barely varies (`cv = 0.044`) and the
sigma-stratified cross-entropy trends downward (`mono_spearman = -0.43`). Sigma-binned ECE is `0.019`
(below `0.05`), so the model stays calibrated within strata, but the gate as a whole fails. Artifact:
`vfe3_policy_results/sigma_gate/wikitext103_ed20_15k.json`, bound to spec commit `c05b927`.

Consequence (per pre-registration and spec Section 4.5): the information-gain term stays inert; the
`sigma_mc`, epistemic-only, and shuffled-sigma arms remain reported-only; `policy_sigma_ambiguity_validated`
cannot be set (Guard 4 refuses this FAIL artifact). The EFE policy scorer is a validated pragmatic
preference reranker (risk + likelihood-entropy ambiguity), not an exploratory agent with a live epistemic
value. No code changes; documentation only.
