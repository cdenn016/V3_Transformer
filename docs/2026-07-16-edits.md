## Head-mixer evaluation and driver import correction

The July 16 head-mixer identity shortcut incorrectly assumed that every
optimizer write changes `Tensor._version`. CUDA fused AdamW can update the
mixer delta while leaving that counter unchanged. Training therefore learned
the mixer while live no-grad validation continued returning the unmixed input.
Checkpoint loading already disabled the shortcut, which explains why held-out
test evaluation after reload was internally correct but evaluated the
checkpoint selected by corrupted live validation. Replaying the stale
shortcut reproduced the reported step-4,500 validation PPL of 376.249 and
final validation PPL of 879.1. Fresh loading of the final checkpoint produced
validation PPL 141.01 and held-out test PPL 139.95.

`HeadMixer.forward` now invalidates the initialization-only shortcut on its
first grad-enabled call. Untouched models retain the synchronization-free
object-identity passthrough before training, while all later evaluations use
the learned mixer independently of optimizer backend or tensor version-counter
behavior. A regression simulates a versionless update and proved the old code
returned the original input object before passing with the correction. The
click-to-run driver also now imports `Optional`, which is required by the
`_validated_data_seed` return annotation added in the final July 16 audit
remediation commit.
