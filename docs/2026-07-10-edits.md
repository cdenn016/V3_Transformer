# Edits — 2026-07-10

- Added a shared runtime-state seam for reversible deterministic execution. The three click-to-run entry points now seed through `vfe3.runtime.seed_everything` with the effective `cfg.deterministic` value, and provenance records the effective PyTorch, cuDNN, and cuBLAS state. Single-run launches now reject disagreement between a nonempty `SEEDS` value and the config seed instead of silently choosing one source.
- Closed the S1 formal-review gaps by routing the documented in-process `vfe3.train.run_training` entry through the same runtime seam. Determinism and run-naming tests now own their expected config values, cuBLAS restoration is pinned for both absent and preexisting environments, and the finalize regression requires the exact effective deterministic-state provenance payload.
- Corrected the S2 baseline tests without changing production defaults. The Phase-0 golden fixture now explicitly owns seed 0, preserving its frozen numeric checks independently of the click-config seed, and the run-artifact test now pins the production checkpoint interval at 25000.
- Restored exact auxiliary continuation state for checkpoint resume. Checkpoints now retain the private Metropolis generator and the gauge optimizer's omega reorthogonalization cadence; resume keeps current optimizer-group metadata while preserving loaded parameter objects, accepts legacy optimizer state with an explicit non-exact-resume warning, and preserves `FileNotFoundError` for a missing checkpoint path.
- Made shuffled checkpoint resume batch-exact. Checkpoints now retain a cloned epoch-start loader-generator state, the epoch, and the consumed-batch cursor; resume recreates that epoch's permutation and skips the exact cursor across epoch transitions, while a shuffled loader without an exposed generator fails explicitly instead of approximating the stream.
- Hardened shuffled-cursor persistence against malformed numeric fields. Checkpoint save and load now require `batches_consumed` and `epoch` to be nonnegative Python integers, explicitly reject booleans and coercible fractional values, and pass validated cursors into training without a second conversion.
- Bounded cached-token loading and made the cache/model contract explicit. Limited `.pt` reads now mmap and clone only the requested slice, `.bin` reads slice before materialization, banner counts use cache metadata, dataset/split/run-label components reject unsafe names, tokenizer vocabulary bounds are checked before model indexing, and every production loader receives the effective `VFE3Config.vocab_size` without changing any click-run configuration value or toggle.
- Bound run and scaling-cache reuse to code and data identity. Finalization now records SHA-256 and token count for the train, validation, and test streams together with the effective data seed, train-token cap, and tokenizer tag; Git probes use a resolved executable, constrained noninteractive environment, five-second timeout, and persisted errors. Scaling resume now requires matching HEAD identity and rejects dirty reuse unless both saved and current dirty-tree fingerprints are identical.
- Closed the S6 formal-review fail-closed gap for structurally malformed cache artifacts. Parseable JSON scalars and arrays at the top level of `scaling_cell.json` or `provenance.json` now make scaling resume rerun the cell instead of raising from mapping access; the approved SHA and dirty-fingerprint comparison is unchanged.
- Corrected analysis identity at corpus, tokenizer, and multiseed boundaries. Finalization now builds one training-corpus unigram table and uses it only for frequency-bucket membership while retaining sampled held-out cross-entropy values under `corpus_freq_strata_ce`; vocabulary comparison rejects mixed tokenizer tags before loading models or plotting; and every multiseed aggregation path rejects missing, unreadable, or semantically mixed configs using deterministic seed-normalized fingerprints that preserve every non-seed setting. The required regressions were authored, but all test, lint, compile, and runtime execution remains deferred to the consolidated validation matrix requested after the correctness and performance series.
- Corrected the S7 frequency-cutoff semantics after formal source review. Rare/middle/frequent thresholds now come from the positive-count token types across the complete training corpus, each vocabulary type contributes once regardless of held-out duplication, and evaluation targets absent from training are explicitly rare; sampled evaluation data now affect only the cross-entropy values aggregated within those fixed corpus buckets. An imbalanced-evaluation regression was authored but not executed, and validation remains deferred to the consolidated matrix.
- Hardened artifact publication and ablation resume state. Sigma-gate JSON now writes through a same-directory temporary file and publishes with `os.replace`, removing the temporary file after serialization or publication failures; `RunArtifacts.save_json` now accepts only regular bare filenames. Ablation markers now record both requested collection flags, `error_kind`, and an explicit `success` or `failed` terminal state, while resume requires a successful finite final validation PPL and any requested diagnostic or extrapolation output. The required regressions were authored but not executed, and all pytest, lint, compile, import, and runtime validation remains deferred to the consolidated matrix.
