## Active Inference Build-Out: Phase 1 (one-step EFE reranker + ring task)

The Phase 1 CODE is complete and unit-tested; the compute-bearing experiment (3-seed checkpoints +
5000-episode arms) is shipped as a click-to-run, GPU-aware script because the iterative E-step is too
slow to train on CPU in this environment (no CUDA here).

`vfe3/inference/policy.py`: the one-step EFE scorer. `PolicyScore` (the return tuple keeping score and
each diagnostic separate); preference builders `flat` / `task` (peaked) / `held_out_predictive`;
ambiguity estimators `likelihood_entropy` (default) and gated `sigma_mc`; scorers `efe_one_step`,
`logprob_control` (matched-compute control), and `efe_rollout` (hard-errors; gated on a cache). The
headline honesty invariant holds by construction: at horizon=1 over the point belief the MI bridge
`epistemic = predictive_entropy - ambiguity` is identically 0, and `G = risk + ambiguity` collapses to
the pragmatic cross-entropy `-E_q[log p(o|C)]`. Pinned by `tests/test_efe_scorer.py` (12 tests:
algebra, epistemic==0 exactly, the cross-entropy collapse, flat->log V constant, posterior formula,
gating raises).

`vfe3/model/model.py`: the `generate()` guarded branch + `_policy_select`. `policy_mode=='none'` runs
the verbatim pre-existing body (byte-identical, re-indented under the guard); a non-default mode routes
through `_policy_select`, which builds the top-`policy_top_k` candidate menu (generator E, candidate
prior = base softmax over the menu) and selects the policy-posterior argmax.

`vfe3/inference/ring_task.py` (new): the controlled closed-loop ring goal-steering environment
(V=32, m=16, deterministic transition), the rendered-transition training data, the predictive-adequacy
gate, `train_ring_checkpoint` (GPU-aware), and a batched closed-loop `run_episodes`. Pinned by
`tests/test_ring_task.py` (6 tests). Correction to the spec's literal preference wording: the task
preference is instantiated DISTANCE-GRADED, `U_C(o) = -ring_distance(o, g)` (sealed `beta_C=5.0`), not a
pure peak on the goal symbol -- a pure peak carries no one-step gradient at ring-distance>1 (both
neighboring actions land on equal-mass non-goal states), so greedy H=1 could not steer; the spec's own
solvability argument requires the graded form. Documented in `ring_preference`.

`efe_ring_experiment.py` (new, root, click-to-run): the pre-registered Section 4 run. Trains the
three-seed set (6/23/64, 15k steps), applies the predictive-adequacy precondition (>=0.98), dev-tunes
gamma over {0.5,1,2,4,8}, runs the v1-live arm matrix (full EFE, gamma=1 sensitivity, risk-only,
ambiguity-only, flat [reported], p_data control, logprob baseline, random) on a paired test set, and
applies the conjunctive go/no-go gate with paired McNemar (Holm-Bonferroni over the primary gates) plus
bootstrap CIs. CONFIG carries the sealed constants; any reduced smoke budget is logged (no silent caps).
Orchestration validated end-to-end on a tiny budget; the sealed run must execute on the GPU.

Full suite: 1303 passed, 1 skipped, 1 xpassed, 0 failed (+18 new). Phase 1 go/no-go and the demotion

## Active-Inference Phase 3 start: sigma-validation gate measurement

Phase 3's epistemic payoff (sigma_mc, epistemic-only, the masked-retrieval task) is gated on the
sigma-validation gate, because the MI-bridge information gain is identically zero at any horizon with a
sigma-free point belief. The gate is the cheap decisive prerequisite, so it was built and run first
(user's choice). A three-reviewer adversarial pass found the statistics correct and faithful to spec
Section 4.5; one medium finding (Guard-4 content check enforced nowhere) was fixed.

- `vfe3/inference/sigma_gate.py` (new): the measurement -- `spearman_bootstrap_ci`, `permutation_floor`,
  `sigma_stratified_ce` (monotonicity), `sigma_binned_ece`, the conjunctive `evaluate_sigma_gate` PASS
  rule (spearman >= 0.2 with CI lower bound above zero and the floor, monotone stratified CE, ECE
  < 0.05), `write_sigma_gate_artifact`, and `verify_gate_artifact` (the content check).
- `vfe3/viz/extract.py`: `belief_ce_bank` now also returns per-token `conf` (max softmax prob) and
  `correct` (argmax == gold) for the sigma-binned ECE; additive, existing callers unaffected.
- `vfe3/config.py`: Guard 4 hardened -- when `policy_sigma_ambiguity_validated` is True, config now
  content-checks the artifact (exists + `status=='PASS'` via `verify_gate_artifact`), so a FAIL/missing
  record cannot silently validate the flag (the prior check was structural-only; the spec_commit match
  is the Phase-3 consumer's job). Field comment corrected.
- `sigma_gate_measure.py` (new, click-to-run): rebuilds the model arch from the checkpoint's stored
  config (any saved checkpoint loads as-is), runs the gate on a held-out split, writes the artifact.
- `docs/research/active-inference/2026-06-28-sigma-gate-prereg.md` (new): the binding pre-registration.
- Tests: `+8` (sigma_gate statistics, PASS/FAIL decision, artifact, `verify_gate_artifact`) plus the
  hardened guard test. Full suite `1319 passed, 1 skipped, 1 xpassed` (JUnit `tests=1321 failures=0
  errors=0 skipped=1`).
