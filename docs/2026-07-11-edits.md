A full CPU `pytest` run timed out past 10 minutes. Per-file and per-test timing (135 test

Fix (in `tests/conftest.py` only — no edits to any test file, so no conflict surface): added the
standard `--runslow` opt-in. Fourteen figure/artifact/UMAP integration tests, listed by
`<file>::<test>`, are marked `slow` and **skipped by default** via
`pytest_collection_modifyitems`, so a plain `pytest` CPU run completes quickly; the full set runs
with `pytest --runslow` (CI / pre-merge / GPU). The gated group is the five `test_report.py`
figure-driver tests, four `test_run_artifacts.py` finalize/figure tests,
`test_round3_artifacts.py::test_emit_closes_figure_registered_by_raising_thunk`, the three

`test_run_diagnostics_2026_06_13.py::test_finalize_writes_tier3_research_and_provenance`. The five
heavy files dropped from ~360 s to ~18 s in the default run (93 passed, 14 skipped). Correctness
tests (the `test_train.py` learnability gates, the `test_forward_kl_uniqueness.py` theorem) are
intentionally NOT gated and still run by default.

## Standard (parameter-free) LayerNorm norm option

Added a `"layernorm"` key to the belief-mean normalization registry
(`vfe3/geometry/norms.py`), joining `"none"` and `"mahalanobis"` as a valid `norm_type_block` /
`norm_type_final` value.

`LayerNorm` applies the conventional transformer LayerNorm standardization to the belief mean:
`mu_norm = (mu - mean(mu)) / sqrt(var(mu) + eps)` over the belief dim, biased variance, matching
`torch.nn.LayerNorm(elementwise_affine=False)`. `sigma` is ignored (LN acts on the point `mu`; its
shape is never read, so diagonal and full-covariance sigma pass through untouched).

It is **parameter-free** (no learned `gamma`/`beta` affine), so it stays in the existing stateless
registry pattern and wires through the current `get_norm(cfg.norm_type_*)(cfg.embed_dim,
eps=cfg.eps)` seam with **zero** changes to `model.py`, `block.py`, or `gauge_optim.py` — a clean
integration alongside the active WIP in those files. It adds no `nn.Parameter`, so no optimizer
wiring and no new `NO neural networks` exception.

It is an **opt-in, NON-gauge-equivariant** baseline/ablation: centering and per-coordinate variance
in the fixed basis do not commute with `g` in `GL(K)`, so `LN(g mu) != g LN(mu)`. The gauge-pure
paths remain `"none"` (identity) and `"mahalanobis"` (gauge-invariant scale) and stay the default
(`norm_type_*` = `"none"`).

Files: `vfe3/geometry/norms.py` (`LayerNorm` class + `@register_norm("layernorm")` builder,
docstring), `vfe3/config.py` (norm doc comment; validation already reads the registry, so no
validator edit was needed), `tests/test_norms.py` (5 unit tests: torch-reference match,
standardization, sigma-ignored, non-equivariance pin, registry/build), `tests/test_model.py`
(end-to-end wiring: `layernorm` on both seams changes logits vs `"none"` and adds zero learnable
parameters). Verification: `pytest tests/test_norms.py` -> 7 passed; the wiring test passes and a
training-path backward smoke gives a finite loss with finite, nonzero gradients through the
layernorm forward.

Deferred follow-up: learned-affine LayerNorm (`gamma`/`beta`) is a separate build — it needs M-step
param-group wiring in `model.py` + `gauge_optim.py` and a new documented no-NN exception, and a
`block_norm` affine would carry the `detach_e_step` freeze footgun (it is applied inside the
E-step). The parameter-free variant here delivers the essential standardization cleanly without
touching those files.

## Explicit numerical shapes and honest degenerate statistics

`condition_number` helper now accepts `kind="auto"`, `"full"`, or `"diagonal"`. Auto mode preserves
the existing square-trailing-dimensions inference, full mode requires square matrices and uses the
symmetric eigenspectrum, and diagonal mode always treats the last axis as the variance spectrum.
This resolves the ambiguous `(N, K)` case when `N == K`. The converged numerical-health report now
passes the covariance kind explicitly, so a square diagonal belief table cannot be mistaken for one
full covariance matrix. The gauge optimizer likewise declares its pullback Gram metric as full rather
than relying on shape inference.

Spearman correlation now removes non-finite observations with a paired mask and assigns fractional
average ranks to ties. Fewer than two paired-finite observations return NaN, while a finite constant
input retains the established zero-correlation convention. The validation-sequence bootstrap now
returns NaN for its point estimate and both interval endpoints when the aggregate contains zero
evaluated tokens, rather than reporting a finite-looking zero cross-entropy.

Source regressions cover explicit diagonal and full conditioning, invalid shape and kind requests,
tie-aware ranks, paired-finite filtering, insufficient and constant samples, unchanged strictly
ordered correlations, the sigma-gate point-estimate seam, and the zero-token bootstrap. Test, import,
compile, lint, CPU, and CUDA execution remain deferred to the consolidated validation pass.
