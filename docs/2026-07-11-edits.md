## Standard (parameter-free) LayerNorm norm option

Added a `"layernorm"` key to the belief-mean normalization registry
(`vfe3/geometry/norms.py`), joining `"none"` and `"mahalanobis"` as a valid `norm_type_block` /
`norm_type_final` value.

`LayerNorm` applies the conventional transformer LayerNorm standardization to the belief mean:
`mu_norm = (mu - mean(mu)) / sqrt(var(mu) + eps)` over the belief dim, biased variance, matching
`torch.nn.LayerNorm(elementwise_affine=False)`. `sigma` is ignored (LN acts on the point `mu`; its
shape is never read, so diagonal and full-covariance sigma pass through untouched).

It is **parameter-free** (no learned `gamma`/`beta` affine), so it stays in the existing stateless
registry pattern and wires through the current `get_norm(cfg.norm_type_*)(cfg.embed_dim,
eps=cfg.eps)` seam with **zero** changes to `model.py`, `block.py`, or `gauge_optim.py` — a clean
integration alongside the active WIP in those files. It adds no `nn.Parameter`, so no optimizer
wiring and no new `NO neural networks` exception.

It is an **opt-in, NON-gauge-equivariant** baseline/ablation: centering and per-coordinate variance
in the fixed basis do not commute with `g` in `GL(K)`, so `LN(g mu) != g LN(mu)`. The gauge-pure
paths remain `"none"` (identity) and `"mahalanobis"` (gauge-invariant scale) and stay the default
(`norm_type_*` = `"none"`).

Files: `vfe3/geometry/norms.py` (`LayerNorm` class + `@register_norm("layernorm")` builder,
docstring), `vfe3/config.py` (norm doc comment; validation already reads the registry, so no
validator edit was needed), `tests/test_norms.py` (5 unit tests: torch-reference match,
standardization, sigma-ignored, non-equivariance pin, registry/build), `tests/test_model.py`
(end-to-end wiring: `layernorm` on both seams changes logits vs `"none"` and adds zero learnable
parameters). Verification: `pytest tests/test_norms.py` -> 7 passed; the wiring test passes and a
training-path backward smoke gives a finite loss with finite, nonzero gradients through the
layernorm forward.

## Learned-affine LayerNorm (`layernorm_affine`)

Follow-up to the parameter-free option above (built once GPT-sol's WIP merged to `main`): a new
default-OFF toggle `layernorm_affine` adds a learned per-feature affine `gamma`/`beta` to any
`"layernorm"` norm seam, so `mu_norm = gamma * LN(mu) + beta` — exactly
`torch.nn.LayerNorm(elementwise_affine=True)` on the belief mean. `gamma` inits to 1 and `beta` to 0,
so at construction the affine is the identity and the forward/loss is **byte-identical** to the
parameter-free `"layernorm"` (the step-0 contract the `t5_bias`/`learnable_kappa` exceptions carry).
It is a sanctioned learned-scalar exception (raw `nn.Parameter` diagonal scale/shift, no
`nn.Linear`/MLP/activation) — added to the `CLAUDE.md` no-NN exception list as (6). It stays
non-gauge-equivariant (the same break `"layernorm"` itself has); the gauge-pure `"none"`/
`"mahalanobis"` paths are untouched and remain the default.

Design: the affine lives on a new `AffineLayerNorm` `nn.Module` (the builder returns it when
`affine=True`, else the parameter-free `LayerNorm`), so assigning it to `self.block_norm` /
`self.final_norm` auto-registers `gamma`/`beta` as model submodule parameters. Each seam gets its own
pair; the block seam's affine is **shared across the L blocks** (the block norm is a single shared
instance — the minimal extension of the existing architecture; per-block affine would be a larger
change). `build_optimizer` groups them at `m_p_mu_lr`, `weight_decay=0`, `role='mu'` (like
`t5_bias`/`log_kappa`); the exact-coverage guard fail-closes if they were ever left ungrouped.

E-step coupling: the **final** norm is post-stack and its `gamma`/`beta` train under any estimator.
The **block** norm is applied to the belief VALUE inside the stack, so — unlike `learnable_kappa`,
which enters only the E-step tangent and is frozen by `straight_through` — the block affine trains
under both `'unroll'` and `'straight_through'` (both keep the belief value differentiable, verified
by a gradient probe) and is frozen **only** by the fully-detached E-step (effective `'detach'`, which
`no_grad`s the whole stack). `model.__init__` warns for exactly that case, and also when
`layernorm_affine=True` but no seam is `"layernorm"` (inert).

Files: `vfe3/geometry/norms.py` (`AffineLayerNorm`, shared `_ln_standardize` helper, builder
`affine` arg), `vfe3/config.py` (`layernorm_affine` field + doc), `vfe3/model/model.py` (pass
`affine=`, footgun/inert warnings), `vfe3/train.py` (optimizer group), `CLAUDE.md` (exception 6),
`tests/test_norms.py` (5 `AffineLayerNorm` unit tests), `tests/test_layernorm_affine.py` (12 tests:
defaults, param creation/registration, step-0 byte-identity, per-estimator gradient flow/freeze,
inert warning, optimizer grouping, end-to-end training update). Verification: the affine + norm +
model + kappa + optimizer + config + checkpoint + reflection test set runs green (250 passed across
the touched files); adversarially verified across math / integration / variational / gauge-purity
lenses.

## ablation.py <-> train_vfe3.py reproducibility parity

`ablation.py` and `train_vfe3.py` produced different trajectories under the "same" settings. Three
causes were found and closed, all in `ablation.py`:

- **`DATA_SEED`** (new module knob mirroring `train_vfe3.DATA_SEED`, default 3): the train loader's
