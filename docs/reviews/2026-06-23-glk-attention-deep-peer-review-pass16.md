# GL(K) Attention Manuscripts - Deep Peer Review Pass 16

Date: 2026-06-23

Scope: This pass reviews the current desktop checkout of `Manuscripts-Theory/GL(K)_attention.tex` and `Manuscripts-Theory/GL(K)_supplementary.tex`. It intentionally avoids judging live empirical values, including PPL, CE, loss curves, benchmark ratios, and run-dependent numerical claims. The focus is proof hygiene, objective consistency, gauge covariance, and claim grading after the pass-15 fixes.

Recommendation: major clarification, not rejection. The current manuscript has absorbed several earlier corrections: the RG transport average is now explicitly described as a flat-coordinate diagnostic, the real/complex SU(N) issue is fixed in the supplement, the backprop row has been softened, and the envelope-versus-surrogate distinction is much clearer. The next layer of risk is subtler: several statements still present local or structural constructions as if they were descent of one global free energy or exact derivations of standard architecture.

## Major Comments

1. The default filtering update is not coordinate descent on the stated global reduced free energy. The manuscript now states that there is a single reduced objective `F_red`, then says the default implementation updates each `q_i` using only the query-side row partial and omits the key-side column term from later positions (`GL(K)_attention.tex:901`-`919`). For a single pairwise objective, a coordinate update of `q_i` should include every factor in which `q_i` appears. Omitting the column terms is a legitimate filtering approximation for an autoregressive recognition family, but it is not coordinate ascent or descent on the full `F_red`. The clean fix is to define a local filtering objective, for example `F_i^filter(q_i; q_{<i} detached)`, and reserve the global `F_red` for the opt-in smoothing path that includes the column terms. Otherwise reviewers can correctly say that the default algorithm is not optimizing the objective the theory just derived.

2. The temperature parameter still moves between three roles that should be separated. The row mixture KL gives the temperature-one objective; the text then introduces `tau` as a genuine modeling choice by tempering the alignment row (`GL(K)_attention.tex:785`). Later the global free-energy paragraph says substituting `beta*` into `F_align` from Eq. `mixture_energy_entropy` yields `-tau log Z_i` (`861`), but that equation is the untempered temperature-one expression. The reference should be to Eq. `F_align_canonical_tau`, or the sentence should explicitly say that the tempered objective has replaced the literal mixture KL. The abstract and conclusion should then distinguish three claims: the softmax form follows from entropy-regularized source assignment, `tau` is a tempered alignment parameter, and the `sqrt(d_k)` scaling is recovered in the dot-product limit by dimensional concentration.

3. The content-dependent key-side bias should not be called a prior unless the manuscript defines a state-dependent prior. In the untied query-key route, the `j`-only term `r_j = ||K_j||^2` is absorbed into the additive prior slot (`GL(K)_attention.tex:1169`, `1202`). Algebraically this is a valid logit rewrite. Probabilistically, however, a prior over source identity should be specified before observing the current key belief; `r_j` is a function of the current transported key state. Calling it `log pi_ij` makes the prior depend on the inference variables. The safer wording is "key-side logit bias" or "empirical-Bayes/source-salience prior" with the state dependence declared. This matters because the causal mask, ALiBi, and T5 priors are genuine row priors or positional biases, while the key-norm term is an energy residue.

4. Value aggregation is still graded too strongly as a derivation of standard `W_V`. The gauge model gives the generative aggregation `hat mu_i = sum_j beta_ij Omega_ij mu_j`, while standard attention uses an independently learned value projection. The text says the constant transport can be absorbed into a head-space `W_V` and uses a thin-SVD lift for rectangular value projections (`GL(K)_attention.tex:1362`-`1374`). That proves representability after choosing a head subspace, but it does not derive the standard transformer's independent `W_V` parameterization from the same gauge element that already fixes the query-key bilinear. The correspondence-table row at `1719` should be `D#` or `S`, not plain `D`, unless the paper explicitly assumes a tied value map or proves an independent value-channel transport.

5. RoPE and VFE-iteration correspondences still look overgraded. The RoPE section gives a useful SO(2) subgroup embedding and handles the transpose/sign convention carefully, but it does not derive the RoPE frequency schedule or the fact that standard RoPE rotates queries and keys but not values. The table row at `1728` should therefore be structural or derived-up-to-choice, not plain `D`. Similarly, the row marking VFE inner iterations as "Tied-weight recurrence; DEQ at L=1" with status `D` (`1752`) overstates the result. Iterating a fixed VFE map is structurally analogous to tied recurrence or a deep equilibrium solve, but the standard DEQ architecture also specifies an input injection, solver, readout, and training objective. Mark this row `S` unless exact algorithmic equivalence is proved.

6. The FFN and activation analogy should be fenced more tightly. The Boltzmann gate section correctly says GELU, SiLU, and the VFE gate are not functionally identical (`GL(K)_attention.tex:1986`-`1997`). It then says multiple VFE iterations constitute the analogue of the FFN sublayer (`2013`). The default canonical path cancels the softmax-gradient channel, leaving only the bare Boltzmann gate, while a transformer FFN supplies learned per-token channel mixing with expansion and contraction matrices. This remains an interpretive or structural analogy, not a derivation of the FFN. The manuscript should avoid implying that VFE iterations replace an MLP at depth until deep-stack experiments and rank-collapse diagnostics are available.

7. The GL(K), GL+(K), and subgroup story is fixed in the supplement but not fully fixed in the main text. The supplement now correctly says that U(N) and SU(N) require complexification or realification (`GL(K)_supplementary.tex:1337`). The main text still lists `SO(N), SU(N), Lorentz group, etc.` as subgroups that can be imposed as inductive biases (`GL(K)_attention.tex:2441`). That sentence needs the same real-representation qualification. Separately, the local gauge corollary is stated for arbitrary `h_i, h_j in GL(K)` (`565`), while the realized vertex-frame transport is `GL+(K)` (`638`). This is fine as a score-invariance theorem, but the implemented gauge-frame parameterization realizes only the identity component unless the reflection extension is enabled. Say this every time the text moves from theorem-level GL(K) to trained-model GL+(K).

8. The statement "standard attention transformers are a 0-dimensional gauge theory" remains too strong. The current line (`GL(K)_attention.tex:2409`) should read as a reconstruction claim: standard transformers can be represented, under the paper's gauge reconstruction and limiting assumptions, as a 0-dimensional flat-gauge case. Without that qualifier, the sentence attributes gauge-theoretic structure to ordinary transformers rather than to the manuscript's representation of them.

9. Appendix H still hides a load-bearing richness assumption in the proof. The theorem statement requires the geometric-mean stationary form for all priors and neighbor beliefs (`GL(K)_supplementary.tex:1212`-`1228`), but the reverse implication later chooses neighbor beliefs so all transported densities match a common reference density and assumes those inverse-transported densities are normalizable (`1289`). That admissibility/richness condition is essential for isolating a single `f'` evaluation and should be included in the theorem statement. This is not a fatal flaw, but it is exactly the sort of hidden assumption a mathematical reviewer will mark.

10. The Renyi extension should be described as an energy-substitution path, not as the same variational derivation. The text now correctly says that geometric-mean belief updates and the forward-KL uniqueness theorem do not extend to `alpha != 1` (`GL(K)_attention.tex:1117`). The abstract and result framing should mirror that caution: using Renyi-1/2 as the alignment score preserves the row softmax once a scalar energy is chosen, but it is not the exact KL mixture-of-sources free-energy derivation and does not inherit Appendix H's uniqueness result.

## Minor Comments

1. At `GL(K)_attention.tex:861`, replace the reference to Eq. `mixture_energy_entropy` with `F_align_canonical_tau`, or explicitly say the untempered mixture KL has been replaced by the tempered row objective.

2. At `GL(K)_attention.tex:1202`, "vanish under the closure" should be "become row-only and cancel under the row softmax." The terms do not vanish as algebraic terms; they stop affecting the row distribution.

3. At `GL(K)_attention.tex:942`, use "a priori" rather than "apriori."

4. At `GL(K)_attention.tex:1362`, the square per-head `W_V in R^{d_k x d_k}` convention is fine inside the head-space reconstruction, but the transition to the usual rectangular implementation should be stated before the boxed equality so readers do not think the paper is claiming literal parameter equality with standard library `W_V`.

5. The "two equivalent routes" sentence at `GL(K)_attention.tex:1070` should be softened. The routes coincide only after closure assumptions, row-only cancellations, and key-norm control; before those assumptions they produce different residual terms and different interpretations of the key-side bias.

6. The line `GL(K)_attention.tex:588` says taking the gauge group dimension equal to the embedding dimension "does not restrict the generality of the derivations below." That is true for the full GL(K) construction, but subgroup/product representations require a representation-specific statement. A safer wording is that the simplifying choice is sufficient for the derivations shown.

## Strengths Of The Current Revision

The manuscript is now much more honest about envelope versus surrogate gradients, about the BERT comparison as a consistency check, and about the RG transport average being non-intrinsic. The proof of GL(K) invariance is clean. The dot-product limit is also substantially better than earlier versions because it now names the approximate key-norm step and distinguishes rectangular projection factors from the invertible head-space kernel. The most useful next revision is not another empirical run; it is a tightening pass that assigns every architectural correspondence to the right claim type: exact derivation, derived up to equivalence class, structural analogy, or interpretation.

## Bottom Line

The main remaining theory risk is objective drift. The paper derives a tempered local row functional, then presents a global reduced free energy, then describes an implementation that defaults to a query-only filtering update rather than the full gradient of that global objective. This can be made rigorous, but only if the manuscript explicitly separates the local filtering objective from the global smoothing objective. After that, the remaining work is mostly claim calibration: value projection, RoPE, FFN/DEQ, subgroup language, and the Renyi path should all be stated at their actual proof strength.
