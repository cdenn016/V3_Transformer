## Verdict

No CRITICAL findings. On the pure toggles (gaussian_diagonal/gaussian_full + renyi at order 1, include_attention_entropy=True, flat transport, use_prior_bank=True, constant α/λ) every formula in scope is the exact textbook object: the canonical F with the attention-entropy term exists and softmax β is its stationary point; transport is a genuine product of two exponentials with no exp(φ_i−φ_j) shortcut anywhere; every covariance transport is the sandwich ΩΣΩᵀ (diagonal approximation is the exact diagonal of the full sandwich, and an exact full path exists under gaussian_full); the canonical-vs-surrogate gradient split is routed correctly (envelope kernel only when the entropy term makes β stationary, autograd oracle otherwise); τ = κ√d is computed once with no double-scaling; decode is the exact unclamped −KL/τ_eff against the same bank encode uses; the no-NN constraint holds (zero nn.Linear/activation hits outside the documented exception family); entry points are clean of CLI parsing; all claimed registries are real dict+decorator registries with fail-closed duplicates; no silently-dead config key was found in a full sweep of all 130 fields.

One MAJOR silent-freeze footgun was found (below), plus a set of MINOR hazards and NOTEs.

## MAJOR

**M1. `t5_bias` gradient silently severed under `precision_weighted_attention=True` + `t5_learnable_bias=True`.** `_fold_precision_bias` is decorated `@torch.no_grad()` (`vfe3/model/model.py:1430-1431`) and returns `log_prior + kb` under it (`model.py:1458`). When the log prior carries the live learnable T5 table (built cache-bypassed precisely to keep the graph, `model.py:472-482`), the forward folds the precision bias into it at `model.py:762`; the no_grad addition emits a `requires_grad=False` tensor, severing `t5_bias`'s only gradient path. Values are unchanged; `t5_bias.grad` is silently None; no existing warning covers this combination (the model-level T5 warning gates only on detach/straight-through, `model.py:407-424`). The per-branch `.detach()` calls on `kb` (`model.py:1450, 1453, 1456`) show the intent was to detach only the precision bias, not the prior it is added to — the decorator over-detaches. Independently re-verified. Fix shape (not applied): drop the decorator; `kb` is already detached in every branch. Reachable only under the double opt-in, but it is exactly the silent-freeze class this codebase otherwise warns for.

## MINOR

**Math / free energy**

- **m1. Unconditional ε-ridge on the full-covariance KL.** `sigma_q_reg = sigma_q + eps*eye; sigma_t_reg = sigma_t + eps*eye` is applied always, not only on Cholesky failure (`vfe3/families/gaussian.py:342-343`; same ε at `prior_bank.py:494, 566`). Unlike the diagonal `clamp(min=eps)` (inert whenever retraction keeps σ ≥ eps), the full path always scores N(μ, Σ+εI): a standing 1e-6 bias in values and gradients on the full-cov pure path, not a fallback.
- **m2. `kl_max` saturation flattens β rows and zeroes pair gradients.** Every energy passes `safe_kl_clamp` (`families/base.py:19-27`), so a row whose raw KLs all exceed `kl_max` softmaxes to uniform and the kernel's pair mask zeroes its gradient (`gradients/kernels.py:295`). Config-adjustable (`config.py:42`); decode correctly opts out with `kl_max=inf` (`prior_bank.py:369, 789`). Hazard, not error.
- **m3. s-channel blocks reduce with `.mean()` over (B,N)** (`model/model.py:1122, 1184, 1188`) while belief F sums (`free_energy.py:379-400`). Declared a deliberate calibration of λ_h/λ_γ (`model.py:997-1000`) and absorbable into those weights at fixed shape, but a sum-scale pure variant does not exist under any toggle.
- **m4. Registry metric wrapper drops λβ/entropy/α-reg.** `_m_free_energy_terms` (`metrics.py:1201-1210`) forwards neither `lambda_beta` nor `include_attention_entropy` nor `alpha_reg`, so its total matches runtime F only at λβ=1/entropy-ON/constant-α. The live diagnostics path passes all three (`model.py:1574-1578`); only registry-driven consumers are affected.
- **m5. Decode chunked/full paths differ on the KL floor**: `_decode_diagonal` floors `kl_v` at 0 (`prior_bank.py:746`), `_decode_full_chunked` does not (`prior_bank.py:823`). Cosmetic; softmax ranking unaffected. Related hazard: the hard ±3 clamp on `decode_log_scale` zeroes its gradient at the boundary, so the decode temperature can park there (`prior_bank.py:319-321`).

**Geometry / gauge**

- **m6. Gauge-RoPE breaks global gauge equivariance for every group and is absent from the CLAUDE.md exception list.** The repo's own probe records it (`config.py:1302-1306`: β residual 0.44–0.61 under rope vs 7e-6 without; the config comments call RoPE a deliberate gauge-fixing layer). Runtime warnings exist only for sp/so_n/sp_n structure-group departures (`config.py:1309-1333`) and means-only incoherence (`config.py:1339-1348`); a `block_glk` + `rope_full_gauge=True` run gets no warning although β-equivariance breaks there too. Pure path intact (`pos_rotation="none"` and the equivariant `pos_phi` composition, `positional_phi.py:123-157`); the gap is documentation parity with the other accepted exceptions.
- **m7. `transport_covariance` rank-gap dispatch is fragile.** Diagonal detection is `sigma.dim() == omega.dim() - 2` (`transport.py:1039`); a batch-independent (N,N,K,K) Ω with a batched (B,N,K) diagonal σ misclassifies as full covariance and shape-errors (or mis-broadcasts if B==N). Currently unreachable — every live call site passes `diagonal_out` explicitly or runs unbatched (`kernels.py:282`, `oracle.py:123`, `config.py:1619-1622` gate) — guarded by call-site discipline, not the primitive.
- **m8. M-step gauge update is an unbounded coordinate step.** `GaugeNaturalGradAdamW.step` moves φ tables by plain `p.add_` (`gauge_optim.py:199, 206`) with no trust region; a φ row drifting past Frobenius 15 silently enters `stable_matrix_exp_pair`'s clamped surrogate `exp(15·M/‖M‖)` (`transport.py:742-749`), and the clamp monitor is opt-in (`clamp_monitor=False`, `transport.py:775-786`).

**Gradients / training**

- **m9. Config freeze-warning predicate omits `t5_bias` on the oracle route.** The detached-oracle-unroll warning (`config.py:1792-1813`) lists log_alpha/connection_W/log_lambda_beta/log_lambda_h/r/pos_phi_free but not `t5_bias`, whose only path is likewise the E-step tangent. Mitigated by the generic runtime truncation warning (`e_step.py:495-510`); a warning-coverage gap, not a silent freeze.
- **m10. fp32 islands key autocast-disable to `'cuda'` only** (`retraction.py:143, 183, 284, 357, 390`; `transport.py:797`) while `_amp_context` supports CPU autocast (`model.py:522-545`). Matmul-class ops inside those islands remain autocast-eligible under CPU bf16 AMP. Irrelevant on the CUDA path.

**Config / architecture / tests**

- **m11. `e_step_mu_precond` is silently not honored by the s-channel E-step.** `vfe_block` threads it (`block.py:75`) and `e_step_iteration` consumes it (`e_step.py:556-559`), but `_refine_s`'s `e_step(...)` call (`model.py:597-643`) omits it, so the s-refine always runs the default `'fisher'`. Unlike the s-channel's flat-transport tie and hardcoded Gaussian family (both warned: `config.py:1680-1688`, `config.py:1148-1158`), this divergence has no warning. Under `s_e_step=True` a `raw`-preconditioner ablation is contaminated: raw on the belief channel, Fisher on the s channel. Verified against the call site.
- **m12. `lambda_h` under `lambda_h_mode='state_dependent'` + `s_e_step` is only a channel-on gate**, not a coupling weight — `alpha_state_dependent` reads only b0_h/c0_h and drops `value` (`model.py:613` with the in-code note at `model.py:610-612`; `alpha_i.py:69`). Documented in a comment only; no config-time warning, unlike sibling inert-toggle warnings.
- **m13. `oracle_unroll_grad=False` is silently coerced to True** for regime_ii/covariant and conditionally for the link modes (`config.py:1761-1787`). Deliberate and correctness-improving, but it is the only `__post_init__` coercion that does not warn (`gauge_transport` coercions warn at `config.py:650-668`; `use_head_mixer` auto-disable warns at `config.py:794-802`).
- **m14. Adding a transport variant that carries learned parameters requires editing call sites.** Transport build is registry-dispatched, but parameter creation is a `transport_mode` literal chain in `VFEModel.__init__` (`model.py:223, 245, 263`), optimizer grouping enumerates names (`train.py:137-150`), and the `__post_init__` guard blocks enumerate transport names literally. Param-free variants remain pure write-and-register.
- **m15. `gauge_parameterization` is validated but never read at runtime** — `"phi"` is the only accepted value, `'omega_direct'` raises NotImplementedError (`config.py:809, 823-828`), and nothing dispatches on the field. Cannot misbehave, but it is a pure placeholder.
- **m16. `pyproject.toml` dependencies understated**: `dependencies = ["torch"]` (`pyproject.toml:10`) but `viz/figures.py:15-21` unconditionally imports matplotlib and numpy, and `generate_figures=True` is the default (`config.py:591`) — a clean-env install completes training then crashes at finalize. pytest/tiktoken/umap-learn also undeclared.
- **m17. `pytest_full_run.txt` at repo root is stale (Jun 14) and records a crashed run** (tail is a mid-line access-violation dump inside `prior_bank._decode_linear`). Gitignored, but it documents no pass count and conflicts with the tidy-worktree discipline.

## NOTES

- n1. The `-E_q[log p(o|x)]` term has no live E-step path: `free_energy.py:401-402` accepts `log_likelihood` but no production caller supplies it; the data term is realized as the decode CE in the M-step (design documented at `free_energy.py:358-368`). Coherent, but no toggle puts the likelihood into the E-step objective.
- n2. Even under `unroll`, the φ substep is always straight-through (`create_graph=False`, `e_step.py:612`); "unroll" is a full second-order unroll for μ/σ only. The φ substep is Gauss–Seidel: post-update (μ,σ) with pre-update φ (`e_step.py:579-584`).
- n3. EMA has no decay warmup/bias correction — shadow seeded from init weights (`ema.py:38-42`), init-biased for runs shorter than ~1/(1−decay) steps; the non-finite skip in `update` is silent (`ema.py:66-70`).
- n4. `belief_cache` holds no persistent state and cannot leak gradients across steps (`belief_cache.py:147, 169-196`); it does not gate on `amp_dtype`, so the cached-rollout equality tolerance loosens under AMP (inference-only).
- n5. Isotypic head-mixer "exactly equivariant" holds for means + full-covariance; the diagonal-σ closed form `σ' = Σ A²σ` is diagonal-gauge-only (`head_mixer.py:43-45, 179-181`) — same status as the diagonal transport sandwich. CG path-weight equivariance is structural (intertwiners are null-space solutions with a build-time equivariance-residual raise, `cg.py:126-133`) and pinned (`tests/test_cg.py:28, 93`).
- n6. so_n irrep images are skew only to f64 construction precision (`irreps.py:183`), so the `exp(−M)=exp(M)ᵀ` fast path carries ~1e-15 asymmetry — negligible.
- n7. BCH compose is Dynkin order ≤ 4 with stated truncation (`lie_ops.py:262-271`); no exact `log(exp X exp Y)` path exists, but Ω is always exp of coordinates so the frame never leaves the group.
- n8. Generator sets verified against their defining relations in code and pinned by tests (`generators.py:210-279`; `tests/test_gauge_groups.py:42, 62, 91`); irrep towers raise at build time on dimension or bracket-homomorphism failure (`irreps.py:202-251`).
- n9. Laplace family KL and Rényi are exact closed forms including the removable c_p=c_q singularity (Taylor sinhc below fp64 rounding, `laplace.py:152-191`); its gauge transport is a disclosed projection valid under permutation/sign gauges (`laplace.py:42-54`).
- n10. `alpha = c0/(b0 + kl).clamp(min=eps)` guards division but not blow-up: user-set b0=0 with D=0 yields α ≈ c0·1e12 (`alpha_i.py:101, 160`).
- n11. `_LAMBDA_H_MODES` is a static tuple, not a registry (`lambda_h_i.py:38`); a newly registered alpha form is not selectable as a lambda_h_mode without editing it.
- n12. `viz/extract.py:63-88` manually mirrors `vfe_block`'s E-step knob bag; every new E-step knob must be added in two places or diagnostics silently diverge from training — same drift class as m11. Duplicated `_as_coeff` helper (`block.py:21`, `belief_cache.py:50-53`, the latter to break an import cycle).
- n13. Tests: 1,398 test functions across 118 files; effectively zero assert-free tests (all 15 AST candidates are false positives); golden pins, finite-difference oracle checks, and gauge-equivariance property tests all exist as claimed; every vfe3 module has coverage (weakest is integration-only: `model/block.py`, `model/cg_coupling.py`). conftest honors `VFE3_TEST_DEVICE` with clean CUDA skip.
