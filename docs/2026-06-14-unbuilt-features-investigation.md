# Unbuilt-but-mandated features — investigation report (2026-06-14)

## Executive summary

A seven-domain sweep surfaced 30 genuine gaps (`confirmed_unbuilt` or `partial_stub`) between what the VFE3 manuscripts and the binding in-repo roadmap call for and what exists in code, plus 5 speculative-only items correctly filed as non-gaps, with nothing rejected as already-built or misattributed. After deduplicating features that recurred across domains (the observation channel, the per-agent clamp mask, and the model.step/state-carry seam each appeared in two or three domains), the genuine set collapses to roughly 17 distinct features. The single dominant theme is that the **canonical free energy's fifth term — the observation/likelihood coupling `-E_{q_i}[log p(o|k_i)]` — and everything it enables (persistent beliefs, a real interaction loop, action) is unbuilt**, while the entire multi-scale meta-agent tower (cross-scale shadow priors, consensus detection, barycenters, RG harness) is a second large absent cluster that the manuscript itself sanctions deferring as an extension realized in a separate codebase. The three or four highest-leverage gaps are all in the first cluster and all already designed: the **exogenous observation channel in both gradient paths** (P1/P2), **prior-carry persistence plus a `model.step` entry point** (P1), and the **cross-entropy covariance sector** that makes evidence actually accumulate (P2). The highest-leverage strictly-theoretical item outside that cluster is the **token-dependent cross-scale shadow prior** `p_i = Omega_{i,I}[q_I^{(s+1)}]` (P1), which the manuscript calls "a central economy of the framework" but which is gated behind the absent scale-(s+1) object.

## Priority tiers

### P1 — load-bearing, mandated or near-mandated

#### Cross-scale shadow prior `p_i = Omega_{i,I}[q_I^{(s+1)}]` (token-dependent top-down prior from a parent)

The prior `p_i` and hyper-prior `r_i` at scale `s` are not independent fields but the gauge-transported posterior/model of the enveloping scale-(s+1) meta-agent. The manuscript states this verbatim as a structural commitment: `Participatory_it_from_bit.tex:547` — "`p_i^{(s)}(c) = Omega_{i,I}[q_I^{(s+1)}](c), r_i^{(s)}(c) = tilde-Omega_{i,I}[s_I^{(s+1)}](c)`" — restated at `:2298` (eq:topdown_priors), with `:628-629` listing `p_i, r_i` as derived sections. Code is PARTIAL: the only realization of `r_i` is the frozen/learnable global `(K,)` parameter at `prior_bank.py:195-199`, with an explicit `TODO(B)` that the manuscript-pure form "awaits the scale-(s+1) meta-agent (not built)"; `model.py:733-738` carries the same TODO; `config.py:264-267` (`prior_source='model_channel'`) takes the deliberately-different same-scale reading. No Omega-transport is ever applied to a parent posterior. Building it requires the scale-(s+1) object (below) to supply `q_I/s_I`, then reuse the existing same-scale transport (`geometry/transport.py compute_transport_operators`) to push the parent posterior into each child's frame. **Effort XL** (strictly downstream of and scoped as one unit with meta-agent formation; the transport math is reusable, the parent field is not). **Strength: mandated** for the full theory, but the manuscript sanctions the single-scale frozen-`r` boundary as a legitimate `s_max` boundary condition (`PIFB:554`), so the pure single-scale path is not violated.

#### Persistent beliefs (prior-carry filtering) + a `model.step` entry point

The fast subsystem updates beliefs to explain observations while priors stay fixed on the slow timescale (`PIFB:1564`); the loop closes by routing a belief into the prior slot, `p_i <- Omega_{i,I}[q_I]` (`PIFB:2298`). The temporal degenerate of this — an agent's own transported past posterior carried into its next prior slot, `p_{t+1} = Omega q*_t + inflation Q` — is the only memory-bearing temporal mechanism and the resolution of the observation term's vacuity. Code is ABSENT: `VFEModel.forward` (`model.py:523-530`) is stateless with no `state` argument; `generate` (`model.py:1006-1063`) re-runs the full stateless forward on the running sequence and its own docstring (`model.py:1037`) calls incremental belief reuse "a future optimization"; `encode` returns `q=p` (`prior_bank.py:201-206`); the prior slot routes only between two `(V,K)` vocab tables (`prior_bank.py:267-280`) and cannot host a runtime posterior. The persistence seam does exist — `e_step(belief, mu_p, sigma_p, ...)` (`e_step.py:508-512`) takes init belief and prior as separate tensors — but no caller threads a carried posterior. Building it is a new `model.step(token_ids, state) -> (logits, state)` branching at the encode call site (`model.py:539`), seeding belief (init-carry) and/or substituting the transported carried posterior + inflation `Q` into `mu_p/sigma_p` (prior-carry), state detached; no kernel edit. **Effort M.** **Strength: mandated** for the cross-scale shadow it instantiates, but **endorsed_extension** for the specifically *temporal* self-shadow reading — `PIFB:2298/550` is cross-scale (scale s+1 to s), not temporal (t to t+1); the temporal reading is the roadmap's interpretive move (`docs/2026-06-10-realtime-interaction-model-roadmap.md:94`). Init-carry (Phase 0) is objective-neutral compute amortization; only prior-carry (Phase 2) is memory-bearing, and `Q=0` gives a precision ratchet, so inflation is mandatory.

#### Exogenous observation channel in both gradient paths (analytic kernel + autograd oracle)

The canonical free energy carries `-E_{q_i}[log p(o_i|k_i)]` as term 5; for Gaussian observations it reduces to a precision-weighted MSE pull `Lambda_o(mu_i - c)` on the belief mean. The manuscript proves mean-gradient equivalence to the likelihood for any `Sigma_o > 0` (`PIFB:1477`) with the closed forms at `:1487-1493`, and puts the term in the fast-subsystem belief F (`PIFB:1571`) and canonical F (`GL(K)_attention.tex:1392`). Code is ABSENT in the live belief descent and a NAMED_STUB in the oracle mirror: `belief_gradients` (verified `kernels.py:200-229`) has no observation argument; `belief_gradients_autograd` (`oracle.py:51-80`) likewise; `free_energy_value` (`e_step.py:150-187`) likewise; `free_energy`'s `log_likelihood` arg (verified `free_energy.py:340`, subtracted at `:394-395`) is a documented gated stub (`free_energy.py:357-367`) with zero production callers, and the kernel route never calls `free_energy()`. A repo grep for `Lambda_o|F_obs|env_agent|observation_channel` returns nothing. Building it adds `F_obs = sum_i[(1/2)(mu_i-c)^T Lambda_o (mu_i-c) + (1/2)tr(Lambda_o Sigma_i)]` with `dF/dmu = Lambda_o(mu_i-c)` and the covariance sector into both `gradients/kernels.py` and `gradients/oracle.py`, plus its own `kl_max=inf` policy and a FD gradient check, and makes `free_energy()`'s `log_likelihood` the live oracle mirror. **Effort M-L.** **Strength: mandated** as a stated proposition with explicit construction. The severity is split P1/P2 only because the term is mathematically vacuous in the current teacher-forced next-token LM (encode returns `q=p`, next-token CE already books the data term) until the interaction loop and/or prior-carry exists — the roadmap orders it after Phase 0.

### P2 — mandated/endorsed, real but gated or research-flavored

#### Cross-entropy covariance sector (precision addition, not averaging)

Full variational equivalence to the likelihood requires the cross-entropy coupling `-E_{q_i}[log q_{e_k}] = KL + H(q_i)`: the pure-KL covariance gradient is `(Lambda_o - Sigma_i^{-1})/2` (precision averaging — confidence saturates at sensor precision, evidence never accumulates) while the negative-log-likelihood's is `Lambda_o/2` (Bayesian precision addition). Manuscript: `PIFB:1491` gives both forms verbatim; `:1497` gives the substitution. Code: this distinction has no locus today because no observation term feeds any covariance gradient — the only Sigma-sector divergence machinery is the KL/Renyi self-divergence (`kernels.py:42-141`). This is a correctness constraint *inside* the observation-channel build, not a separable item: the roadmap already commits to xent-by-default with KL behind a mean-tracking toggle (`roadmap:41-46, 85`). The manuscript offers two valid routes to full equivalence — the xent substitution or fixed-covariance dynamics (`PIFB:1477, 1500-1501`) — so the strength is conditional. **Effort M** (rides the same kernel+oracle plumbing). **Strength: endorsed_extension / conditional-mandate.** Pre-registered falsifier (`roadmap:92`): the xent route must reach `Sigma*^{-1} = Lambda_p + Lambda_o/alpha` on a linear-Gaussian tracking task.

#### Per-agent clamp mask in the E-step (boundary agents held fixed)

Environmental agents are a boundary class whose fast/slow channels are fixed by construction (`q_e = p_e, beta_{i,e}=1`); they carry sensory information rather than autonomously minimize F (`PIFB:1463`, verbatim and accurate). Realizing this needs the inner loop to hold env-agent rows immobile while ordinary rows settle. Code is ABSENT: `e_step_iteration` (`e_step.py:430-505`) applies one uniform update to all rows (`mu` at `:462`, sigma retraction at `:465`, phi at `:503`) with no row selector; `e_step`, `vfe_block` (`block.py:56`), and the model handoff carry no mask; grep for `clamp_mask|env_agent|boundary_agent` returns nothing. Building it threads an optional `(N,)` boolean mask through the E-step update sites that zeroes `delta_mu/delta_sigma` and the phi substep on clamped rows; verifiable by a two-line invariant (clamped rows immobile, unclamped still descend F). **Effort M.** **Strength:** the manuscript *concept* is mandated; the clamp-mask *mechanism* is mandated by the roadmap (`:62`), not the manuscript. P2 because a clamped env row is inert without the observation term, so the mask delivers value only inside the larger Phase-1 buildout.

#### Forward-KL uniqueness theorem as an executable property test  [BUILT 2026-06-14]

The supplementary proves forward KL is the unique f-divergence (within the convex, real-analytic, locality+linear-coupling class) yielding the closed-form geometric-mean Boltzmann belief update — the load-bearing justification for why the inter-agent coupling must be KL even though the self-coupling `alpha` is free. `GL(K)_supplementary.tex:1218` — "if, and only if, D is the forward Kullback-Leibler divergence"; `GL(K)_attention.tex:1096` gives the falsifiable corollary that the geometric-mean target is specific to `alpha=1` and is transcendental otherwise. Code: the divergence registry (`families/base.py:437-440`) allows renyi/hellinger/bhattacharyya/jeffreys for all coupling terms with no role distinction, and one knob (`cfg.divergence_family/renyi_order`) drives both inter-agent and self coupling. No test asserts `q_i* propto p_i^{1/2} prod_j (Omega_ij q_j)^{beta_ij/2}` at `alpha=1` or its breakdown at `alpha != 1`. Building it is a property test (no runtime code) running the E-step to convergence and checking the geometric-mean identity at `renyi_order=1` (passes) vs `0.5/1.5` (fails). **Effort M** (finite-LR descent requires iterating to near-convergence with a tight-but-non-vacuous tolerance). **Strength: mandated** theorem with a falsifiable numerical corollary; this is a test gap, not a capability gap.

#### Cross-scale (meta-agent) transport operators `Lambda^{s->s+1}` and `r_i = tilde-Omega[s_I^{(s+1)}]`

The transport hierarchy requires bottom-up aggregation `Lambda^{s->s+1}` (gauge-covariant averaging) and top-down transport realizing the shadows, with `Omega_{i,I} = exp(phi_i^{(s)}) exp(-phi_I^{(s+1)})` given explicitly (`PIFB:895, 897, 547-552`). Code is ABSENT: `build_belief_transport` and `compute_transport_operators` (`transport.py:384`) build only intra-scale Omega from a single `(B,N,n_gen)` phi field; the only barycenter (`prior_bank.barycenter_r_`, `:227-265`) is by its own docstring a same-scale untransported uniform-weight centroid; the TODOs at `model.py:735-738` and `prior_bank.py:197-199` confirm the scale-(s+1) object is not built. **Effort XL** (a whole scale-(s+1) data structure plus the aggregation operator; the cocycle infrastructure is reusable). **Strength: mandated** for the full theory but explicitly deferred in the implemented single-scale simplification (`PIFB:554, 924-926`); the frozen global `r` is the sanctioned `s_max` boundary, so the pure path is not violated. This is the same gap as the cross-scale shadow prior above — they are one unit.

#### Consensus / coherence detector (Gibbs `Gamma = P * C_q * C_s` threshold)

Meta-agent formation is triggered by a threshold detector: belief coherence `C_q = exp(-avg post-transport belief KL / tau_q)`, model coherence `C_s`, presence `P`, combined as `Gamma = P*C_q*C_s in [0,1]`, forming a meta-agent when `Gamma > Gamma_min` and `|I| >= N_min` (`PIFB:2211, 2216`, verbatim). Code is ABSENT: grep for `cluster_gibbs/Gamma_min/N_min/tau_q/presence/chi_i` returns nothing; no config fields exist; the manuscript marks it "Implemented" only in a separate `MAgent_Model/gauge_agent/meta_agents.py` repo not present here. Building it is a self-contained module computing per-cluster post-transport pairwise KL (reusing `free_energy.pairwise_energy + geometry/transport`), the bounded Gibbs score, and a connected-component cluster selector, gated by new config fields. **Effort M.** **Strength:** the manuscript's own framing is "candidate-selection surrogate," "heuristic and partial rather than a derivation" (`PIFB:2207, 2216`), so endorsed-canonical, not a derived theorem. P2 because it is the entry gate to the (wholesale absent) tower; on its own it produces a score with no downstream consumer.

Once a cluster is licensed, the parent is the gauge-covariant forward-KL barycenter: `mu_I = sum_i w_i Omega_{I,i}[mu_i] / sum w_i`, `Sigma_I` the transported-covariance pool, and `phi_I = sum_i w_i phi_i / sum w_i` (first-order BCH of the Karcher mean) — `PIFB:2226, 2229, 2236`, verbatim. Code is ABSENT: the only barycenter is the uniform-over-vocab `prior_bank.barycenter_r_`; `geometry/lie_ops` has `matrix_exp` but no principal `matrix_log` of a group element (only SPD-covariance `logm` in `retraction.py:240-329`), so a small primitive must be added. Building it transports constituent `(mu,Sigma,phi)` into a common frame, forms the coherence-weighted forward-KL barycenter and the log-Euclidean phi average, emitting a parent `BeliefState+s`. **Effort L.** **Strength: mandated** but scoped — the full GL+(K) frame barycenter is itself flagged TODO at `PIFB:2202` (only the compact-SO(N) / first-order-BCH form is well-posed, and vfe3's default group is `glk`). Depends on the consensus detector for membership and weights.

#### Wilson holonomy observable `Re Tr(H)` and per-head decomposition  [BUILT 2026-06-14]

The gauge-invariant Regime-II observable is `W_ijk = Re Tr(H_ijk)`, with a Wilson-style penalty `S = beta sum_ijk (1 - W_ijk/K)` and a per-head factorization `H = oplus_h H^{(h)}`, `W = sum_h W^{(h)}` (`PIFB:862-869, 886`; `GL(K)_attention.tex:665`). Code is PARTIAL: `metrics.py:617-618` (`holonomy_deviation_sampled`) forms `h = omega[i,j] @ omega[j,k] @ omega[k,i]` then takes only the unsquared Frobenius `||h - I||_F`; `curvature_field` (`:631-644`) likewise; grep finds no `Re Tr` observable, no holonomy penalty, no per-head decomposition. Building it adds `Re Tr(H_ijk)` alongside the existing norm (the triangle products are already materialized) plus a per-head-sliced variant, and optionally an opt-in `holonomy_penalty` term reading the regime_ii Omega. **Effort S.** **Strength:** the trace form is the manuscript's canonical observable, but `PIFB:874` calls the Frobenius variant "equivalent up to bounded reparameterization" and points the penalty at the old codebase, so the genuinely mandated missing piece is the per-head decomposition. The diagnostic is degenerate on the default flat cocycle and acquires content only under the opt-in regime_ii connection.

#### General / non-Gaussian exponential-family belief (a second registered family)  [BUILT 2026-06-14 — non-EF route]

The framework is not Gaussian-specific: GL(K) invariance of f-divergences holds for any exponential family (`PIFB:938`, verbatim; `:1007`). Code is PARTIAL/seam-built: `families/base.py:217-266` (`_renyi_from_log_partition`) is a complete generic Bregman/Renyi-from-`A` path, dispatched whenever a family lacks `renyi_closed_form` (`:291-294`), and `config.py:722` validates `family` against the live registry — but the only `@register_family` decorations are the two Gaussians (`gaussian.py:28, 191`); the generic path is exercised only by test fixtures.
